Data Transfer Object (DTO) class for representing comprehensive log entry details in the OpenFrame platform's logging system.

## Key Components

- **LogDetails** - Main DTO class with Lombok annotations for automatic getter/setter generation
- **Core Identifiers** - `toolEventId`, `eventType`, `toolType` for log classification
- **Organizational Context** - `organizationId`, `organizationName`, `userId` for multi-tenant support
- **Device Information** - `deviceId`, `hostname` for asset tracking
- **Log Metadata** - `severity`, `timestamp`, `ingestDay` for log management
- **Content Fields** - `message`, `details` for log payload data

## Usage Example

```java
// Create a new log details entry
LogDetails logEntry = LogDetails.builder()
    .toolEventId("evt_123456")
    .eventType("SECURITY_ALERT")
    .toolType("ANTIVIRUS")
    .severity("HIGH")
    .userId("user_789")
    .deviceId("device_456")
    .hostname("workstation-01")
    .organizationId("org_123")
    .organizationName("Acme Corp")
    .message("Malware detected and quarantined")
    .timestamp("2024-01-15T10:30:00Z")
    .ingestDay("2024-01-15")
    .details("{\"file\": \"/path/to/file\", \"action\": \"quarantine\"}")
    .build();

// Access log information
String severity = logEntry.getSeverity();
String organization = logEntry.getOrganizationName();
```

This DTO enables structured logging across MSP tools, supporting the OpenFrame platform's unified log aggregation and analysis capabilities with proper organizational isolation and device tracking.
