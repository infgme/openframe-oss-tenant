MeshCentral desktop remote control implementation with full KVM (keyboard, video, mouse) support through canvas-based input handling and binary protocol communication.

## Key Components

- **DisplayInfo**: Interface defining display properties (id, position, dimensions, primary status)
- **DesktopInputHandlers**: Type defining required methods for desktop interaction including attach/detach, input handling, and multi-display support
- **MeshDesktop**: Main class implementing the desktop control protocol with canvas event handling, binary message encoding, and tile-based rendering

## Key Features

- **Input Handling**: Complete mouse and keyboard event capture with proper coordinate mapping and button state tracking
- **Binary Protocol**: MeshCentral-compatible message encoding for KVM commands including initialization, compression settings, and input events
- **Multi-Display Support**: Display enumeration and switching capabilities for remote systems with multiple monitors
- **Tile-Based Rendering**: Asynchronous image decoding with concurrent processing limits and draw queue management
- **State Management**: Proper cleanup, view-only mode, and pressed key tracking with window blur handling

## Usage Example

```typescript
import { MeshDesktop } from './meshcentral-desktop'

// Initialize desktop control
const desktop = new MeshDesktop()

// Set up WebSocket sender for binary protocol
desktop.setSender((data: Uint8Array) => {
  websocket.send(data)
})

// Attach to canvas element
const canvas = document.getElementById('remote-canvas') as HTMLCanvasElement
desktop.attach(canvas)

// Configure options
desktop.setViewOnly(false)
desktop.setSwapMouseButtons(false)
desktop.setUseRemoteKeyboardMap(true)

// Handle multi-display scenarios
desktop.onDisplayListChange((displays) => {
  console.log('Available displays:', displays)
})

// Switch to different display
desktop.switchDisplay(1)

// Clean up when done
desktop.detach()
```
