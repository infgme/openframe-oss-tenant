A comprehensive skeleton component that provides loading placeholders for the main application shell and dashboard sections, matching the exact layout structure of the real components.

## Key Components

- **AppShellSkeleton** - Main export that recreates the complete app layout (sidebar + header + content)
- **InfoCardSkeleton** - Placeholder for dashboard info cards with progress indicators
- **OnboardingStepCardSkeleton** - Skeleton for onboarding workflow step cards
- **OnboardingSkeleton** - Complete onboarding section with header and 5 step cards
- **DevicesSkeleton** - Devices overview section with title and 2 info cards grid
- **ChatsSkeleton** - Chats overview section with title and 4 info cards grid
- **OrganizationCardSkeleton** - Placeholder for organization cards
- **OrganizationsSkeleton** - Organizations section with 3 rows of org + info card combinations

## Usage Example

```typescript
import { AppShellSkeleton } from './app-shell-skeleton'

// Use as a loading fallback in layout
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <Suspense fallback={<AppShellSkeleton />}>
      <AppShell>
        {children}
      </AppShell>
    </Suspense>
  )
}

// Or during session checking
function AuthWrapper({ children }: { children: React.ReactNode }) {
  const { isLoading } = useSession()
  
  if (isLoading) {
    return <AppShellSkeleton />
  }
  
  return <>{children}</>
}
```

The skeleton maintains pixel-perfect dimensions (w-56 sidebar, h-14 header, p-6 pt-0 main content) and includes proper ARIA labeling for accessibility during loading states.
