A DTO class that encapsulates filtering criteria and options for device queries in the OpenFrame MSP platform, supporting multi-dimensional filtering across device attributes and providing result counts.

## Key Components

- **statuses** - List of device status filter options (active, inactive, etc.)
- **deviceTypes** - List of device type filter options (server, workstation, mobile, etc.)
- **osTypes** - List of operating system filter options (Windows, Linux, macOS, etc.)
- **organizationIds** - List of organization filter options for multi-tenant filtering
- **tags** - List of tag-based filter options for custom categorization
- **filteredCount** - Total count of devices matching the applied filters

## Usage Example

```java
// Building device filters for a search query
DeviceFilters filters = DeviceFilters.builder()
    .statuses(Arrays.asList(
        new DeviceFilterOption("active", "Active Devices"),
        new DeviceFilterOption("maintenance", "Under Maintenance")
    ))
    .deviceTypes(Arrays.asList(
        new DeviceFilterOption("server", "Servers"),
        new DeviceFilterOption("workstation", "Workstations")
    ))
    .osTypes(Arrays.asList(
        new DeviceFilterOption("windows", "Windows"),
        new DeviceFilterOption("linux", "Linux")
    ))
    .organizationIds(Arrays.asList(
        new DeviceFilterOption("org1", "Customer A"),
        new DeviceFilterOption("org2", "Customer B")
    ))
    .filteredCount(145)
    .build();

// Using filters in a device query service
List<Device> devices = deviceService.findDevices(filters);
```

This class is typically used in conjunction with device management APIs to provide flexible filtering capabilities across the MSP platform's device inventory.
