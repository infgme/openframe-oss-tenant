A React modal component for configuring Single Sign-On (SSO) providers with OAuth credentials, domain allowlisting, and provider-specific settings like Microsoft single-tenant authentication.

## Key Components

- **SsoConfigModal** - Main modal component with form state management and validation
- **SsoConfigModalProps** - TypeScript interface defining component props including callback handlers
- **Form Fields** - OAuth Client ID, Client Secret with visibility toggle, and Microsoft Tenant ID
- **Domain Allowlist** - Auto-provisioning controls with email domain validation (feature-flagged)
- **URL Management** - Dynamic redirect URL generation with copy-to-clipboard functionality

## Usage Example

```typescript
import { SsoConfigModal } from './edit-sso-config-modal'

function SSOSettings() {
  const [isModalOpen, setIsModalOpen] = useState(false)

  const handleSsoSubmit = async (data: {
    clientId: string
    clientSecret: string
    msTenantId?: string | null
    autoProvisionUsers?: boolean
    allowedDomains?: string[]
  }) => {
    // Save SSO configuration to backend
    await saveSsoConfig(data)
  }

  return (
    <SsoConfigModal
      isOpen={isModalOpen}
      onClose={() => setIsModalOpen(false)}
      providerKey="microsoft"
      providerDisplayName="Microsoft Azure AD"
      isEnabled={true}
      initialClientId="existing-client-id"
      initialAutoProvisionUsers={false}
      initialAllowedDomains={['company.com']}
      onSubmit={handleSsoSubmit}
      onDisable={async () => await disableSso()}
    />
  )
}
```

The modal adapts its layout based on feature flags - expanding to a 2-column layout when domain allowlisting is enabled, and includes provider-specific controls like Microsoft's single-tenant mode.
