A minimal message structure for machine heartbeat signals where the machine identifier is derived from the message topic and timestamp is server-generated.

## Key Components

- **`MachineHeartbeatMessage`** - Empty struct representing a heartbeat signal
- **`new()`** - Constructor method that creates a new heartbeat message instance
- **Serde traits** - Automatic JSON serialization/deserialization with camelCase field naming

## Usage Example

```rust
use machine_heartbeat_message::MachineHeartbeatMessage;

// Create a new heartbeat message
let heartbeat = MachineHeartbeatMessage::new();

// Serialize to JSON (results in empty object)
let json = serde_json::to_string(&heartbeat)?;
println!("{}", json); // prints: {}

// The actual machine identification happens at the transport layer
// via topic names like "machine/heartbeat/{machine_id}"
```

The design follows a lightweight approach where the heartbeat payload is intentionally empty since contextual information (machine ID, timestamp) is handled by the messaging infrastructure rather than embedded in the message body.
