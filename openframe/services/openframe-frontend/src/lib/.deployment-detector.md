Utility for detecting deployment environment based on hostname patterns to determine if OpenFrame is running as cloud, self-hosted, or development instance.

## Key Components

- **`detectDeployment()`** - Main detection function that analyzes hostname and returns complete deployment information
- **`DeploymentType`** - Type union defining possible deployment environments ('cloud' | 'self-hosted' | 'development')
- **`DeploymentInfo`** - Interface containing deployment type, boolean flags, and hostname
- **Quick check functions** - `isCloud()`, `isSelfHosted()`, `isDevelopment()` for simple boolean checks
- **`getDeploymentType()`** - Returns just the deployment type string

The utility uses predefined hostname patterns:
- **Cloud**: `openframe.ai`, `auth.openframe.ai`, `app.openframe`
- **Development**: `localhost`, `127.0.0.1`, `0.0.0.0`, `.local`
- **Self-hosted**: Any other hostname (default fallback)

## Usage Example

```typescript
import { detectDeployment, isCloud, getDeploymentType } from './deployment-detector'

// Get complete deployment information
const deployment = detectDeployment()
console.log(deployment.type) // 'cloud' | 'self-hosted' | 'development'
console.log(deployment.hostname) // Current hostname

// Quick checks for conditional logic
if (isCloud()) {
  // Enable cloud-specific features
  enableCloudAnalytics()
}

// Simple type check
const deploymentType = getDeploymentType()
if (deploymentType === 'self-hosted') {
  // Show self-hosted configuration options
}
```
