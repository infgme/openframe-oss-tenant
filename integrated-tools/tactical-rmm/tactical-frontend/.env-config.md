A runtime environment configuration file that exposes environment variables to the browser through a global `window._env_` object with template placeholders for dynamic substitution.

## Key Components

- **`window._env_`** - Global configuration object containing runtime environment variables
- **`PROD_URL`** - Production API URL constructed from protocol and host variables
- **`NODE_ENV`** - Application environment mode (development/production)
- **`DOCKER_BUILD`** - Docker build configuration flag
- **`WS_PROTO`** - WebSocket protocol specification

## Usage Example

This configuration file is typically processed during build/deployment to replace template variables with actual values:

```javascript
// After template substitution, the file might look like:
window._env_ = {
    PROD_URL: "https://api.flamingo.com",
    NODE_ENV: "production",
    DOCKER_BUILD: "true",
    WS_PROTO: "wss"
};

// Access in application code:
const apiUrl = window._env_.PROD_URL;
const isProduction = window._env_.NODE_ENV === 'production';
const websocketUrl = `${window._env_.WS_PROTO}://api.flamingo.com/ws`;
```

The template variables (`${VARIABLE_NAME}`) are replaced by build tools or deployment scripts with actual environment-specific values, allowing the same configuration file to work across different deployment environments.
