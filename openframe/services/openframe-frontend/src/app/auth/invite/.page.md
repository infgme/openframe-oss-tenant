This Next.js page component handles user invitation acceptance for the OpenFrame authentication system, supporting both form-based and SSO registration workflows.

## Key Components

- **Invitation Validation**: Validates invitation ID from URL parameters and handles expired/invalid invitations
- **Dual Registration Methods**: Supports both traditional form signup and SSO provider authentication
- **Tenant Management**: Handles user switching between different tenant organizations
- **Form State Management**: Manages user input for personal details and password validation
- **Loading States**: Provides skeleton UI during data fetching and API operations
- **Error Handling**: Comprehensive error handling with toast notifications

## Usage Example

```typescript
// URL structure for invitation links
// /auth/invite?id=invitation-uuid-here

// Component handles multiple scenarios:
// 1. Valid invitation - shows registration form
// 2. Invalid/expired invitation - shows error message
// 3. User exists in another tenant - prompts for tenant switch

// API integration example
const response = await authApiClient.acceptInvitation({
  invitationId: 'uuid',
  firstName: 'John',
  lastName: 'Doe',
  password: 'password123',
  switchTenant: false
})
```

The component integrates with the OpenFrame authentication system, providing a complete invitation acceptance flow with proper validation, error handling, and user feedback through toast notifications.
