Defines TypeScript interfaces for GraphQL dialog data structures and query options in the Flamingo MSP platform, supporting paginated dialog retrieval with machine owner information.

## Key Components

**Core Interfaces:**
- `DialogNode` - Individual dialog entity with status, owner machine details, and timestamps
- `DialogEdge` - Cursor-based pagination wrapper for dialog nodes
- `DialogConnection` - Complete paginated response with edges and page info
- `DialogsResponse` - GraphQL response wrapper for dialog collections
- `DialogResponse` - GraphQL response wrapper for single dialog queries
- `UseMingoDialogsOptions` - Configuration options for dialog hooks with search and pagination

## Usage Example

```typescript
import { DialogsResponse, UseMingoDialogsOptions } from './dialog.types'

// Configure dialog query options
const options: UseMingoDialogsOptions = {
  enabled: true,
  search: "server maintenance",
  limit: 20
}

// Process dialogs response
function processDialogs(response: DialogsResponse) {
  const { edges, pageInfo } = response.data.dialogs
  
  edges.forEach(edge => {
    const dialog = edge.node
    console.log(`Dialog ${dialog.title} - Status: ${dialog.status}`)
    
    if (dialog.owner?.machine) {
      console.log(`Machine: ${dialog.owner.machine.hostname}`)
    }
  })
  
  if (pageInfo.hasNextPage) {
    // Load more dialogs using endCursor
    console.log(`Next cursor: ${pageInfo.endCursor}`)
  }
}
```
