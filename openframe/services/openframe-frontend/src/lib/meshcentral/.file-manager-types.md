Type definitions for the MeshCentral file manager system, providing interfaces for file operations, transfers, and connection management in remote desktop environments.

## Key Components

- **FileEntry** - Represents files and directories with metadata (name, type, size, date)
- **DirectoryListing** - Server response containing directory contents
- **FileOperationRequest/Response** - Generic file operation communication
- **UploadRequest/DownloadRequest** - Specific transfer operation types
- **FileTransferProgress** - Progress tracking for uploads/downloads
- **BinaryHeader/BinaryAccumulator** - Binary protocol handling
- **FileManagerOptions** - Configuration interface with event callbacks
- **FileConnectionState** - Connection status enumeration
- **MeshRights/SiteRights** - Permission constants for file access

## Usage Example

```typescript
import type { FileManagerOptions, FileEntry, FileTransferProgress } from './file-manager-types'

const fileManagerConfig: FileManagerOptions = {
  nodeId: 'mesh-node-123',
  isRemote: true,
  consent: 7,
  onStateChange: (state) => {
    console.log('Connection state:', state)
  },
  onDirectoryChange: (files: FileEntry[]) => {
    const directories = files.filter(f => f.t === 2)
    const regularFiles = files.filter(f => f.t === 3)
    console.log(`Found ${directories.length} dirs, ${regularFiles.length} files`)
  },
  onTransferProgress: (progress: FileTransferProgress) => {
    const percent = (progress.progress * 100).toFixed(1)
    console.log(`${progress.file}: ${percent}% complete`)
  }
}
```
