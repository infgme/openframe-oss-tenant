Central export module that provides a unified interface for accessing all Zustand stores and their selectors across the Flamingo application. This barrel export pattern simplifies imports and improves code organization.

## Key Components

**Store Exports:**
- `useAuthStore` - Authentication state management store
- `useDevicesStore` - Device management and filtering store

**Type Exports:**
- `AuthState` - Authentication state interface
- `DevicesState`, `Device`, `DeviceFilter` - Device-related type definitions

**Selector Exports:**
- Authentication selectors: `selectUser`, `selectIsAuthenticated`, `selectAuthLoading`, `selectAuthError`
- Device selectors: `selectDevices`, `selectSelectedDevice`, `selectFilter`, `selectDevicesLoading`, `selectDevicesError`, `selectFilteredDevices`, `selectDeviceStats`

## Usage Example

```typescript
// Import stores and selectors from central location
import { 
  useAuthStore, 
  useDevicesStore,
  selectUser,
  selectFilteredDevices 
} from '@/stores'

function Dashboard() {
  // Use stores directly
  const { login, logout } = useAuthStore()
  const { setFilter, selectDevice } = useDevicesStore()
  
  // Use performance-optimized selectors
  const user = useAuthStore(selectUser)
  const filteredDevices = useDevicesStore(selectFilteredDevices)
  
  return (
    <div>
      <h1>Welcome, {user?.name}</h1>
      <DeviceList devices={filteredDevices} />
    </div>
  )
}
```

The selector exports enable fine-grained subscriptions to specific state slices, preventing unnecessary re-renders when unrelated state changes occur.
