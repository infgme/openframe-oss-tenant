A WebSocket-based tunnel client for establishing secure connections to MeshCentral devices, supporting terminal sessions and remote desktop connections with automatic reconnection and authentication handling.

## Key Components

- **`MeshTunnel`** - Main class managing WebSocket connections to MeshCentral devices
- **`TunnelState`** - State enum (0: stopped, 1: connecting, 2: open, 3: connected)
- **`TunnelOptions`** - Configuration for terminal dimensions and login requirements
- **`TunnelCallbacks`** - Event handlers for data, state changes, and control messages

## Usage Example

```typescript
import { MeshTunnel } from './meshcentral-tunnel'

const tunnel = new MeshTunnel({
  authCookie: 'your-auth-cookie',
  nodeId: 'device-node-id',
  protocol: 1, // Terminal protocol
  options: {
    cols: 80,
    rows: 24,
    requireLogin: true
  },
  onData: (data) => {
    console.log('Received data:', data)
  },
  onStateChange: (state) => {
    console.log('Tunnel state changed:', state)
  },
  onConsoleMessage: (msg) => {
    console.log('Console message:', msg)
  }
})

// Start the tunnel connection
tunnel.start()

// Send text to the remote device
tunnel.sendText('ls -la\n')

// Check connection status
if (tunnel.isConnected()) {
  tunnel.sendBinary(new Uint8Array([1, 2, 3]))
}

// Clean up
tunnel.stop()
```

The class handles WebSocket lifecycle management, authentication tokens, automatic reconnection with exponential backoff, and protocol handshaking for MeshCentral device communication.
