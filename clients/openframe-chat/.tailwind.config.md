Comprehensive Tailwind CSS configuration for the Flamingo OpenFrame platform, featuring dark mode support, custom design tokens, and integration with OpenFrame frontend components.

## Key Components

- **Dark Mode**: Class-based dark mode implementation
- **Content Paths**: Includes OpenFrame frontend core components from node_modules
- **Safelist**: Preserves CMS/database-generated classes that aren't detected at build time
- **ODS Design System**: Complete OpenFrame Design System (ODS) color palette and semantic tokens
- **Responsive Typography**: Fluid and responsive font sizing system
- **Extended Spacing**: Component and section-based spacing with responsive variants
- **Custom Breakpoints**: Optimized breakpoints for 2-column (860px) and 3-column (1550px) layouts
- **Design Tokens**: Shadows, transitions, z-index values, and animations

## Usage Example

```typescript
// Using ODS semantic colors
<div className="bg-ods-card border border-ods-border">
  <h2 className="text-ods-text-primary text-fluid-h2">Title</h2>
  <p className="text-ods-text-secondary text-responsive-body-base">Content</p>
  <button className="bg-ods-accent hover:bg-ods-accent-hover">Action</button>
</div>

// Using responsive spacing and typography
<section className="py-responsive-section-md px-responsive-component-lg">
  <h1 className="font-heading text-responsive-h1 mb-fluid-component-md">
    Responsive Heading
  </h1>
</section>

// Using custom breakpoints
<div className="grid grid-cols-1 md:grid-cols-2 xl:grid-cols-3">
  <!-- Responsive grid layout -->
</div>
```

The configuration extends Tailwind with Flamingo's complete design system, ensuring consistent styling across the platform while maintaining compatibility with both custom components and the OpenFrame frontend core library.
