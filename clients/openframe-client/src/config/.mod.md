Configuration module for OpenFrame client that defines application-wide settings and constants for various subsystems. Provides serializable configuration structures with sensible defaults for logging and other operational parameters.

## Key Components

- **`Configuration`** - Main configuration structure containing all subsystem configs
- **`LoggingConfig`** - Logging-specific settings including level, format, and rotation parameters
- **`update_config`** - Submodule for update-related configuration (referenced but not shown)
- **Default implementation** - Provides sensible defaults for all configuration values

## Configuration Structure

The configuration uses a hierarchical structure with the main `Configuration` struct containing specialized config objects:

```rust
use openframe_config::Configuration;

// Use default configuration
let config = Configuration::default();
println!("Log level: {}", config.logging.level); // "info"

// Or deserialize from JSON/YAML
let config: Configuration = serde_json::from_str(json_config)?;
```

## Default Settings

The default configuration provides production-ready settings:
- **Log level**: `info` 
- **Log format**: JSON enabled for structured logging
- **Log rotation**: 10MB file size limit with 5 maximum files
- All settings are serializable via serde for external configuration files

## Usage Example

```rust
use serde_json;
use openframe_config::Configuration;

// Create with defaults
let mut config = Configuration::default();

// Modify settings
config.logging.level = "debug".to_string();
config.logging.json = false;

// Serialize to JSON
let json_config = serde_json::to_string_pretty(&config)?;
println!("{}", json_config);
```
