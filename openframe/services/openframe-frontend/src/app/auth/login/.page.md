A Next.js client-side page component that dynamically imports and renders the login page with server-side rendering disabled for enhanced client-side authentication handling.

## Key Components

- **Login** - Default exported React component that serves as the login route page
- **LoginPage** - Dynamically imported login page component from the auth module
- **Dynamic Import** - Uses Next.js `dynamic()` with SSR disabled for client-side only rendering

## Usage Example

```typescript
// This page is automatically mounted by Next.js routing at /login
// The dynamic import ensures the LoginPage component only loads on the client

// Example of similar dynamic import pattern:
const DynamicComponent = dynamic(
  () => import('@app/components/client-only-component'),
  { 
    ssr: false,
    loading: () => <p>Loading...</p>
  }
)

export default function MyPage() {
  return <DynamicComponent />
}
```

The `ssr: false` configuration is particularly important for authentication pages where client-side state management, browser APIs, or third-party auth libraries need to run exclusively in the browser environment. This pattern prevents hydration mismatches and ensures consistent behavior across different authentication flows.
