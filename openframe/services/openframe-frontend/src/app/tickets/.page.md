A tickets management page component that conditionally renders based on SaaS tenant mode and provides a full-screen tickets interface within the app layout.

## Key Components

- **Tickets**: Default export page component with mode-based conditional rendering
- **Dynamic routing**: Forces dynamic rendering with `'force-dynamic'` export
- **Mode checking**: Uses `isSaasTenantMode()` to control access and rendering
- **Layout structure**: Combines `AppLayout` with `ContentPageContainer` for consistent UI
- **TicketsView**: Core tickets management component rendered without padding or header

## Usage Example

```typescript
// This page is automatically routed by Next.js App Router
// Access via: /tickets (when in SaaS tenant mode)

// The component structure provides:
// - AppLayout wrapper for consistent navigation/sidebar
// - ContentPageContainer without padding/header for full-screen tickets
// - Automatic redirect to /dashboard if not in SaaS tenant mode

// Example of the conditional rendering pattern:
useEffect(() => {
  if (!isSaasTenantMode()) {
    router.replace('/dashboard')
    return
  }
}, [router])

// Component only renders TicketsView when in correct mode
if (!isSaasTenantMode()) {
  return null
}
```

The page implements a security pattern where access is restricted to SaaS tenant environments, automatically redirecting users to the dashboard if accessed in the wrong application mode.
