A service that downloads and extracts agent binaries from GitHub releases with automatic fallback to jsDelivr CDN when GitHub rate limits are encountered.

## Key Components

- **`GithubDownloadService`** - Main service struct for handling downloads and extractions
- **`download_and_extract`** - Downloads archives and extracts specific agent binaries 
- **`download_with_retry`** - Implements retry logic with CDN fallback on rate limits
- **`extract_from_zip`** - ZIP extraction (Windows only)
- **`extract_from_tar_gz`** - tar.gz extraction (Unix/Linux only)
- **`github_to_cdn_url`** - Converts GitHub URLs to jsDelivr CDN URLs
- **`find_config_for_current_os`** - Selects appropriate download configuration

## Usage Example

```rust
use reqwest::Client;
use crate::models::download_configuration::DownloadConfiguration;

// Create service instance
let http_client = Client::new();
let service = GithubDownloadService::new(http_client);

// Download and extract agent binary
let config = DownloadConfiguration {
    link: "https://github.com/owner/repo/releases/download/v1.0/agent.zip".to_string(),
    file_name: "agent.zip".to_string(),
    agent_file_name: "agent.exe".to_string(),
    // ... other fields
};

let binary_bytes = service.download_and_extract(&config).await?;

// Find OS-specific configuration
let configs = vec![/* download configurations */];
let os_config = GithubDownloadService::find_config_for_current_os(&configs)?;
```

The service automatically handles GitHub rate limits by falling back to jsDelivr CDN, supports both ZIP (Windows) and tar.gz (Unix) archives, and includes comprehensive error handling with retry logic.
