Configuration structure for OpenFrame agent authentication and machine identification, handling OAuth tokens and machine-specific credentials.

## Key Components

### AgentConfiguration Struct
- **machine_id**: Unique identifier for the agent's host machine
- **client_id**: OAuth client identifier for API authentication
- **client_secret**: OAuth client secret for secure authentication
- **access_token**: Current OAuth access token for API requests
- **refresh_token**: Token used to refresh expired access tokens

### Traits Implemented
- `Debug`: Enables debug printing of the configuration
- `Clone`: Allows deep copying of the configuration
- `Serialize/Deserialize`: JSON serialization support via serde
- `PartialEq`: Enables equality comparisons between configurations
- `Default`: Provides empty default values for all fields

## Usage Example

```rust
use agent_configuration::AgentConfiguration;

// Create a new configuration with default empty values
let mut config = AgentConfiguration::default();

// Set authentication credentials
config.machine_id = "workstation-001".to_string();
config.client_id = "flamingo_client_123".to_string();
config.client_secret = "secret_key_456".to_string();
config.access_token = "eyJhbGciOiJIUzI1NiIs...".to_string();

// Serialize to JSON for storage
let json_config = serde_json::to_string(&config)?;

// Deserialize from JSON
let loaded_config: AgentConfiguration = serde_json::from_str(&json_config)?;
```

This configuration structure is typically used for persisting agent credentials and managing OAuth token lifecycle in the OpenFrame MSP platform.
