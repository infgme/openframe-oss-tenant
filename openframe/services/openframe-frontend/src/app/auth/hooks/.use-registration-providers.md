A React hook that fetches and manages SSO (Single Sign-On) provider configurations for user registration, providing loading states and error handling for authentication provider setup.

## Key Components

- **`SSOProvider`** - Interface defining provider structure with name and enabled status
- **`RegistrationProvidersResponse`** - API response type containing available provider list
- **`useRegistrationProviders`** - Main hook that fetches SSO providers and manages state
- **State Management** - Handles providers list, loading status, and error states
- **Auto-fetch Effect** - Automatically loads providers on component mount

## Usage Example

```typescript
import { useRegistrationProviders } from './use-registration-providers'

function RegistrationForm() {
  const { providers, loading, error } = useRegistrationProviders()

  if (loading) return <div>Loading SSO providers...</div>
  if (error) return <div>Error: {error}</div>

  return (
    <div>
      <h2>Sign up with:</h2>
      {providers.map(({ provider, enabled }) => (
        <button 
          key={provider}
          disabled={!enabled}
          onClick={() => initiateSSO(provider)}
        >
          Continue with {provider}
        </button>
      ))}
    </div>
  )
}
```

The hook automatically transforms the API response into a standardized format where all providers are marked as enabled by default, making it easy to conditionally render authentication options in registration flows.
