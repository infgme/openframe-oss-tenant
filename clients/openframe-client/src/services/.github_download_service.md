A service that downloads and extracts GitHub release binaries with retry logic, CDN fallback, and platform-specific archive handling.

## Key Components

- **GithubDownloadService**: Main service struct for downloading GitHub releases
- **download_and_extract()**: Downloads archives and extracts specific binary files
- **download_with_retry()**: Implements retry logic with jsDelivr CDN fallback on rate limits
- **extract_from_zip()**: Platform-specific ZIP extraction (Windows only)
- **extract_from_tar_gz()**: Platform-specific tar.gz extraction (Unix/Linux only)
- **find_config_for_current_os()**: Selects appropriate download configuration for current platform
- **download_and_save()**: Complete download and file system persistence workflow

## Usage Example

```rust
use reqwest::Client;
use github_download_service::GithubDownloadService;
use crate::models::download_configuration::DownloadConfiguration;

// Initialize service
let client = Client::new();
let service = GithubDownloadService::new(client);

// Download and extract binary
let config = DownloadConfiguration {
    link: "https://github.com/owner/repo/releases/download/v1.0/binary.tar.gz".to_string(),
    file_name: "binary.tar.gz".to_string(),
    agent_file_name: "agent".to_string(),
    // ... other fields
};

let binary_bytes = service.download_and_extract(&config).await?;

// Or download directly to filesystem
let tool_path = Path::new("./tools");
let agent_path = Path::new("./agent");
let executable_name = service.download_and_save(&config, tool_path, agent_path).await?;
```

The service automatically handles GitHub rate limiting by falling back to jsDelivr CDN, validates file sizes, and sets appropriate executable permissions on Unix systems.
