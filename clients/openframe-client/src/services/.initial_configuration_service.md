A service for managing the initial configuration file that contains bootstrap settings for the Flamingo agent, including server connection details, authentication keys, and operational mode settings.

## Key Components

- **InitialConfigurationService** - Main service struct that handles reading, writing, and accessing configuration data
- **new()** - Constructor that initializes the service with a secured directory path for the config file
- **Configuration Getters** - Methods to retrieve specific config values: `get_initial_key()`, `get_server_url()`, `is_local_mode()`, `get_org_id()`, `get_local_ca_cert_path()`
- **get()** - Private method that loads and deserializes the JSON configuration file
- **save()** - Persists configuration changes to disk as formatted JSON
- **clear_initial_key()** - Security method to remove the initial authentication key after use

## Usage Example

```rust
use crate::platform::directories::DirectoryManager;
use crate::services::InitialConfigurationService;

// Initialize the service
let directory_manager = DirectoryManager::new()?;
let config_service = InitialConfigurationService::new(directory_manager)?;

// Access configuration values
let server_url = config_service.get_server_url()?;
let org_id = config_service.get_org_id()?;
let is_local = config_service.is_local_mode()?;

println!("Connecting to: {} (Local mode: {})", server_url, is_local);

// Clear sensitive data after initial setup
config_service.clear_initial_key()?;
```

The service automatically ensures the secured directory exists and provides error context for all file operations, making it safe for production use in the Flamingo agent's bootstrap process.
