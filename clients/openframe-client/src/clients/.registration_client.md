HTTP client for agent registration with the Flamingo platform, handling authentication and JSON communication with the registration API endpoint.

## Key Components

- **RegistrationClient** - Main client struct containing HTTP client and base URL configuration
- **new()** - Constructor method for creating a new registration client instance
- **register()** - Async method to register an agent using initial key authentication and JSON payload

## Usage Example

```rust
use reqwest::Client;
use crate::registration_client::RegistrationClient;
use crate::models::AgentRegistrationRequest;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let http_client = Client::new();
    let client = RegistrationClient::new(
        "https://api.flamingo.run".to_string(),
        http_client
    )?;
    
    let registration_request = AgentRegistrationRequest {
        // populate with required fields
    };
    
    let response = client.register(
        "your-initial-key",
        registration_request
    ).await?;
    
    println!("Agent registered successfully: {:?}", response);
    Ok(())
}
```

The client uses `X-Initial-Key` header authentication and sends JSON payloads to the `/clients/api/agents/register` endpoint. Error handling includes status code validation and detailed error context through the `anyhow` crate.
