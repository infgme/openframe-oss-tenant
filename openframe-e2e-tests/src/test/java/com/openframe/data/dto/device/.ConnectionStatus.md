Enumeration defining the possible connection states for devices in the OpenFrame MSP platform.

## Key Components

- **CONNECTED** - Device is successfully connected and communicating
- **DISCONNECTED** - Device is offline or not reachable
- **ERROR** - Device connection has encountered an error state

## Usage Example

```java
import com.openframe.data.dto.device.ConnectionStatus;

// Setting device connection status
ConnectionStatus status = ConnectionStatus.CONNECTED;

// Checking connection state in conditional logic
if (status == ConnectionStatus.CONNECTED) {
    // Proceed with device operations
    performDeviceOperation();
} else if (status == ConnectionStatus.DISCONNECTED) {
    // Handle offline device
    scheduleReconnection();
} else if (status == ConnectionStatus.ERROR) {
    // Handle error state
    logErrorAndRetry();
}

// Using in device status reporting
public class DeviceMonitor {
    public void updateDeviceStatus(String deviceId, ConnectionStatus status) {
        switch (status) {
            case CONNECTED:
                markDeviceOnline(deviceId);
                break;
            case DISCONNECTED:
                markDeviceOffline(deviceId);
                break;
            case ERROR:
                flagDeviceError(deviceId);
                break;
        }
    }
}
```

This enum provides a standardized way to represent device connectivity states across the OpenFrame platform, enabling consistent status tracking and automated response handling for managed devices.
