A Zustand store that manages script entries for the Flamingo MSP platform, providing centralized state management for script listing, search functionality, and loading states with immutable updates.

## Key Components

- **`ScriptEntry`** - Interface defining the structure of script objects with metadata like name, description, supported platforms, and execution parameters
- **`ScriptsState`** - Main store interface containing scripts array, search term, loading state, error handling, and action methods
- **`useScriptsStore`** - Primary Zustand store hook with devtools and immer middleware for debugging and immutable state updates
- **Selector functions** - Optimized selectors (`selectLogs`, `selectSearch`, `selectIsLoading`, `selectError`) for preventing unnecessary re-renders

## Usage Example

```typescript
import { useScriptsStore, selectSearch } from './scripts-store'

// Component using the store
function ScriptsList() {
  const { scripts, search, isLoading, setSearch, setScripts } = useScriptsStore()
  
  // Using optimized selector to prevent re-renders
  const searchTerm = useScriptsStore(selectSearch)
  
  // Load scripts
  const loadScripts = async () => {
    useScriptsStore.getState().setLoading(true)
    try {
      const data = await fetchScripts()
      useScriptsStore.getState().setScripts(data)
    } catch (error) {
      useScriptsStore.getState().setError(error.message)
    }
  }
  
  return (
    <div>
      <input 
        value={search} 
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Search scripts..."
      />
      {isLoading ? <div>Loading...</div> : 
        <ScriptGrid scripts={scripts.filter(s => 
          s.name.toLowerCase().includes(search.toLowerCase())
        )} />
      }
    </div>
  )
}
```
