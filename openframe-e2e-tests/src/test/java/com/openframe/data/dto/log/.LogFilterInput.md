A data transfer object (DTO) for filtering log entries with various criteria including date ranges, event types, and organizational parameters.

## Key Components

**LogFilterInput** - Main DTO class with the following filter properties:
- `startDate` / `endDate` - Date range strings for temporal filtering
- `eventTypes` - List of specific event type identifiers
- `toolTypes` - List of tool/service type filters
- `severities` - List of log severity levels (e.g., ERROR, WARN, INFO)
- `organizationIds` - List of organization identifiers for multi-tenant filtering
- `deviceId` - Single device identifier for device-specific logs

The class uses Lombok annotations for automatic generation of constructors, getters/setters, and builder pattern support.

## Usage Example

```java
// Create filter for error logs from last week
LogFilterInput filter = LogFilterInput.builder()
    .startDate("2024-01-01")
    .endDate("2024-01-07")
    .severities(List.of("ERROR", "CRITICAL"))
    .eventTypes(List.of("SYSTEM_FAILURE", "SECURITY_BREACH"))
    .organizationIds(List.of("org-123", "org-456"))
    .deviceId("device-789")
    .build();

// Or create an empty filter for all logs
LogFilterInput allLogsFilter = new LogFilterInput();
```

This DTO is typically used in REST API endpoints or service methods that need to query logs with multiple optional filter criteria, supporting flexible log search and analysis across the OpenFrame platform.
