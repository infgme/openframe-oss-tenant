A GraphQL connection wrapper for paginated device data, providing edges and filtered count information for efficient data retrieval and display.

## Key Components

- **edges**: List of `DeviceEdge` objects containing device data and cursor information
- **filteredCount**: Total number of devices matching the current filter criteria
- **Lombok annotations**: Provides automatic generation of getters, setters, constructors, and builder pattern

## Usage Example

```java
// Building a device connection for paginated results
DeviceConnection connection = DeviceConnection.builder()
    .edges(deviceEdges)
    .filteredCount(totalDevices)
    .build();

// Accessing connection data
List<DeviceEdge> devices = connection.getEdges();
Integer totalCount = connection.getFilteredCount();

// Using in GraphQL resolver
public DeviceConnection getDevices(DeviceFilter filter, PaginationInput pagination) {
    List<DeviceEdge> edges = deviceService.getDeviceEdges(filter, pagination);
    Integer count = deviceService.getFilteredDeviceCount(filter);
    
    return DeviceConnection.builder()
        .edges(edges)
        .filteredCount(count)
        .build();
}
```

This DTO follows the GraphQL Connection specification pattern, enabling efficient pagination and filtering of device collections in the OpenFrame platform.
