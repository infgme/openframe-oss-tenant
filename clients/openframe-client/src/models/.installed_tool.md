This file defines data structures for representing tools that have been installed on managed endpoints in the Flamingo MSP platform. It tracks installation metadata, execution parameters, and status information for deployed tools.

## Key Components

### `ToolStatus` Enum
- **`Installed`** - Indicates the tool is successfully installed and available
- Default implementation returns `Installed` status

### `InstalledTool` Struct
- **`tool_agent_id`** - Unique identifier for the tool agent instance
- **`tool_id`** - Base tool identifier  
- **`tool_type`** - Category or type classification of the tool
- **`version`** - Installed version string
- **`session_type`** - Execution context (Service, Interactive, etc.)
- **`run_command_args`** - Command line arguments for tool execution
- **`tool_agent_id_command_args`** - Agent-specific command arguments
- **`uninstallation_command_args`** - Optional removal command parameters
- **`status`** - Current installation status
- **`executable_path`** - Optional path to the tool's executable relative to tool folder

## Usage Example

```rust
use crate::models::{InstalledTool, ToolStatus, SessionType};

// Create a new installed tool record
let installed_tool = InstalledTool {
    tool_agent_id: "agent-123".to_string(),
    tool_id: "monitoring-tool".to_string(),
    tool_type: "monitoring".to_string(),
    version: "1.2.3".to_string(),
    session_type: SessionType::Service,
    run_command_args: vec!["--daemon".to_string(), "--config=/etc/tool.conf".to_string()],
    tool_agent_id_command_args: vec!["--agent-id=agent-123".to_string()],
    uninstallation_command_args: Some(vec!["uninstall".to_string(), "--force".to_string()]),
    status: ToolStatus::Installed,
    executable_path: Some("bin/monitor".to_string()),
};

// Serialize to JSON for storage or transmission
let json = serde_json::to_string(&installed_tool)?;
```
