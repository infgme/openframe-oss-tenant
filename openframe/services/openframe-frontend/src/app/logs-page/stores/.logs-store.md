A Zustand store that manages system logs state with support for GraphQL pagination, search, filtering, and real-time log streaming in the Flamingo MSP platform.

## Key Components

- **LogEntry**: Complete log record interface with severity levels, device metadata, and organization context
- **LogEdge/PageInfo**: GraphQL cursor-based pagination structures for efficient log loading
- **LogFilters**: Available filter options for tool types, event types, severities, and organizations
- **useLogsStore**: Main Zustand store with Immer middleware for immutable state updates
- **Selector functions**: Optimized selectors for component re-render performance

## Usage Example

```typescript
import { useLogsStore, selectLogs, selectIsLoading } from './logs-store'

// Basic usage in a component
function LogsView() {
  const { logs, search, setSearch, setLoading } = useLogsStore()
  const isLoading = useLogsStore(selectIsLoading)
  
  return (
    <div>
      <input 
        value={search}
        onChange={(e) => setSearch(e.target.value)}
        placeholder="Search logs..."
      />
      {isLoading ? (
        <div>Loading logs...</div>
      ) : (
        logs.map(log => (
          <div key={log.toolEventId}>
            <span className={`severity-${log.severity.toLowerCase()}`}>
              {log.severity}
            </span>
            <span>{log.summary}</span>
          </div>
        ))
      )}
    </div>
  )
}

// Pagination with cursor-based loading
function LogsPagination() {
  const { pageInfo, appendEdges, setLoading } = useLogsStore()
  
  const loadMore = async () => {
    if (pageInfo?.hasNextPage) {
      setLoading(true)
      const newEdges = await fetchMoreLogs(pageInfo.endCursor)
      appendEdges(newEdges) // Automatically deduplicates
    }
  }
  
  return (
    <button 
      onClick={loadMore}
      disabled={!pageInfo?.hasNextPage}
    >
      Load More
    </button>
  )
}
```
