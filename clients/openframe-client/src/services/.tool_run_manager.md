Manages asynchronous execution of tools on different platforms, handling session management and process lifecycle for GUI and console applications.

## Key Components

- **Platform-specific launchers**: Windows session management with `get_active_user_session()`, `launch_process_in_console_session()`, and `launch_process_in_user_session()`
- **Session detection**: Enumerates Windows Terminal Services sessions to find active user sessions for proper GUI tool launching
- **Process token management**: Handles Windows security tokens for launching processes in user context from service context
- **Command building**: Constructs properly quoted command lines with arguments for cross-platform execution
- **Windows API integration**: Uses WTS (Windows Terminal Services) APIs for session enumeration and user token management

## Usage Example

```rust
// Windows session detection
let session_id = get_active_user_session();
if let Some(id) = session_id {
    println!("Found active session: {}", id);
}

// Launch process in user session
let command_path = "notepad.exe";
let args = vec!["document.txt".to_string()];
let result = launch_process_in_user_session(command_path, &args);

match result {
    Ok((pid, handle)) => {
        println!("Process launched with PID: {}", pid);
        // Handle process lifecycle
    }
    Err(e) => {
        eprintln!("Failed to launch: {}", e);
    }
}

// Console session launcher for service context
let (pid, handle) = launch_process_in_console_session("cmd.exe", &["/c", "echo hello"])?;
```

The file provides essential Windows-specific functionality for MSP tools that need to run GUI applications from service context, ensuring proper user session targeting and security token handling.
