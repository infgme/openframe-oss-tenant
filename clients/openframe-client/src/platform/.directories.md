Cross-platform directory management utility that provides a unified interface for managing platform-specific directories across Windows, macOS, and Linux, handling application support directories, logs, permissions, and ownership.

## Key Components

- **DirectoryManager**: Main struct providing cross-platform directory management with system-level, user-level, and development modes
- **DirectoryError**: Comprehensive error type for directory operations including creation, permission, and validation failures
- **Directory Functions**: Platform-specific path resolution functions (`get_app_support_directory()`, `get_logs_directory()`, `get_secured_directory()`)
- **Permission Functions**: Cross-platform permission setting with platform-specific ownership and access control

## Usage Example

```rust
use openframe_core::directories::{DirectoryManager, DirectoryError};

// Create a standard system-level directory manager
let dir_manager = DirectoryManager::new();

// For development environments (uses user directories to avoid permission issues)
let dev_manager = DirectoryManager::for_development();

// Create directories with proper permissions
match dir_manager.ensure_directories().await {
    Ok(_) => println!("Directories created successfully"),
    Err(DirectoryError::PermissionDenied(path)) => {
        eprintln!("Need admin privileges for: {}", path.display());
    }
    Err(e) => eprintln!("Directory error: {}", e),
}

// Get platform-specific paths
let logs_dir = dir_manager.logs_dir();
let app_support = dir_manager.app_support_dir();
```

The module automatically handles platform differences like Windows ProgramData, macOS Library paths, and Linux /var directories, with appropriate permission models (ACLs on Windows, Unix permissions on macOS/Linux).
