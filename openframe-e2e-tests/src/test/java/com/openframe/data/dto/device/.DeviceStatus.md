Enumeration defining the possible operational and lifecycle states for devices managed within the OpenFrame platform.

## Key Components

- **Lifecycle States**: `PENDING`, `ACTIVE`, `INACTIVE`, `DECOMMISSIONED`, `DELETED`, `ARCHIVED`
- **Operational States**: `ONLINE`, `OFFLINE`, `MAINTENANCE`

## Usage Example

```java
// Setting device status during provisioning
Device device = new Device();
device.setStatus(DeviceStatus.PENDING);

// Updating status when device comes online
if (deviceHeartbeatReceived()) {
    device.setStatus(DeviceStatus.ONLINE);
}

// Transitioning through lifecycle states
switch (device.getStatus()) {
    case PENDING:
        // Device awaiting configuration
        break;
    case ACTIVE:
        // Device ready for operations
        break;
    case MAINTENANCE:
        // Device temporarily unavailable
        break;
    case DECOMMISSIONED:
        // Device permanently retired
        break;
}

// Filtering active devices
List<Device> activeDevices = devices.stream()
    .filter(d -> d.getStatus() == DeviceStatus.ACTIVE)
    .collect(Collectors.toList());
```

This enum provides standardized status tracking for device management workflows, enabling proper state transitions and filtering capabilities across the OpenFrame MSP platform.
