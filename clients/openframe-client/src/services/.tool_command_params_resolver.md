Resolves placeholders in tool command parameters by substituting client configuration values and asset paths with their actual runtime values.

## Key Components

- **ToolCommandParamsResolver** - Main resolver struct that processes command arguments and replaces placeholders
- **ASSETS_PATH_REGEX** - Static regex for matching asset path placeholders like `${client.assetPath.osquery}`
- **process()** - Core method that resolves all placeholder types in command arguments
- **Standard placeholders** - Predefined constants for server URL, OpenFrame secret, and token path
- **Dynamic asset resolution** - Handles tool-specific asset path placeholders

## Usage Example

```rust
use tool_command_params_resolver::ToolCommandParamsResolver;

// Initialize the resolver
let resolver = ToolCommandParamsResolver::new(
    directory_manager,
    initial_configuration_service
);

// Process command with placeholders
let command_args = vec![
    "--server=${client.serverUrl}".to_string(),
    "--token=${client.openframeTokenPath}".to_string(),
    "--binary=${client.assetPath.osquery}".to_string(),
];

let resolved_args = resolver.process("osquery-agent", command_args)?;

// Results in something like:
// ["--server=https://api.flamingo.com", 
//  "--token=/secure/shared_token.enc",
//  "--binary=/assets/osquery-agent/osquery"]
```

The resolver handles both static placeholders (server URL, secrets, token paths) and dynamic asset path placeholders that vary by tool agent, enabling flexible command parameterization across different deployment environments.
