Custom hook for managing tenant domain configuration and auto-provisioning settings in the Flamingo platform.

## Key Components

- **`TenantDomainInfo`** - Type for tenant domain data including domain string, generic flag, and auto-allow setting
- **`UpdateSharedAutoProvisionPayload/Response`** - Types for auto-provision update operations with error handling
- **`useTenantDomain()`** - Main hook providing domain fetching and updating capabilities with loading states
- **`fetchTenantDomain()`** - Retrieves current tenant domain configuration from the API
- **`updateSharedAutoProvision()`** - Updates the auto-provisioning setting with comprehensive error handling

## Usage Example

```typescript
import { useTenantDomain } from './hooks/use-tenant-domain'

function TenantSettings() {
  const { 
    fetchTenantDomain, 
    updateSharedAutoProvision, 
    isLoading, 
    isUpdating 
  } = useTenantDomain()

  const handleToggleAutoProvision = async (enabled: boolean) => {
    try {
      const result = await updateSharedAutoProvision(enabled)
      if (result.error) {
        console.error('Update failed:', result.error.message)
      } else {
        console.log('Auto-provision updated successfully')
      }
    } catch (error) {
      console.error('Unexpected error:', error)
    }
  }

  const loadDomainInfo = async () => {
    try {
      const domainInfo = await fetchTenantDomain()
      console.log('Domain:', domainInfo.domain)
      console.log('Auto-allow:', domainInfo.autoAllow)
    } catch (error) {
      console.error('Failed to load domain:', error)
    }
  }

  return (
    <div>
      <button onClick={loadDomainInfo} disabled={isLoading}>
        {isLoading ? 'Loading...' : 'Fetch Domain Info'}
      </button>
      <button 
        onClick={() => handleToggleAutoProvision(true)} 
        disabled={isUpdating}
      >
        {isUpdating ? 'Updating...' : 'Enable Auto-Provision'}
      </button>
    </div>
  )
}
```

The hook provides robust error handling for both network failures and API-specific error responses, with proper loading states for UI feedback.
