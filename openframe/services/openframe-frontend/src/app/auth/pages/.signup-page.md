A client-side React page component that handles organization signup with both traditional form submission and SSO provider integration, managing session storage and authentication state.

## Key Components

- **SignupPage** - Main page component with authentication flow logic
- **Session Storage Management** - Retrieves stored organization details from browser session
- **Authentication Integration** - Uses auth hooks for signup operations and state management
- **SSO Support** - Handles Google and Microsoft SSO signup flows
- **Route Protection** - Redirects authenticated users to dashboard

## Usage Example

```typescript
// This component is typically used as a Next.js page
// Located at /auth/signup route

// Session storage keys used:
// - 'auth:org_name' - Organization name
// - 'auth:domain' - Organization domain
// - 'auth:access_code' - Access code for signup
// - 'auth:email' - User email address

// Authentication flow:
const handleSignup = (formData) => {
  // Traditional signup
  registerOrganization(formData)
}

const handleSSOSignup = async (provider: 'google' | 'microsoft') => {
  // SSO signup with stored org details
  await registerOrganizationSSO({
    tenantName: storedOrgName,
    tenantDomain: storedDomain,
    email: storedEmail,
    provider,
    redirectTo: '/auth/login',
    accessCode: storedAccessCode
  })
}
```

The component automatically redirects authenticated users to the dashboard unless running in auth-only mode, and provides fallback navigation to the auth landing page.
