A Windows-specific utility module that locates PowerShell executables on the system by checking PATH environment and common installation directories.

## Key Components

- **`get_powershell_path()`** - Main function that returns a ready-to-use PowerShell executable path as a String
- **`find_powershell_path()`** - Internal helper that searches known PowerShell installation directories
- **Path Resolution Strategy** - Tries PATH first for performance, then falls back to filesystem search
- **Multi-Version Support** - Detects both Windows PowerShell (v1.0) and PowerShell 7 installations

## Usage Example

```rust
use crate::powershell::get_powershell_path;

fn execute_powershell_command() -> Result<(), Box<dyn std::error::Error>> {
    let powershell_path = get_powershell_path()?;
    
    let output = std::process::Command::new(&powershell_path)
        .arg("-Command")
        .arg("Get-Process")
        .output()?;
    
    if output.status.success() {
        println!("PowerShell output: {}", String::from_utf8_lossy(&output.stdout));
    }
    
    Ok(())
}

// Handle the case where PowerShell isn't found
match get_powershell_path() {
    Ok(path) => println!("Found PowerShell at: {}", path),
    Err(e) => eprintln!("Error: {}", e),
}
```

The module prioritizes performance by checking if `powershell.exe` is available in PATH before scanning filesystem locations, and supports both legacy Windows PowerShell and modern PowerShell 7 installations across different architecture directories (System32/SysWOW64).
