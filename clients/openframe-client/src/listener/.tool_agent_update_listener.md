Manages NATS JetStream consumer for receiving tool agent update messages with automatic reconnection, error handling, and machine-specific filtering.

## Key Components

- **`ToolAgentUpdateListener`** - Main listener struct that subscribes to tool update messages via NATS JetStream
- **`start()`** - Spawns background task with automatic reconnection loop
- **`listen()`** - Core message processing loop with acknowledgment handling
- **`create_consumer()`** - Sets up NATS consumer with retry logic and exponential backoff
- **Consumer configuration methods** - Builds machine-specific subjects and durable consumer names

## Usage Example

```rust
use tool_agent_update_listener::ToolAgentUpdateListener;

// Initialize the listener
let listener = ToolAgentUpdateListener::new(
    nats_connection_manager,
    tool_agent_update_service,
    config_service
);

// Start listening in background
let handle = listener.start().await?;

// The listener will:
// - Subscribe to "machine.all.tool.*.update" messages
// - Filter by machine ID
// - Process updates via ToolAgentUpdateService
// - Acknowledge successful messages
// - Leave failed messages for redelivery
// - Automatically reconnect on connection failures

// Optional: wait for completion (runs indefinitely)
handle.await?;
```

The listener creates a durable consumer named `machine_{machine_id}_tool_update_consumer` that delivers messages to `machine.{machine_id}.tool.update.inbox`, ensuring each machine only processes relevant tool updates with proper message acknowledgment and retry semantics.
