A unified layout component that provides consistent 50/50 split design for all OpenFrame authentication pages, featuring responsive behavior and proper vertical centering.

## Key Components

- **AuthLayout** - Main layout component that wraps auth pages
- **AuthLayoutProps** - TypeScript interface defining the component's props
- **AuthBenefitsSection** - Imported component that displays platform benefits

## Usage Example

```typescript
import { AuthLayout } from './auth-layout'
import { LoginForm } from './login-form'

export default function LoginPage() {
  return (
    <AuthLayout>
      <LoginForm />
    </AuthLayout>
  )
}
```

## Layout Structure

The component creates a responsive two-column layout:

- **Left Side (50%)**: Contains the authentication content with proper padding and vertical centering
- **Right Side (50%)**: Displays the benefits section showcasing platform features

On mobile devices, the layout stacks vertically with the auth content appearing first, followed by the benefits section. The design uses Tailwind CSS classes for responsive behavior, ensuring optimal display across all device sizes while maintaining the OpenFrame design system with `ods-bg` background styling.
