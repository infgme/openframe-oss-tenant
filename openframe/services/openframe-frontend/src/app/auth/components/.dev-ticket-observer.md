A client-side React component that monitors URL parameters for development authentication tickets and automatically processes them when detected.

## Key Components

- **DevTicketObserver**: Main functional component that observes URL changes
- **URL Parameter Monitoring**: Uses Next.js navigation hooks to watch for `devTicket` search parameters
- **Exchange Hook Integration**: Delegates ticket processing to `useDevTicketExchange` hook
- **Environment Toggle**: Controlled by `NEXT_PUBLIC_ENABLE_DEV_TICKET_OBSERVER` configuration
- **Duplicate Prevention**: Tracks last processed ticket to avoid redundant exchanges

## Usage Example

```typescript
// In your root layout or app component
import { DevTicketObserver } from './dev-ticket-observer'

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        {/* Observer monitors all route changes automatically */}
        <DevTicketObserver />
        {children}
      </body>
    </html>
  )
}

// When user visits: /dashboard?devTicket=abc123
// Component automatically detects and processes the ticket
```

The component renders nothing visually but provides essential background functionality for development authentication flows. It logs ticket detection events and delegates the actual authentication exchange to specialized hooks, maintaining separation of concerns in the authentication system.
