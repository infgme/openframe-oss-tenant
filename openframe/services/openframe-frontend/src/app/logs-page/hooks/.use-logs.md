A React hook for managing log data fetching, filtering, and pagination with GraphQL integration and state management for the OpenFrame platform.

## Key Components

- **`useLogs(activeFilters)`** - Primary hook for log data management with search, filtering, and pagination
- **`useLogFilters()`** - Secondary hook for fetching available filter options
- **`transformLogEntry()`** - Internal utility function that transforms backend log data to include device structure
- **State Management** - Integrates with Zustand store (`useLogsStore`) for global log state
- **Error Handling** - Comprehensive error handling with toast notifications

## Usage Example

```typescript
import { useLogs, useLogFilters } from './use-logs'

function LogsPage() {
  // Initialize logs hook with optional filters
  const {
    logs,
    edges,
    isLoading,
    error,
    hasNextPage,
    searchLogs,
    fetchNextPage,
    refreshLogs
  } = useLogs({
    severities: ['ERROR', 'WARNING'],
    toolTypes: ['SYSLOG'],
    deviceId: 'device-123'
  })

  // Initialize filters hook
  const {
    logFilters,
    isLoading: filtersLoading,
    fetchLogFilters
  } = useLogFilters()

  // Search logs
  const handleSearch = (query: string) => {
    searchLogs(query)
  }

  // Load more results
  const handleLoadMore = () => {
    if (hasNextPage && !isLoading) {
      fetchNextPage()
    }
  }

  return (
    <div>
      {isLoading ? <LogsSkeleton /> : <LogsList logs={edges} />}
    </div>
  )
}
```

The hook provides cursor-based pagination, real-time search, filter management, and automatic data transformation to ensure consistent device object structure across log entries.
