A Next.js page component that provides file management functionality for devices in the Flamingo MSP platform through MeshCentral integration.

## Key Components

- **FileManagerPage** - Main page component that handles device file management with loading states and error handling
- **FileManagerPageSkeleton** - Loading skeleton component with back navigation functionality
- **useDeviceDetails** - Custom hook for fetching and managing device information
- **FileManagerContainer** - Core file management interface component

## Usage Example

```typescript
// This is a Next.js route component accessed via:
// /devices/details/[deviceId]/file-manager

// The component automatically:
// 1. Extracts deviceId from URL params
// 2. Fetches device details using the custom hook
// 3. Validates MeshCentral agent availability
// 4. Renders the file manager interface

// URL example: /devices/details/abc123/file-manager
// Params: { deviceId: "abc123" }

// Error handling includes:
// - Loading states with skeleton UI
// - Missing MeshCentral agent ID validation
// - Network/API error display with navigation fallback
```

## Features

- **Async Params Handling** - Uses React 18's `use()` hook for promise-based route parameters
- **Device Validation** - Ensures MeshCentral agent is available before rendering file manager
- **Error Recovery** - Provides clear error messages and navigation back to device details
- **Loading States** - Shows skeleton UI during data fetching with consistent branding
- **Responsive Design** - Uses Flamingo design system components and styling tokens
