A client-side rendered page component that displays system logs in a tabular format within the Flamingo MSP platform's application layout.

## Key Components

- **Logs**: Default export React component that renders the main logs page
- **dynamic**: Export configuration forcing dynamic rendering for real-time log updates
- **AppLayout**: Wrapper component providing consistent navigation and layout structure
- **LogsTable**: Child component handling the display and management of log entries

## Usage Example

```typescript
// This page is automatically rendered when users navigate to /logs
// The component structure follows Next.js App Router conventions

import { AppLayout } from '../components/app-layout'
import { LogsTable } from './components/logs-table'

export default function Logs() {
  return (
    <AppLayout>
      <div className="space-y-6">
        <LogsTable/>
      </div>
    </AppLayout>
  )
}

// The dynamic export ensures fresh log data on each page visit
export const dynamic = 'force-dynamic'
```

This page component is designed for MSP technicians to monitor system events, audit trails, and troubleshooting information within the unified OpenFrame interface. The forced dynamic rendering ensures log data remains current for real-time monitoring scenarios.
