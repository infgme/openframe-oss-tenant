A REST API testing utility class that provides methods for interacting with device management endpoints in the OpenFrame platform using GraphQL and REST API calls.

## Key Components

- **getDeviceHostnames()** - Retrieves a list of device hostnames using GraphQL filtering
- **getDeviceIds()** - Fetches device machine IDs based on filter criteria  
- **getDevice()** - Gets a complete device object by machine ID
- **getDevices()** - Returns filtered list of devices using GraphQL queries
- **updateDeviceStatus()** - Updates device status via PATCH endpoint
- **archiveDevice()** / **deleteDevice()** - Convenience methods for status updates
- **searchDevice()** - Searches for devices with text-based filtering
- **getDeviceFilters()** - Retrieves available filter options for device queries

## Usage Example

```java
// Get filtered list of devices
DeviceFilterInput filter = new DeviceFilterInput();
List<Machine> devices = DeviceApi.getDevices(filter);

// Get specific device details
Machine device = DeviceApi.getDevice("machine-123");

// Archive a device
DeviceApi.archiveDevice(device);

// Search for devices
Machine foundDevice = DeviceApi.searchDevice(filter, "hostname-search");

// Get available filter options
DeviceFilters filters = DeviceApi.getDeviceFilters();
```

This API wrapper abstracts GraphQL and REST interactions for device management operations, providing type-safe methods for common device-related tasks in automated testing scenarios.
