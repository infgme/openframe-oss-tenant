A utility class that provides cross-platform file path operations and generates standardized file operation requests for MeshCentral's file management system.

## Key Components

- **FileOperations**: Main class that handles path manipulation and request generation
- **generateRequestId()**: Creates unique request identifiers
- **Path utilities**: `joinPath()`, `parsePath()`, `getParentPath()` with Windows/Unix compatibility  
- **Request factories**: Methods to create structured requests for file operations (list, create, rename, delete, copy, move, zip, etc.)
- **Path validation**: `sanitizeName()`, `validatePath()` for security and compatibility

## Usage Example

```typescript
import { FileOperations } from './file-operations'

const fileOps = new FileOperations()

// Generate file operation requests
const listRequest = fileOps.createListDirectoryRequest('/home/user')
const createDirRequest = fileOps.createMakeDirRequest('/home/user', 'new-folder')
const deleteRequest = fileOps.createDeleteRequest('/home/user', ['file1.txt', 'file2.txt'], true)

// Cross-platform path operations
const joined = fileOps.joinPath('/home/user', 'documents/file.txt')
const parent = fileOps.getParentPath('/home/user/documents/file.txt') // '/home/user/documents'

// Path validation and sanitization
const clean = fileOps.sanitizeName('unsafe<file>name.txt') // 'unsafefilename.txt'
const isValid = fileOps.validatePath('/safe/path') // true
```

The class automatically detects and handles both Windows (`\`) and Unix (`/`) path separators, making it suitable for cross-platform file management operations.
