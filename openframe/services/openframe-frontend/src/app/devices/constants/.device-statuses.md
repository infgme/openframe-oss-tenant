Defines device status constants and filtering utilities for the Flamingo MSP platform's device management system.

## Key Components

- **`DEVICE_STATUS`** - Enumeration of all possible device states (ONLINE, OFFLINE, ACTIVE, INACTIVE, MAINTENANCE, DECOMMISSIONED, PENDING, ARCHIVED, DELETED)
- **`DEFAULT_VISIBLE_STATUSES`** - Array of statuses shown in standard device queries (excludes ARCHIVED and DELETED)
- **`HIDDEN_DEVICE_STATUSES`** - Array of statuses hidden by default for cleanup/archival purposes
- **Type definitions** - `DeviceStatus`, `DefaultVisibleStatus`, and `HiddenDeviceStatus` for type safety

## Usage Example

```typescript
import { DEVICE_STATUS, DEFAULT_VISIBLE_STATUSES, type DeviceStatus } from './device-statuses'

// Filter devices by visible statuses
const visibleDevices = devices.filter(device => 
  DEFAULT_VISIBLE_STATUSES.includes(device.status as DefaultVisibleStatus)
)

// Check specific device status
if (device.status === DEVICE_STATUS.MAINTENANCE) {
  showMaintenanceAlert(device)
}

// Type-safe status assignment
const newDevice: { status: DeviceStatus } = {
  status: DEVICE_STATUS.PENDING
}

// Query devices excluding archived/deleted
const activeDeviceQuery = {
  status: { $in: DEFAULT_VISIBLE_STATUSES }
}
```

This centralized approach ensures consistent device filtering across the OpenFrame platform while maintaining type safety and easy extensibility for new device states.
