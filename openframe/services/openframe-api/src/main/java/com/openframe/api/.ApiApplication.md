Main entry point for the OpenFrame API Spring Boot application, configuring component scanning across multiple OpenFrame modules and initializing the application context.

## Key Components

- **@SpringBootApplication** - Enables auto-configuration, component scanning, and configuration
- **@ComponentScan** - Scans multiple OpenFrame packages for Spring components
- **@Slf4j** - Provides logging capabilities via Lombok
- **main()** - Application entry point with startup logging

## Package Scanning Configuration

The application scans the following package hierarchies:
- `com.openframe.api` - API layer components (controllers, services)
- `com.openframe.data` - Data access layer and repositories
- `com.openframe.core` - Core business logic and utilities
- `com.openframe.notification` - Notification services and handlers
- `com.openframe.kafka` - Kafka messaging components

## Usage Example

```java
// Run the application
java -jar openframe-api.jar

// Or with Spring profiles
java -jar openframe-api.jar --spring.profiles.active=production

// The application will scan all configured packages and start the embedded server
// Log output: "Starting OpenFrame API"
```

## Architecture Notes

This follows Spring Boot's modular architecture pattern, allowing the API application to integrate components from multiple OpenFrame modules. The component scanning ensures all necessary beans are discovered and registered in the application context across the entire OpenFrame ecosystem.
