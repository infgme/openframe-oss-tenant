A background monitoring service that continuously checks and fixes file system permissions for OpenFrame directories, with metrics tracking and automatic remediation capabilities.

## Key Components

- **PermissionMonitor** - Main struct that orchestrates periodic permission checks using a configurable interval (default: 1 hour)
- **start_monitoring()** - Async method that runs the monitoring loop indefinitely, performing checks at regular intervals
- **perform_check()** - Core logic that validates permissions via DirectoryManager health checks and attempts automatic fixes on failures
- **get_metrics()** - Returns metric keys for monitoring permission errors, fixes, and last check timestamps
- **with_interval()** - Builder method to customize the check interval duration

## Usage Example

```rust
use std::sync::Arc;
use std::time::Duration;
use tokio;

// Create permission monitor with custom 30-minute intervals
let directory_manager = Arc::new(DirectoryManager::new());
let monitor = PermissionMonitor::new(directory_manager)
    .with_interval(Duration::from_secs(1800));

// Start background monitoring (runs indefinitely)
tokio::spawn(async move {
    monitor.start_monitoring().await;
});

// Check available metrics
let metrics = monitor.get_metrics();
for (name, value) in metrics {
    println!("Metric: {} = {}", name, value);
}
```

The service automatically records metrics for permission errors, successful fixes, and check timestamps, integrating with OpenFrame's metrics system for observability and alerting.
