Represents message data for installed agent notifications in the Flamingo platform, containing agent type and version information for tracking deployed agents across the MSP infrastructure.

## Key Components

**`InstalledAgentMessage`** - Main struct that holds installed agent metadata:
- `agent_type` - String identifier for the type of agent (e.g., "monitoring", "backup", "security")
- `version` - Version string of the installed agent for compatibility tracking

The struct uses `camelCase` serialization for consistent API communication and implements standard traits for debugging, cloning, and JSON serialization/deserialization.

## Usage Example

```rust
use serde_json;

// Creating an installed agent message
let message = InstalledAgentMessage {
    agent_type: "monitoring_agent".to_string(),
    version: "1.2.3".to_string(),
};

// Serializing to JSON for API communication
let json = serde_json::to_string(&message)?;
println!("{}", json); // {"agentType":"monitoring_agent","version":"1.2.3"}

// Deserializing from JSON response
let json_data = r#"{"agentType":"backup_agent","version":"2.1.0"}"#;
let parsed: InstalledAgentMessage = serde_json::from_str(json_data)?;
println!("Agent: {} v{}", parsed.agent_type, parsed.version);
```

This structure is typically used in OpenFrame's agent management system to track which agents are deployed on client endpoints and their current versions for update management and compatibility checks.
