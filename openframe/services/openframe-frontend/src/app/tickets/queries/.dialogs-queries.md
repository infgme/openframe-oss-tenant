GraphQL query definitions for the Flamingo dialog system, providing queries to fetch dialogs, individual dialog details, statistics, and message threads with support for pagination and filtering.

## Key Components

- **GET_DIALOGS_QUERY** - Fetches paginated list of dialogs with filtering and search capabilities
- **GET_DIALOG_QUERY** - Retrieves individual dialog details by ID
- **GET_DIALOG_STATISTICS_QUERY** - Gets aggregate dialog metrics and statistics
- **GET_DIALOG_MESSAGES_QUERY** - Fetches paginated messages for a specific dialog with various message types

## Usage Example

```typescript
import { GET_DIALOGS_QUERY, GET_DIALOG_MESSAGES_QUERY } from './dialogs-queries';

// Fetch dialogs with pagination
const dialogsResult = await client.query({
  query: GET_DIALOGS_QUERY,
  variables: {
    pagination: { cursor: null, limit: 20 },
    filter: { status: "OPEN" },
    search: "network issue"
  }
});

// Get messages for a specific dialog
const messagesResult = await client.query({
  query: GET_DIALOG_MESSAGES_QUERY,
  variables: {
    dialogId: "dialog_123",
    cursor: null,
    limit: 50
  }
});
```

The queries support complex message types including text, tool execution data, approval requests, and error messages, making them suitable for AI-powered support conversations with integrated tooling and approval workflows.
