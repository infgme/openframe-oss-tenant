A Next.js home page component that automatically redirects users based on authentication status and application mode configuration.

## Key Components

- **HomePage**: Default export component that handles initial routing logic
- **useAuthStore**: Custom hook for accessing authentication state
- **getDefaultRedirectPath**: Utility function that determines redirect destination based on auth status
- **isAuthOnlyMode**: Utility function to check if the app is in authentication-only mode
- **AppShellSkeleton**: Loading component displayed during redirect

## Usage Example

```typescript
// The component automatically redirects on mount:
// - If auth-only mode: redirects to /auth
// - Otherwise: uses getDefaultRedirectPath() based on authentication status

// Example redirect behavior:
// Authenticated user → /dashboard (or configured default)
// Unauthenticated user → /login (or public landing page)
// Auth-only mode → /auth (regardless of status)

export default function HomePage() {
  // Handles automatic redirection based on:
  // 1. Application mode (auth-only vs normal)
  // 2. User authentication status
  // 3. Configured default paths
}
```

The component serves as a smart router entry point, ensuring users land on the appropriate page based on their authentication state and the application's current configuration mode.
