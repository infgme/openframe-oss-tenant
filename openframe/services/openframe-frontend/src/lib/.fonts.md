This file configures Google Fonts for Next.js applications, setting up Azeret Mono and DM Sans fonts with optimized loading and CSS variable support.

## Key Components

- **`azeretMono`** - Azeret Mono font configuration for monospace text (code, terminals)
- **`dmSans`** - DM Sans font configuration for general UI text and content

## Font Configuration

Both fonts are configured with:
- **Latin subset** - Optimized character set for Western languages
- **Display swap** - Fallback font shows immediately while custom font loads
- **CSS variables** - Enable usage via `--font-azeret-mono` and `--font-dm-sans`
- **Multiple weights** - Support for regular (400), medium (500), semi-bold (600), and bold (700)

## Usage Example

```typescript
// In your Next.js app layout or pages
import { azeretMono, dmSans } from './fonts'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={`${dmSans.variable} ${azeretMono.variable}`}>
      <body className="font-dm-sans">
        {/* Use DM Sans as default */}
        <h1>Main Content</h1>
        {/* Use Azeret Mono for code */}
        <code className="font-azeret-mono">console.log('Hello')</code>
        {children}
      </body>
    </html>
  )
}
```

Configure your Tailwind CSS to use the font variables:

```typescript
// tailwind.config.js
module.exports = {
  theme: {
    fontFamily: {
      'dm-sans': ['var(--font-dm-sans)'],
      'azeret-mono': ['var(--font-azeret-mono)'],
    }
  }
}
```
