Configuration utility class that manages Redis connection URI resolution with fallback priority for OpenFrame's data layer connectivity.

## Key Components

- **`getRedisUri()`** - Static method that returns the Redis connection URI using a priority-based resolution system
- **Configuration Priority** - Resolves URI from: 1) JVM system property (`redis.uri`), 2) Environment variable (`REDIS_URI`), 3) Default cluster URI
- **Lazy Loading** - URI is resolved once and cached for subsequent calls
- **Default Configuration** - Points to Kubernetes cluster service at `redis-master.datasources.svc.cluster.local:6379`

## Usage Example

```java
// Get Redis URI - automatically resolves based on configuration priority
String connectionUri = RedisConfig.getRedisUri();

// Example configurations (in order of priority):
// 1. JVM property: -Dredis.uri=redis://localhost:6379
// 2. Environment: export REDIS_URI=redis://prod-redis:6379  
// 3. Default: redis://redis-master.datasources.svc.cluster.local:6379

// Use in Redis client setup
Jedis jedis = new Jedis(URI.create(RedisConfig.getRedisUri()));
```

This configuration pattern supports flexible deployment across development (localhost), staging, and production (Kubernetes cluster) environments without code changes.
