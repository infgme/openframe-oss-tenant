A Next.js page component that handles both creating new organizations and editing existing ones by routing through a single wrapper component.

## Key Components

- **EditOrganizationPageWrapper** - Main page component that determines whether to render in create or edit mode
- **AppLayout** - Provides consistent layout structure for the application
- **NewOrganizationPage** - Core organization form component that handles both create and edit operations
- **useParams** - Next.js hook for extracting route parameters

## Usage Example

```typescript
// Route structure this component supports:
// /organizations/new - Creates new organization (organizationId = null)
// /organizations/[id] - Edits existing organization (organizationId = id)

// The component automatically determines mode based on the URL parameter
const EditOrganizationPageWrapper = () => {
  const params = useParams<{ id?: string }>()
  const rawId = params?.id
  
  // Special handling for 'new' route vs actual organization IDs
  const id = rawId === 'new' ? null : (typeof rawId === 'string' ? rawId : null)
  
  return (
    <AppLayout>
      <NewOrganizationPage organizationId={typeof id === 'string' ? id : null} />
    </AppLayout>
  )
}
```

This pattern allows a single page component to handle both create and edit workflows by passing either a valid organization ID or null to the underlying form component, with the 'new' route specifically triggering creation mode.
