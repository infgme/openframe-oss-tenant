Client-side wrapper that re-exports UI components from the OpenFrame frontend core, ensuring proper client/server component boundaries in Next.js applications.

## Key Components

- **Client Boundary Wrapper**: Prevents Server Components from directly importing UI components that use React hooks
- **Barrel Export**: Re-exports all components from `@flamingo-stack/openframe-frontend-core/components/ui`
- **Next.js Compatibility**: Ensures UI components are only used in client-side contexts

## Usage Example

```typescript
// In a Client Component
'use client'
import { Button, Dialog, Input } from './openframe-core-ui'

export function MyClientComponent() {
  return (
    <div>
      <Button onClick={() => console.log('clicked')}>
        Click me
      </Button>
      <Input placeholder="Enter text..." />
    </div>
  )
}
```

```typescript
// In a Server Component - import individual components instead
import { Button } from '@flamingo-stack/openframe-frontend-core/components/ui/button'
// This would cause hydration issues if Button uses hooks

// Better approach - use this wrapper in Client Components only
import { SomeServerSafeComponent } from './server-components'
```

This wrapper is essential for maintaining the Next.js App Router's server/client component architecture while providing easy access to the OpenFrame UI component library.
