A message structure for communicating client update information, including version details and download configurations for the OpenFrame client.

## Key Components

- **`OpenFrameClientUpdateMessage`** - Main struct containing update metadata
  - `version` - String representing the client version to update to
  - `download_configurations` - Vector of platform-specific download options

## Usage Example

```rust
use openframe_client_update_message::OpenFrameClientUpdateMessage;
use download_configuration::DownloadConfiguration;

// Create an update message
let update_message = OpenFrameClientUpdateMessage {
    version: "1.2.3".to_string(),
    download_configurations: vec![
        DownloadConfiguration {
            platform: "windows".to_string(),
            url: "https://releases.openframe.ai/v1.2.3/windows.exe".to_string(),
            checksum: "abc123...".to_string(),
        },
        DownloadConfiguration {
            platform: "macos".to_string(),
            url: "https://releases.openframe.ai/v1.2.3/macos.dmg".to_string(),
            checksum: "def456...".to_string(),
        },
    ],
};

// Serialize for transmission
let json = serde_json::to_string(&update_message)?;

// Send to clients or deserialize received updates
let received: OpenFrameClientUpdateMessage = serde_json::from_str(&json)?;
```

This structure enables the OpenFrame platform to notify clients about available updates and provide platform-specific download instructions with proper serialization support for network communication.
