Django CORS and authentication configuration for local development environment with localhost frontend integration.

## Key Components

- **CORS Configuration**: Complete Cross-Origin Resource Sharing setup allowing localhost:8080 frontend
- **Session/CSRF Settings**: Cookie domain and security configurations for development
- **Authentication Settings**: Django-allauth configuration with simplified email verification
- **Security Overrides**: Development-friendly settings including disabled 2FA

## Configuration Details

### CORS Settings
- Allows credentials and specific HTTP methods (GET, POST, PUT, DELETE, PATCH, OPTIONS)
- Permits localhost:8080 origin for frontend development
- Includes standard headers plus authentication tokens

### Security Configuration
- Disables secure cookie flags for HTTP development
- Sets SameSite policy to 'Lax' for cross-origin requests
- Configures CSRF trusted origins matching CORS allowlist

### Authentication Features
- Bypasses email verification for faster development
- Uses username-based authentication
- Disables two-factor authentication (TRMM_DISABLE_2FA)
