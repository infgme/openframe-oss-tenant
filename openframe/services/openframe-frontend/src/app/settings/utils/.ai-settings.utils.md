Utility functions for organizing and managing AI policy settings in a hierarchical structure with categorization and grouping capabilities.

## Key Components

- **`slugify()`** - Converts strings to URL-friendly slugs by removing special characters
- **`buildPolicyGroups()`** - Main function that transforms flat policy rules into a nested Map structure grouped by policy group and category
- **`clonePolicyGroups()`** - Creates a deep clone of policy group structures for immutable updates
- **`mapToObject()`** - Converts Map entries to plain JavaScript objects

## Usage Example

```typescript
import { buildPolicyGroups, clonePolicyGroups } from './ai-settings.utils'

// Transform policy rules into grouped categories
const policyRules: PolicyRule[] = [
  {
    naturalKey: 'file-read',
    operation: 'Read Files',
    policyGroup: 'File Operations',
    category: 'Basic Access',
    tool: 'filesystem',
    approvalLevel: 'auto',
    commandPattern: 'read:*'
  }
]

const groupedPolicies = buildPolicyGroups(policyRules)
// Returns: Map<'File Operations', [{ id: 'file-operations:basic-access', name: 'Basic Access', policies: [...] }]>

// Clone for state updates
const clonedGroups = clonePolicyGroups(groupedPolicies)

// Convert to object for serialization
const serializable = Array.from(groupedPolicies.entries())
  .reduce((acc, [key, value]) => ({ ...acc, [key]: value }), {})
```

The utilities enable efficient organization of AI policy rules into a structured hierarchy that supports UI components like expandable permission categories and approval level management.
