A data transfer object (DTO) that represents a log event within the OpenFrame platform, capturing essential information about security, system, or application events from various MSP tools.

## Key Components

- **Event Identification**: `toolEventId`, `eventType`, and `toolType` for categorizing and tracking events
- **Temporal Data**: `ingestDay` and `timestamp` for time-based analysis and retention
- **Severity Classification**: `severity` field for event prioritization and alerting
- **Entity Associations**: `userId`, `deviceId`, `hostname` for linking events to specific assets
- **Organization Context**: `organizationId` and `organizationName` for multi-tenant support
- **Event Details**: `summary` field containing human-readable event description

## Usage Example

```java
// Creating a security alert log event
LogEvent securityAlert = LogEvent.builder()
    .toolEventId("SEC-2024-001")
    .eventType("SECURITY_ALERT")
    .toolType("FIREWALL")
    .severity("HIGH")
    .userId("user123")
    .deviceId("fw-001")
    .hostname("firewall.company.com")
    .organizationId("org456")
    .organizationName("Acme Corp")
    .summary("Blocked suspicious outbound connection to known malicious IP")
    .timestamp("2024-01-15T10:30:00Z")
    .ingestDay("2024-01-15")
    .build();

// The LogEvent can be used in logging pipelines, SIEM integration,
// or stored for compliance and audit purposes
```

This DTO is typically used in OpenFrame's unified logging system to standardize event data from different MSP tools, enabling consistent monitoring, alerting, and compliance reporting across the platform.
