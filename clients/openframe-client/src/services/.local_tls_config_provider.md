A TLS configuration provider that creates secure NATS client configurations using local development certificates, typically from mkcert for development environments.

## Key Components

- **LocalTlsConfigProvider** - Main service that creates TLS configurations for NATS clients
- **create_tls_config()** - Builds a rustls ClientConfig with custom root certificates
- **get_certificate_path()** - Resolves the local CA certificate path from initial configuration
- **initial_configuration_service** - Dependency for retrieving certificate paths

## Usage Example

```rust
use crate::local_tls_config_provider::LocalTlsConfigProvider;
use crate::services::InitialConfigurationService;

// Create the provider
let config_service = InitialConfigurationService::new();
let tls_provider = LocalTlsConfigProvider::new(config_service);

// Generate TLS configuration for NATS client
let tls_config = tls_provider.create_tls_config()?;

// Use with NATS client
let client = async_nats::ConnectOptions::new()
    .tls_client_config(tls_config)
    .connect("tls://localhost:4222")
    .await?;
```

The provider reads PEM-formatted certificates from the file system, parses them using `rustls_pemfile`, and creates a root certificate store for validating TLS connections. This is particularly useful for development setups where services use locally-generated certificates from tools like mkcert rather than traditional certificate authorities.
