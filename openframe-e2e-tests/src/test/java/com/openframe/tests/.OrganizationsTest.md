Comprehensive integration test suite for Organization CRUD operations in the OpenFrame MSP platform, validating API endpoints against database state through ordered test execution.

## Key Components

- **@Tag("authorized")** - Marks as authorized test requiring authentication
- **@TestMethodOrder(MethodOrderer.OrderAnnotation.class)** - Ensures sequential test execution
- **AuthorizedTest** - Base class providing authentication context
- **Test Methods**:
  - `testCreateOrganization()` - Validates organization creation with request-response mapping
  - `testRetrieveOrganization()` - Tests single organization retrieval and data consistency
  - `testUpdateOrganization()` - Verifies organization updates preserve metadata
  - `testDeleteOrganization()` - Confirms soft deletion functionality
  - `testRetrieveAllOrganizations()` - Validates active organizations filtering

## Usage Example

```java
// Run the complete test suite
@Test
void runOrganizationTests() {
    // Tests execute in order (1-5)
    // 1. Creates new organization with validation
    // 2. Retrieves and compares with DB
    // 3. Updates existing organization
    // 4. Soft deletes organization
    // 5. Verifies active organizations list excludes deleted
}

// Individual test validation pattern
Organization org = createOrganization(request);
assertThat(org.getId()).isNotNull();
assertThat(org.getDeleted()).isFalse();
assertThat(org).usingRecursiveComparison()
    .ignoringFields("id", "createdAt", "updatedAt")
    .isEqualTo(request);
```

The test suite follows OpenFrame's integration testing patterns, using static imports for API calls and database operations while maintaining proper test isolation and validation coverage.
