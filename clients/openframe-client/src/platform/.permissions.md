A cross-platform file permissions and privilege management utility that handles Unix file permissions, Windows read-only flags, and administrative privilege elevation across different operating systems.

## Key Components

- **`Permissions`** - Core structure for managing file/directory permissions with Unix mode support
- **`PermissionError`** - Comprehensive error type for permission-related failures
- **`PermissionUtils`** - Static utility methods for checking and obtaining admin/root privileges
- **Platform-specific constants** - `ROOT_UID`, `ADMIN_GID` for Unix systems
- **Global state** - `ADMIN_PRIVILEGES_GRANTED` atomic flag to track elevation status

## Usage Example

```rust
use std::path::Path;

// Create and apply standard directory permissions
let dir_perms = Permissions::directory(); // 755
dir_perms.apply(Path::new("/opt/myapp"))?;

// Create and apply file permissions  
let file_perms = Permissions::file(); // 644
file_perms.apply(Path::new("/opt/myapp/config.txt"))?;

// Check if running with admin privileges
if !PermissionUtils::is_admin() {
    // Request elevation with platform-appropriate UI
    PermissionUtils::ensure_admin()?;
}

// Verify permissions match expected values
let current_perms = Permissions::from_path(Path::new("/opt/myapp"))?;
assert!(current_perms.verify(Path::new("/opt/myapp"))?);
```

The module provides cross-platform abstractions for file permissions and handles privilege elevation through UAC on Windows, AppleScript dialogs on macOS, and pkexec/sudo on Linux.
