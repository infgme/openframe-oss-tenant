Configuration utility that sets up REST Assured with standardized settings for HTTP client timeouts, SSL handling, and logging behavior in OpenFrame API tests.

## Key Components

- **`configure()`** - Static method that initializes REST Assured with base URI and client configuration
- **`DEFAULT_TIMEOUT`** - 60-second timeout constant for connection and socket operations
- **Base URI Setup** - Retrieves base URL from `ApiConfig.getBaseUrl()`
- **SSL Configuration** - Enables relaxed HTTPS validation for testing environments
- **Logging Configuration** - Automatically logs requests/responses when validation fails
- **HTTP Client Configuration** - Sets connection and socket timeout parameters

## Usage Example

```java
// Initialize REST Assured configuration before running tests
@BeforeAll
public static void setup() {
    RestAssuredConfig.configure();
}

// After configuration, REST Assured is ready to use
@Test
public void testApiEndpoint() {
    given()
        .when()
        .get("/api/users")
        .then()
        .statusCode(200);
}

// Configuration automatically handles:
// - Base URI prepending
// - 60-second timeouts
// - SSL certificate validation
// - Request/response logging on failures
```

The configuration ensures consistent behavior across all API tests by centralizing timeout settings, enabling development-friendly SSL handling, and providing automatic diagnostic logging when tests fail.
