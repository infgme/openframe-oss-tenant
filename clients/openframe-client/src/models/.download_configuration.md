Configuration structure for handling OS-specific file downloads, providing platform detection and cross-platform download management capabilities.

## Key Components

- **`DownloadConfiguration`** - Main struct containing OS, file names, and download link information
- **`matches_current_os()`** - Method to verify if configuration matches the current operating system
- **Platform Detection** - Built-in support for Windows, macOS, and Linux detection using Rust's `cfg!` macros

## Usage Example

```rust
use serde_json;

// Create a download configuration
let config = DownloadConfiguration {
    os: "windows".to_string(),
    file_name: "app-installer.exe".to_string(),
    agent_file_name: "agent.exe".to_string(),
    link: "https://releases.example.com/windows/app-installer.exe".to_string(),
};

// Check if configuration matches current OS
if config.matches_current_os() {
    println!("This configuration is compatible with the current OS");
    println!("Download from: {}", config.link);
}

// Serialize to JSON with camelCase fields
let json = serde_json::to_string(&config)?;
// Results in: {"os":"windows","fileName":"app-installer.exe","agentFileName":"agent.exe","link":"..."}

// Deserialize from JSON
let config: DownloadConfiguration = serde_json::from_str(&json)?;
```

The struct uses camelCase serialization for JSON compatibility and provides a convenient method to determine platform compatibility without external dependencies.
