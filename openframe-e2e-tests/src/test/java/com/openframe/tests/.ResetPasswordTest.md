Integration test for the password reset functionality in the OpenFrame platform, validating the complete reset flow from token generation to successful authentication.

## Key Components

- **ResetPasswordTest**: Main test class extending `UnauthorizedTest` for password reset scenarios
- **testResetPassword()**: Core test method that validates the complete password reset workflow
- **External Dependencies**: Uses `UserApi`, `AuthHelper`, `Redis`, and data generators for test execution

## Usage Example

```java
// The test validates this password reset flow:

// 1. Get test user and initiate reset
User user = getUser();
resetPassword(user);

// 2. Retrieve reset token from Redis
String token = getResetToken(user.getEmail());
assertThat(token).isNotNull();

// 3. Confirm reset with new password
ResetConfirmRequest confirmRequest = resetConfirmRequest(token);
confirmReset(confirmRequest);

// 4. Update user and verify authentication
user.setPassword(confirmRequest.getNewPassword());
saveUser(user);
MeResponse me = me();
assertThat(me.isAuthenticated()).isTrue();
```

This test ensures the password reset system works end-to-end: generating reset tokens, storing them in Redis, processing reset confirmations, and maintaining user authentication state throughout the process.
