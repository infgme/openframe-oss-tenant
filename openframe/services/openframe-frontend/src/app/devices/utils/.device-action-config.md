Configuration utility that provides a unified source of truth for device action buttons (remote control, shell, file management) used across dropdown menus and detail pages in the OpenFrame device management interface.

## Key Components

- **`DeviceActionButtonConfig`** - Interface defining structure for action button configurations including icons, labels, and href paths
- **`DeviceActionButtons`** - Main interface containing all device action configurations and availability status
- **`getDeviceActionButtons()`** - Core function that generates action button configurations based on device type (Windows vs Linux)
- **`getWindowsShellSubmenu()`** - Returns shell options (CMD, PowerShell) for Windows devices
- **`toActionsMenuItem()`** - Adapter function to convert configurations into dropdown menu item format

## Usage Example

```typescript
import { getDeviceActionButtons } from './device-action-config'
import type { Device } from '../types/device.types'

// Get action button configurations for a device
const device: Device = { /* device data */ }
const deviceId = "device-123"
const isWindows = device.os === 'windows'

const actionButtons = getDeviceActionButtons(device, deviceId, isWindows)

// Access individual button configurations
const remoteControlButton = actionButtons.remoteControl
const shellButton = actionButtons.remoteShell

// Check availability
if (actionButtons.availability.remoteControlEnabled) {
  // Render remote control button
  console.log(`Navigate to: ${remoteControlButton.href}`)
}

// Handle Windows shell submenu
if (isWindows && shellButton.type === 'submenu') {
  shellButton.submenu?.forEach(shellOption => {
    console.log(`${shellOption.label}: ${getShellHref(deviceId, shellOption.id)}`)
  })
}
```
