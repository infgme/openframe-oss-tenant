Cross-platform updater module that provides a unified interface for launching OS-specific update processes. This module handles the configuration and platform-specific delegation for binary updates across Windows, macOS, and Linux systems.

## Key Components

- **`UpdaterParams`** - Configuration struct containing paths and metadata needed for the update process:
  - `binary_path`: Path to the updater binary
  - `target_exe`: Path to the executable being updated
  - `service_name`: Name of the service to manage during update
  - `update_state_path`: Path for storing update state information

- **`launch_updater`** - Platform-specific function imported from OS-specific modules (windows, macos, linux) that handles the actual update process

## Usage Example

```rust
use std::path::PathBuf;

// Configure update parameters
let params = UpdaterParams {
    binary_path: PathBuf::from("/path/to/updater"),
    target_exe: PathBuf::from("/path/to/target.exe"),
    service_name: "MyService".to_string(),
    update_state_path: "/tmp/update_state.json".to_string(),
};

// Launch the platform-appropriate updater
launch_updater(params)?;
```

The module automatically selects the correct implementation based on the target OS using conditional compilation, ensuring consistent behavior across all supported platforms while allowing for OS-specific optimizations.
