A reusable button component that navigates users to device details pages using either device or machine IDs.

## Key Components

- **DeviceDetailsButton**: Main component that renders a navigation button to device details
- **DeviceDetailsButtonProps**: TypeScript interface defining component props including ID options, styling, and behavior
- **Button**: Core UI component from the Flamingo OpenFrame frontend library
- **useRouter**: Next.js navigation hook for client-side routing

## Usage Example

```typescript
import { DeviceDetailsButton } from './device-details-button'

// Basic usage with device ID
<DeviceDetailsButton 
  deviceId="dev-123" 
  label="View Device" 
/>

// Using machine ID with custom styling
<DeviceDetailsButton 
  machineId="machine-456"
  variant="primary"
  className="ml-4"
  openInNewTab={true}
/>

// Minimal usage (falls back to "Details" label and outline variant)
<DeviceDetailsButton deviceId="dev-789" />
```

The component prioritizes `machineId` over `deviceId` when both are provided, and returns null if neither ID is available. It supports all standard button variants and can optionally open the details page in a new tab.
