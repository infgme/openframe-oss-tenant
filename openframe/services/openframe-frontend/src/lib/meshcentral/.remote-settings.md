Configuration interface and settings management for MeshCentral remote desktop connections, handling quality, scaling, frame rate, and input options.

## Key Components

- **RemoteSettingsConfig** - Interface defining remote desktop configuration options
- **QUALITY_OPTIONS** - Predefined JPEG quality levels (10-100%)
- **SCALING_OPTIONS** - Screen scaling values mapped to MeshCentral format (25-200%)
- **FRAME_RATE_OPTIONS** - Frame rate presets with millisecond timing
- **RemoteDesktopSettings** - Class for managing and applying settings via WebSocket

## Usage Example

```typescript
import { RemoteDesktopSettings, DEFAULT_SETTINGS } from './remote-settings'

// Create settings manager with defaults
const remoteSettings = new RemoteDesktopSettings(DEFAULT_SETTINGS)

// Configure custom settings
const customSettings = {
  quality: 80,        // High quality (80%)
  scaling: 512,       // 50% scaling
  frameRate: 'fast',  // 20 FPS
  swapMouseButtons: false,
  useRemoteKeyboardMap: true
}

// Apply settings via WebSocket
remoteSettings.setWebSocket(websocket)
remoteSettings.applySettings(customSettings)

// Estimate bandwidth usage
const estimatedKBps = remoteSettings.estimateBandwidth()
console.log(`Estimated bandwidth: ${estimatedKBps} KB/s`)
```

The class handles binary protocol communication with MeshCentral, encoding settings into the expected format with proper endianness for compression commands.
