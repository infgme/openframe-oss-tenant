A simple Next.js layout component that provides metadata configuration for the OpenFrame dashboard pages and renders child components without additional wrapper elements.

## Key Components

- **metadata**: Static metadata object defining page title and description for SEO
- **DashboardLayout**: React component that passes through child components unchanged

## Usage Example

```typescript
// This layout is automatically applied to all pages in the dashboard route group
// Pages in the same directory will inherit the metadata and be wrapped by this layout

// Example page that would use this layout:
export default function DashboardPage() {
  return (
    <div>
      <h1>Dashboard Content</h1>
      {/* This content will be passed as children to DashboardLayout */}
    </div>
  )
}

// The rendered output would be:
// <div>
//   <h1>Dashboard Content</h1>
// </div>
```

The layout serves as a minimal container that primarily exists to set consistent metadata across all dashboard pages while allowing maximum flexibility for page content structure.
