A DTO class that represents the response payload returned after successful user registration in the OpenFrame platform, containing user account details and metadata.

## Key Components

- **User Identity Fields**: `id`, `name`, `domain` - Core user identification information
- **Ownership & Organization**: `ownerId` - Links user to their organization owner
- **Account Status**: `status`, `plan`, `active` - User account state and subscription details
- **Integration Fields**: `hubspotId` - External CRM system integration identifier
- **Audit Fields**: `createdAt`, `updatedAt` - Timestamp tracking for user lifecycle
- **Lombok Annotations**: `@Data`, `@Builder`, `@NoArgsConstructor`, `@AllArgsConstructor` - Auto-generates getters, setters, constructors, and builder pattern

## Usage Example

```java
// Building a registration response after user creation
UserRegistrationResponse response = UserRegistrationResponse.builder()
    .id("usr_12345")
    .name("Acme Corp")
    .domain("acme.com")
    .ownerId("owner_67890")
    .status("ACTIVE")
    .plan("PROFESSIONAL")
    .hubspotId("hs_contact_123")
    .createdAt("2024-01-15T10:30:00Z")
    .updatedAt("2024-01-15T10:30:00Z")
    .active(true)
    .build();

// Returning from registration endpoint
@PostMapping("/register")
public ResponseEntity<UserRegistrationResponse> registerUser(@RequestBody UserRegistrationRequest request) {
    User user = userService.createUser(request);
    UserRegistrationResponse response = mapToResponse(user);
    return ResponseEntity.ok(response);
}
```

This response DTO ensures consistent data structure when communicating user registration results to client applications, supporting both internal OpenFrame operations and external integrations like HubSpot CRM.
