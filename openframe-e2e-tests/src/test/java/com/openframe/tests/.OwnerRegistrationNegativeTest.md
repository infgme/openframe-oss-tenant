Negative test suite for OpenFrame owner user registration functionality, validating error handling for various failure scenarios like closed registration and duplicate email attempts.

## Key Components

- **Test Class**: `OwnerRegistrationNegativeTest` - Extends `UnauthorizedTest` for testing registration failures
- **Test Methods**:
  - `testRegistrationClosed()` - Validates error response when registration is disabled
  - `testRegisterExistingUser()` - Validates error response when attempting to register with existing email
- **Dependencies**: Uses registration API, data generators, database collections, and error response DTOs

## Usage Example

```java
// This test class is executed as part of the test suite to verify registration error handling
// Example test execution flow:

@Test
public void testRegistrationClosed() {
    // 1. Find existing user in database
    AuthUser existingUser = findUser();
    
    // 2. Create registration request with existing user data
    UserRegistrationRequest user = existingUserRequst(existingUser);
    
    // 3. Generate expected error response for closed registration
    ErrorResponse expectedResponse = registrationClosedResponse();
    
    // 4. Attempt registration and verify error response
    ErrorResponse response = attemptRegistration(user);
    assertThat(response).isEqualTo(expectedResponse);
}
```

These tests ensure the registration system properly handles edge cases and returns appropriate error messages, maintaining system security and user experience consistency.
