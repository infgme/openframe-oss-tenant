This utility class generates fake user registration data and response objects for testing purposes in the OpenFrame platform, leveraging the DataFaker library to create realistic test scenarios.

## Key Components

- **newUserRegistrationRequest()** - Creates a new user registration request with fake data
- **existingUserRequst()** - Generates a registration request for an existing user
- **newUserRegistrationResponse()** - Creates a registration response with default tenant settings
- **meResponse()** - Builds an authenticated user response
- **registrationClosedResponse()** / **existingUserResponse()** - Generate error responses for registration failures
- **registeredOwner()** - Creates a User object with OWNER and ADMIN roles from registration data

## Usage Example

```java
// Generate a new user registration request
UserRegistrationRequest request = RegistrationGenerator.newUserRegistrationRequest();

// Create corresponding registration response
UserRegistrationResponse response = RegistrationGenerator.newUserRegistrationResponse(request);

// Generate owner user from registration
User owner = RegistrationGenerator.registeredOwner(request, response);

// Create error response for closed registration
ErrorResponse error = RegistrationGenerator.registrationClosedResponse();

// Generate authenticated user response
MeResponse meResp = RegistrationGenerator.meResponse(owner);
```

The generator uses a consistent tenant domain ("localhost"), applies alphanumeric filtering to company names, and assigns default passwords from the UserConfig. All generated users receive both OWNER and ADMIN roles, making this ideal for testing administrative scenarios in the OpenFrame MSP platform.
