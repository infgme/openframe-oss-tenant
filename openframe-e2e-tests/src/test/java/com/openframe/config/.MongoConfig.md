MongoDB configuration utility class that provides centralized access to database connection parameters through environment variables, system properties, or sensible defaults for the OpenFrame platform.

## Key Components

- **Configuration Methods**: Static getter methods for MongoDB connection parameters
  - `getMongoDbUri()` - Returns MongoDB connection URI
  - `getDatabaseName()` - Returns target database name
  - `getMongoUser()` - Returns MongoDB username
  - `getMongoPassword()` - Returns MongoDB password
  - `getAuthDatabase()` - Returns authentication database name

- **Configuration Priority**: Each parameter follows a hierarchical configuration approach:
  1. System properties (JVM arguments)
  2. Environment variables
  3. Default values for Kubernetes deployment

- **Default Values**: Pre-configured for OpenFrame's Kubernetes environment with MongoDB service discovery

## Usage Example

```java
// Get MongoDB connection parameters
String uri = MongoConfig.getMongoDbUri();
String dbName = MongoConfig.getDatabaseName();
String user = MongoConfig.getMongoUser();
String password = MongoConfig.getMongoPassword();
String authDb = MongoConfig.getAuthDatabase();

// Example MongoDB client configuration
MongoClientSettings settings = MongoClientSettings.builder()
    .applyConnectionString(new ConnectionString(uri))
    .credential(MongoCredential.createCredential(user, authDb, password.toCharArray()))
    .build();

MongoClient mongoClient = MongoClients.create(settings);
MongoDatabase database = mongoClient.getDatabase(dbName);
```

The class supports both containerized environments (via environment variables) and development setups (via system properties), with lazy loading and debug logging for configuration transparency.
