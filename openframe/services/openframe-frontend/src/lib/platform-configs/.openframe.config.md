OpenFrame configuration file that defines the complete application settings, branding, navigation, and UI behavior for the OpenFrame platform.

## Key Components

- **openframeConfig** - Main configuration object implementing the `AppConfig` interface
- **Brand Configuration** - Name, logo, colors, and SEO metadata for OpenFrame platform
- **Navigation Setup** - Header logo component, allowed/restricted routes, and navigation visibility
- **UI Configuration** - Header actions, mobile navigation, user menu, and interactive elements
- **Footer Configuration** - Logo, sections, and company links
- **Contact & Social** - Email, support URLs, and social media links

## Usage Example

```typescript
import { openframeConfig } from './openframe.config'

// Access platform configuration
const platformName = openframeConfig.name // 'OpenFrame'
const baseUrl = openframeConfig.url // Dynamic base URL
const primaryColor = openframeConfig.brandColors.primary // '#5efaf0'

// Use in app initialization
function initializeApp() {
  document.title = openframeConfig.seo.title
  
  // Configure routing with allowed routes
  const allowedRoutes = openframeConfig.navigation.allowedRoutes
  // ['/docs', '/examples', '/api', '/profile', '/contact']
  
  // Setup header with custom logo component
  const logoElement = openframeConfig.navigation.logo.getElement()
}

// Get header actions for authenticated user
const headerActions = openframeConfig.ui.getHeaderActions({
  user: currentUser,
  router: appRouter,
  pathname: '/docs',
  onSignUp: handleSignUp
})
```

The configuration uses dynamic URL generation, custom React components for branding, and provides flexible header actions based on authentication state. Brand colors are marked for metadata only as the actual UI uses the ODS (OpenFrame Design System) CSS classes.
