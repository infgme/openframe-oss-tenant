Defines the message structure for communicating tool installation instructions between agents in the Flamingo MSP platform, handling installation, uninstallation, and execution configurations.

## Key Components

- **`ToolInstallationMessage`** - Main struct containing all tool installation parameters including agent IDs, version info, command arguments, and optional assets
- **`SessionType`** - Enum defining execution contexts (Service, Console, User) for tool operations  
- **`Asset`** - Struct representing downloadable files with metadata for tool installation
- **`AssetSource`** - Enum specifying where assets are sourced from (Artifactory or Tool API)

## Usage Example

```rust
use serde_json;

// Create a tool installation message for a monitoring agent
let installation_msg = ToolInstallationMessage {
    tool_agent_id: "agent-001".to_string(),
    tool_id: "monitoring-tool".to_string(),
    tool_type: "system-monitor".to_string(),
    version: "1.2.3".to_string(),
    reinstall: false,
    session_type: Some(SessionType::Service),
    download_configurations: None,
    installation_command_args: Some(vec![
        "--silent".to_string(),
        "--config=/etc/monitor.conf".to_string()
    ]),
    uninstallation_command_args: Some(vec!["--remove-config".to_string()]),
    run_command_args: vec!["--daemon".to_string()],
    tool_agent_id_command_args: None,
    assets: Some(vec![Asset {
        id: "config-file".to_string(),
        local_filename: "monitor.conf".to_string(),
        source: AssetSource::Artifactory,
        path: Some("/etc/".to_string()),
        executable: false,
    }]),
};

// Serialize for transmission
let json = serde_json::to_string(&installation_msg)?;
```
