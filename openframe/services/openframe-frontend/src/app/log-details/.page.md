A Next.js page component that displays detailed log information, requiring specific search parameters for log identification and redirecting to the logs overview if any parameters are missing.

## Key Components

- **LogDetailsPageProps**: Interface defining expected search parameters including log ID, ingest day, tool type, event type, and timestamp
- **LogDetailsPage**: Main page component that validates required parameters and renders the log details view
- **Dynamic rendering configuration**: Forces dynamic rendering to support `useSearchParams` in the AppLayout component

## Usage Example

This page is accessed via URL with search parameters:

```typescript
// Example URL structure:
// /log-details?id=log123&ingestDay=2024-01-15&toolType=monitoring&eventType=error&timestamp=1642234567

// The component automatically validates all required parameters
// and redirects to /logs-page if any are missing:
if (!id || !ingestDay || !toolType || !eventType || !timestamp) {
  redirect('/logs-page')
}

// On successful validation, renders the log details:
<LogDetailsView 
  logId={id} 
  ingestDay={ingestDay} 
  toolType={toolType} 
  eventType={eventType} 
  timestamp={timestamp} 
/>
```

The page serves as a parameter validation layer before displaying detailed log information through the `LogDetailsView` component, ensuring all necessary data is present for proper log rendering.
