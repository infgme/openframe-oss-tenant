A Zustand-based authentication store that manages user login state, profile data, and session persistence following OpenFrame patterns with optimized re-rendering through selectors.

## Key Components

- **AuthState Interface** - Defines the complete store structure with user data, loading states, and actions
- **User Interface** - Comprehensive user model with profile fields, organization data, and image handling
- **useAuthStore Hook** - Main store hook created with Zustand, Immer, persistence, and devtools middleware
- **fetchUserProfile Helper** - Async function to retrieve full user profile data from the API
- **Optimized Selectors** - Pre-built selectors (`selectUser`, `selectIsAuthenticated`, etc.) for performance

## Usage Example

```typescript
import { useAuthStore, selectUser, selectIsAuthenticated } from './auth-store'

// In a component - using the full store
function UserProfile() {
  const { user, login, logout, fetchFullProfile, isLoadingProfile } = useAuthStore()
  
  const handleLogin = async () => {
    const newUser = { id: '123', email: 'user@example.com' }
    login(newUser)
    // Optionally fetch full profile after login
    await fetchFullProfile()
  }
  
  return (
    <div>
      {user ? (
        <div>
          <h1>Welcome {user.firstName || user.email}</h1>
          <button onClick={logout}>Logout</button>
        </div>
      ) : (
        <button onClick={handleLogin}>Login</button>
      )}
    </div>
  )
}

// Using optimized selectors for better performance
function UserStatus() {
  const isAuthenticated = useAuthStore(selectIsAuthenticated)
  const user = useAuthStore(selectUser)
  
  return <div>Status: {isAuthenticated ? `Logged in as ${user?.email}` : 'Not authenticated'}</div>
}
```

The store automatically persists authentication state to localStorage and provides both basic user data from initial login and the ability to fetch extended profile information on demand.
