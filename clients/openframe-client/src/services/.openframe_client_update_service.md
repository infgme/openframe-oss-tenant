Coordinates the complete client update process for OpenFrame, handling version validation, binary downloads, and platform-specific updater deployment with race condition protection and comprehensive error handling.

## Key Components

- **OpenFrameClientUpdateService** - Main service managing the update workflow with thread-safe operations
- **update_in_progress** - Mutex-protected flag preventing concurrent updates
- **process_update()** - Public entry point with race condition protection
- **execute_update()** - Core update logic handling download, extraction, and updater launch
- **create_temp_archive()** - Platform-specific archive creation (ZIP on Windows, direct binary on Unix)
- **Version validation** - Semver parsing and format validation for update requests

## Usage Example

```rust
use openframe_client_update_service::OpenFrameClientUpdateService;
use openframe_client_update_message::OpenFrameClientUpdateMessage;

// Initialize the service with required dependencies
let update_service = OpenFrameClientUpdateService::new(
    client_info_service,
    github_download_service,
    update_state_service,
);

// Process an update request
let message = OpenFrameClientUpdateMessage {
    version: "v2.1.0".to_string(),
    download_configurations: vec![/* platform-specific configs */],
};

// Service handles validation, download, and updater launch
match update_service.process_update(message).await {
    Ok(_) => println!("Update process initiated successfully"),
    Err(e) => eprintln!("Update failed: {}", e),
}
```

The service ensures only one update runs at a time, validates version formats using semver, downloads platform-appropriate binaries, and launches the updater with proper cleanup on failure. Update state is tracked throughout the process for monitoring and recovery.
