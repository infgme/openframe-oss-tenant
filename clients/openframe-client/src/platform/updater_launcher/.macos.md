Implements macOS-specific update launcher that creates a temporary launchd job to execute update scripts with service survival guarantees.

## Key Components

- **`launch_updater`** - Main function that orchestrates the macOS update process using launchd jobs
- **Script Management** - Creates temporary executable bash scripts with proper Unix permissions
- **Launchd Integration** - Manages plist creation and job loading for service-independent execution
- **Template Substitution** - Replaces placeholders in plist templates with runtime parameters

## Usage Example

```rust
use crate::updater::macos::launch_updater;
use crate::UpdaterParams;
use std::path::PathBuf;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let params = UpdaterParams {
        binary_path: PathBuf::from("/usr/local/bin/openframe"),
        service_name: "com.openframe.service".to_string(),
        target_exe: PathBuf::from("/Applications/OpenFrame.app"),
        update_state_path: "/tmp/update.state".to_string(),
    };

    launch_updater(params).await?;
    Ok(())
}
```

The function creates a temporary bash script and plist file, sets executable permissions, and uses `launchctl` to load a one-shot job that survives the current service shutdown. This ensures reliable updates on macOS systems by leveraging the system's native service management.
