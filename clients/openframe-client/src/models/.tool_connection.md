Defines a data structure representing the connection between AI agents and tools in the Flamingo platform, tracking which tools are available to which agents and their publication status.

## Key Components

- **`ToolConnection` struct** - Core data structure with fields for tool-agent relationships
  - `tool_agent_id` - Identifier for the tool from the agent's perspective
  - `agent_tool_id` - Identifier for the agent from the tool's perspective  
  - `published` - Boolean flag indicating if the connection is active/published
- **Serde traits** - Automatic serialization/deserialization support for JSON/API operations
- **Default implementation** - Provides sensible defaults with empty strings and unpublished status

## Usage Example

```rust
use tool_connection::ToolConnection;

// Create a new tool connection
let connection = ToolConnection {
    tool_agent_id: "mingo-ai-assistant".to_string(),
    agent_tool_id: "network-scanner-v2".to_string(),
    published: true,
};

// Create with defaults and update
let mut connection = ToolConnection::default();
connection.tool_agent_id = "fae-client-bot".to_string();
connection.agent_tool_id = "ticket-manager".to_string();
connection.published = true;

// Serialize for API transmission
let json = serde_json::to_string(&connection)?;
```

This structure enables the OpenFrame platform to manage which AI tools (like Mingo AI for technicians or Fae for clients) can access specific MSP automation tools, with granular control over publication and availability.
