This file provides cross-platform uninstallation functionality for the OpenFrame client service, with robust directory removal and integrated tool cleanup capabilities.

## Key Components

- **remove_directory_with_retry()** - Asynchronous directory removal with exponential backoff retry logic and file unlocking
- **unlock_directory_files()** - Platform-specific file attribute modification to handle locked files (Windows: attrib, Unix: chmod)
- **force_remove_directory()** - System command-based forced removal as fallback (Windows: rd/takeown/icacls, Unix: rm -rf)
- **uninstall_integrated_tools()** - Orchestrates graceful uninstallation of all integrated tools using service layer
- **uninstall_windows()** - Complete Windows uninstallation workflow including service removal and cleanup

## Usage Example

```rust
use crate::platform::DirectoryManager;
use crate::uninstall::{uninstall_windows, remove_directory_with_retry};
use std::path::Path;

async fn perform_uninstall() -> anyhow::Result<()> {
    let dir_manager = DirectoryManager::new()?;
    let install_path = Path::new("C:\\Program Files\\OpenFrame");
    
    // Complete Windows uninstallation
    uninstall_windows(&dir_manager, install_path).await?;
    
    // Or remove specific directory with retry logic
    remove_directory_with_retry(Path::new("C:\\temp\\mydir"), 5).await?;
    
    Ok(())
}
```

The module handles common uninstallation challenges like locked files, permission issues, and service cleanup with platform-specific optimizations and comprehensive error recovery mechanisms.
