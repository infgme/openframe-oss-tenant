Manages application mode configuration to control feature availability across different deployment scenarios (OSS tenant, SaaS tenant, and SaaS shared). This utility determines which authentication and application features are enabled based on runtime environment settings.

## Key Components

- **`AppMode`** - Type definition for the three supported modes: `'oss-tenant' | 'saas-tenant' | 'saas-shared'`
- **`getAppMode()`** - Retrieves current app mode from runtime environment, defaults to 'oss-tenant'
- **Mode Check Functions** - `isOssTenantMode()`, `isSaasTenantMode()`, `isSaasSharedMode()` for specific mode detection
- **Feature Availability** - `isAuthEnabled()`, `isAppEnabled()` to determine which features are active
- **Route Validation** - `isRouteAllowedInCurrentMode()` checks if routes are accessible in current mode
- **Navigation Helpers** - `getDefaultRedirectPath()`, `shouldShowNavigationSidebar()` for UI behavior

## Usage Example

```typescript
import { 
  getAppMode, 
  isAuthEnabled, 
  isRouteAllowedInCurrentMode,
  getDefaultRedirectPath 
} from './app-mode'

// Check current mode and configure features
const mode = getAppMode()
console.log(`Running in ${mode} mode`)

// Conditionally show auth features
if (isAuthEnabled()) {
  renderAuthPages()
}

// Validate route access
const canAccess = isRouteAllowedInCurrentMode('/dashboard')
if (!canAccess) {
  redirect(getDefaultRedirectPath(isAuthenticated))
}

// Configure navigation based on mode
if (shouldShowNavigationSidebar()) {
  renderSidebar()
}
```
