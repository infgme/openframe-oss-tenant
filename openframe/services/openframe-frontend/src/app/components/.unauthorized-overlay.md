A client-side React component that displays an overlay prompting users to sign in when they encounter unauthorized content.

## Key Components

- **UnauthorizedOverlay** - Main component that renders a centered sign-in prompt with authentication redirection
- **UnauthorizedOverlayProps** - Interface defining optional retry callback prop
- **handleLogin** - Function that redirects to auth login URL or executes retry callback as fallback

## Usage Example

```typescript
import { UnauthorizedOverlay } from './unauthorized-overlay'

// Basic usage with auth redirect
function ProtectedPage() {
  const [isAuthorized, setIsAuthorized] = useState(false)
  
  if (!isAuthorized) {
    return <UnauthorizedOverlay />
  }
  
  return <div>Protected content</div>
}

// With custom retry logic
function ProtectedPageWithRetry() {
  const [isAuthorized, setIsAuthorized] = useState(false)
  
  const handleRetry = () => {
    // Custom retry logic
    checkAuthStatus().then(setIsAuthorized)
  }
  
  if (!isAuthorized) {
    return <UnauthorizedOverlay onRetry={handleRetry} />
  }
  
  return <div>Protected content</div>
}
```

The component uses OpenFrame design system (ODS) classes for consistent styling and automatically redirects to the configured authentication URL when the sign-in button is clicked. If no auth URL is configured, it falls back to the provided retry callback.
