Configuration module for establishing secure WebSocket connections to MeshCentral server instances, handling URL construction and authentication for remote management capabilities.

## Key Components

- **getMeshBaseHostPort()** - Extracts and normalizes the base host/port from tenant configuration, stripping protocol prefixes
- **getMeshWsScheme()** - Determines WebSocket scheme (`ws`/`wss`) based on browser protocol, defaulting to secure `wss`
- **buildWsUrl(path)** - Constructs complete WebSocket URLs with proper routing through the proxy path `/ws/tools/meshcentral-server`
- **MESH_USER/MESH_PASS** - Default authentication credentials from environment variables
- **MeshUrlParts** - Type definition for URL components with scheme validation

## Usage Example

```typescript
import { buildWsUrl, MESH_USER, MESH_PASS } from './meshcentral-config'

// Build WebSocket URL for MeshCentral connection
const wsUrl = buildWsUrl('/control.ashx')
// Result: "wss://your-domain.com/ws/tools/meshcentral-server/control.ashx"

// Create WebSocket connection with authentication
const socket = new WebSocket(wsUrl)
socket.onopen = () => {
  // Send auth credentials
  socket.send(JSON.stringify({
    action: 'authcookie',
    username: MESH_USER,
    password: MESH_PASS
  }))
}

// Handle different environments
const baseHost = getMeshBaseHostPort()
if (baseHost) {
  console.log(`Connecting to MeshCentral at: ${baseHost}`)
}
```

The module automatically handles protocol detection, proxy routing, and fallback configurations for both development and production environments.
