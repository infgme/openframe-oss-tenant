A utility function that returns the appropriate icon component for different SSO (Single Sign-On) providers in the Flamingo authentication system.

## Key Components

- **`getProviderIcon`** - Main function that maps provider keys to their corresponding icon components
- **Supported Providers** - Microsoft, Google, and Slack SSO providers
- **Icon Components** - Uses pre-built icons from the OpenFrame frontend core library

## Usage Example

```typescript
import { getProviderIcon } from './get-provider-icon'

// Get Microsoft icon with default styling
const microsoftIcon = getProviderIcon('microsoft')

// Get Google icon with custom styling
const googleIcon = getProviderIcon('google', 'h-8 w-8 text-blue-500')

// Handle unsupported provider (returns null)
const unknownIcon = getProviderIcon('unsupported') // returns null

// Use in JSX
function LoginButton({ provider }: { provider: string }) {
  const icon = getProviderIcon(provider)
  
  return (
    <button className="flex items-center gap-2">
      {icon}
      Sign in with {provider}
    </button>
  )
}
```

The function is case-insensitive and returns `null` for unsupported providers, making it safe to use in conditional rendering scenarios. Default styling applies Tailwind classes for a 24x24px icon that won't shrink in flex containers.
