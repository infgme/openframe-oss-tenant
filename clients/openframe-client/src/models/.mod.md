A module definition file that centralizes all data structures and message types used for agent communication and management within the OpenFrame platform.

## Key Components

**Core Agent Management:**
- `AgentRegistrationRequest/Response` - Agent registration flow
- `AgentConfiguration` - Agent configuration settings
- `InitialConfiguration` - Initial setup parameters
- `AgentTokenResponse` - Authentication token handling

**Tool Management:**
- `ToolInstallationResult/Message` - Tool installation handling
- `ToolConnectionMessage` - Tool connection management
- `InstalledTool` with `ToolStatus` - Tool state tracking
- `ToolConnection` - Active tool connections

**Client Communication:**
- `OpenFrameClientUpdateMessage` - Client update notifications
- `ToolAgentUpdateMessage` - Agent-specific updates
- `OpenFrameClientInfo` - Client information structure
- `MachineHeartbeatMessage` - System health monitoring

**System State:**
- `InstalledAgentMessage` - Agent installation status
- `DownloadConfiguration` - Download settings
- `UpdateState` with `UpdatePhase` - Update process tracking

## Usage Example

```rust
use crate::messages::{
    AgentRegistrationRequest,
    ToolInstallationResult,
    UpdateState,
    SessionType
};

// Register a new agent
let registration = AgentRegistrationRequest {
    agent_id: "agent-001".to_string(),
    capabilities: vec!["monitoring".to_string()],
};

// Handle tool installation
let install_result = ToolInstallationResult::success(
    "tool-id".to_string(),
    SessionType::Interactive
);

// Track update progress
let update_state = UpdateState::new(UpdatePhase::Downloading);
```
