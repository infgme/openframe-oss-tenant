WebSocket client for controlling MeshCentral agent connections and managing remote device operations through the MeshCentral control interface.

## Key Components

**MeshControlClient**
- Main client class for MeshCentral control operations
- Manages WebSocket connection with authentication and reconnection logic
- Handles tunnel creation for desktop/file access and power management commands

**ControlAuthCookies Type**
- Defines authentication cookie structure for MeshCentral sessions
- Contains required auth cookie and optional relay cookie

**Core Methods**
- `openSession()` - Establishes authenticated WebSocket connection
- `getAuthCookies()` - Retrieves session authentication cookies
- `sendDesktopTunnel()` / `sendFileTunnel()` - Creates tunnels for remote access
- `powerAction()` - Executes power commands (wake, sleep, reset, poweroff)

## Usage Example

```typescript
// Initialize with credentials
const client = new MeshControlClient({
  user: 'admin',
  pass: 'password'
})

// Open authenticated session
await client.openSession()

// Create desktop tunnel for remote access
client.sendDesktopTunnel('node-id-123', 'relay-id-456')

// Execute power action
await client.powerAction('node-id-123', 'wake')

// Get authentication cookies for other requests
const cookies = await client.getAuthCookies()

// Clean up
client.close()
```

The client automatically handles authentication, maintains active tunnel state, and provides reconnection capabilities for reliable MeshCentral integration.
