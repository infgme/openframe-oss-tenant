A fluent API implementation for orchestrating OAuth2 authentication flows in OpenFrame's test automation framework. This class provides a builder-pattern interface to execute the complete authentication process from initial OAuth flow to token extraction.

## Key Components

- **`login(User user)`** - Static entry point that executes the complete OAuth2 flow and returns authentication tokens
- **`AuthFlow(User user)`** - Private constructor that initializes authentication parts using the provided user
- **Flow Methods** - Chain of private methods implementing each OAuth2 step:
  - `startFlow()` - Initiates OAuth flow and extracts state/challenge parameters
  - `initAuth()` - Handles authorization initialization
  - `postCredentials()` - Submits user credentials
  - `getAuthCode()` - Retrieves authorization code from redirect
  - `extractTokens()` - Exchanges code for access/refresh tokens

## Usage Example

```java
// Execute complete authentication flow for a user
User testUser = User.builder()
    .username("test@example.com")
    .password("password123")
    .build();

Map<String, String> tokens = AuthFlow.login(testUser);

// Extract tokens for API calls
String accessToken = tokens.get("access_token");
String refreshToken = tokens.get("refresh_token");

// Use tokens in subsequent API requests
given()
    .cookie("access_token", accessToken)
    .when()
    .get("/api/protected-resource")
    .then()
    .statusCode(200);
```

The class uses a fluent builder pattern where each method returns `this` to enable method chaining, making the authentication flow readable and maintainable. All HTTP responses are validated for expected status codes (302 redirects) and cookies/headers are automatically managed throughout the flow.
