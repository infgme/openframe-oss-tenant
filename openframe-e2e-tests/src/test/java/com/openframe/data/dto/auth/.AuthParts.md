A data transfer object (DTO) that encapsulates authentication-related parameters and state information for OAuth2 and multi-tenant authentication flows. This class uses Lombok annotations for automatic generation of getters, setters, and builder pattern methods.

## Key Components

- **Basic Auth Fields**: `email`, `password` - Standard username/password credentials
- **Multi-tenant Support**: `tenantId` - Identifies the specific tenant for multi-tenant applications
- **OAuth2/PKCE Fields**: `state`, `codeChallenge`, `codeVerifier`, `authorizationCode` - Support for OAuth2 authorization code flow with PKCE (Proof Key for Code Exchange)
- **Session Management**: `cookies` - Map for storing authentication-related cookies
- **Lombok Annotations**: `@Data` for getters/setters, `@Builder` for builder pattern

## Usage Example

```java
// Building auth parts for OAuth2 flow
AuthParts oauthParts = AuthParts.builder()
    .email("user@example.com")
    .tenantId("tenant-123")
    .state("random-state-value")
    .codeChallenge("challenge-hash")
    .codeVerifier("verifier-string")
    .cookies(Map.of("session", "abc123", "csrf", "xyz789"))
    .build();

// Building for simple password auth
AuthParts passwordAuth = AuthParts.builder()
    .email("admin@company.com")
    .password("securePassword")
    .tenantId("main-tenant")
    .build();

// Processing authorization code callback
AuthParts callbackParts = AuthParts.builder()
    .authorizationCode("auth-code-from-callback")
    .state("original-state-value")
    .codeVerifier("original-verifier")
    .build();
```
