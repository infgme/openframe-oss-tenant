Centralized authentication API client that manages OAuth flows, token refresh, and user authentication endpoints for the Flamingo platform. Handles both shared SaaS mode and standalone deployments with automatic token management and error recovery.

## Key Components

- **AuthApiClient**: Main client class handling authentication requests and token refresh logic
- **buildAuthUrl()**: URL builder that switches between shared host and relative URLs based on deployment mode
- **getDomainSuffix()**: Extracts domain suffix for SaaS tenant management
- **request/requestRefresh**: HTTP wrapper functions with automatic token attachment and error handling
- **Token Management**: Automatic access token refresh with concurrent request handling
- **OAuth Methods**: Complete OAuth flow support including SSO, registration, and password reset

## Usage Example

```typescript
import { authApiClient } from './auth-api-client'

// Authenticate user
const loginResponse = await authApiClient.oauth('login', {
  email: 'user@example.com',
  password: 'password'
})

// Register new organization
const registerResponse = await authApiClient.registerOrganization({
  email: 'admin@company.com',
  firstName: 'John',
  lastName: 'Doe',
  password: 'securepass',
  tenantName: 'ACME Corp',
  tenantDomain: 'acme.flamingo.run'
})

// Handle SSO registration redirect
authApiClient.registerOrganizationSSO({
  tenantName: 'ACME Corp',
  tenantDomain: 'acme.flamingo.run',
  email: 'admin@company.com',
  provider: 'google',
  accessCode: 'abc123'
})

// Check domain availability
const availability = await authApiClient.checkDomainAvailability(
  'acme', 
  'ACME Corporation'
)
```

The client automatically handles token refresh, domain resolution, and provides a consistent interface for all authentication operations across different deployment modes.
