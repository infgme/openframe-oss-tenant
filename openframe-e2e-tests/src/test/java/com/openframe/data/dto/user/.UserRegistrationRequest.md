A Data Transfer Object (DTO) that encapsulates user registration data for API requests and testing purposes in the OpenFrame platform. This class provides a structured way to handle user registration information including personal details and tenant configuration.

## Key Components

- **@Data** - Lombok annotation that generates getters, setters, toString, equals, and hashCode methods
- **@Builder** - Enables fluent builder pattern for object construction with `toBuilder = true` for creating modified copies
- **@NoArgsConstructor/@AllArgsConstructor** - Generates default and full parameter constructors
- **Core Fields**:
  - `email` - User's email address (typically used as username)
  - `firstName/lastName` - User's personal information
  - `password` - User's chosen password for authentication
  - `tenantName/tenantDomain` - Multi-tenant configuration for MSP platform isolation

## Usage Example

```java
// Using builder pattern for clean object construction
UserRegistrationRequest request = UserRegistrationRequest.builder()
    .email("john.doe@example.com")
    .firstName("John")
    .lastName("Doe")
    .password("securePassword123")
    .tenantName("Acme IT Services")
    .tenantDomain("acme-it")
    .build();

// Creating a modified copy with different tenant
UserRegistrationRequest modifiedRequest = request.toBuilder()
    .tenantName("New MSP Company")
    .tenantDomain("new-msp")
    .build();

// Using in test scenarios or API calls
registrationService.registerUser(request);
```

This DTO is particularly useful for test scenarios and API integration where user registration data needs to be passed between layers of the OpenFrame MSP platform while maintaining tenant isolation.
