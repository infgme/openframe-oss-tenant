This module provides cross-platform directory management for the OpenFrame application, handling platform-specific paths and permissions for Windows, macOS, and Linux systems.

## Key Components

### Types
- **`DirectoryError`** - Error types for directory operations including creation failures, permission issues, and validation problems
- **`DirectoryManager`** - Main struct providing unified directory management across platforms

### Functions
- **`get_app_support_directory()`** - Returns platform-specific application support directory
- **`get_logs_directory()`** - Returns platform-specific logs directory (supports `OPENFRAME_LOG_DIR` override)
- **`get_secured_directory()`** - Returns admin/root-only secured directory path
- **`set_directory_permissions()`** - Sets appropriate permissions (755 on Unix, default on Windows)
- **`set_secured_directory_permissions()`** - Sets admin-only permissions (700 on Unix)

### DirectoryManager Methods
- **`new()`** - Creates manager with default platform paths
- **`with_custom_dirs()`** - Creates manager with custom directory paths
- **`with_user_logs_dir()`** - Creates manager with user-specific logging
- **`for_development()`** - Creates development mode manager using only user directories

## Usage Example

```rust
use openframe::directories::DirectoryManager;

// Standard production setup
let manager = DirectoryManager::new();

// Development setup (avoids permission issues)
let dev_manager = DirectoryManager::for_development();

// Custom directories
let custom_manager = DirectoryManager::with_custom_dirs(
    PathBuf::from("/custom/logs"),
    PathBuf::from("/custom/app"),
    PathBuf::from("/custom/secured")
);

// Create directories with proper permissions
if let Err(e) = std::fs::create_dir_all(&manager.logs_dir) {
    eprintln!("Failed to create logs directory: {}", e);
}
```

The module automatically handles platform differences like Windows `ProgramData`, macOS `/Library`, and Linux `/var/lib` paths, along with appropriate ownership and permission settings for each platform.
