A Next.js page component that wraps the script editing functionality, handling both creating new scripts and editing existing ones based on URL parameters.

## Key Components

- **EditScriptPageWrapper**: Main page component that processes route parameters to determine edit vs. create mode
- **Parameter handling logic**: Converts URL parameter 'new' to null for create mode, or passes script ID for edit mode
- **AppLayout wrapper**: Provides consistent application layout structure
- **EditScriptPage**: The actual script editing component

## Usage Example

This page is accessed through Next.js routing at `/scripts/[id]` where:

```typescript
// URL: /scripts/new
// Result: scriptId = null (create new script)

// URL: /scripts/abc123
// Result: scriptId = "abc123" (edit existing script)

// The component structure:
<AppLayout>
  <EditScriptPage scriptId={scriptId} />
</AppLayout>
```

The component uses Next.js 13+ App Router conventions with the `'use client'` directive for client-side routing, and `useParams()` to extract the dynamic `id` parameter. The logic ensures that only the string 'new' triggers create mode, while any other string value is treated as a script ID for editing.
