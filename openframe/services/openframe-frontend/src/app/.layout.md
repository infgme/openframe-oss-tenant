Root layout component for the OpenFrame application that configures metadata, fonts, styling, and core providers for the MSP management platform.

## Key Components

- **`metadata`** - Comprehensive SEO metadata including OpenGraph, Twitter cards, favicons, and manifest for OpenFrame branding
- **Font Configuration** - Integrates Azeret Mono and DM Sans custom fonts with CSS variables
- **Core Providers** - Sets up QueryClient, RouteGuard, and authentication-dependent services
- **Development Tools** - Includes DevTicketObserver and GraphQLIntrospectionInitializer for development workflows
- **App Shell** - Implements loading states with AppShellSkeleton and manages authentication flows

## Usage Example

```typescript
// This layout automatically wraps all pages in the app
// Pages are rendered within the children prop:

export default function HomePage() {
  return (
    <div>
      <h1>Welcome to OpenFrame</h1>
      {/* Content inherits fonts, providers, and styling from layout */}
    </div>
  )
}

// The layout provides:
// - Dark theme with OpenFrame branding
// - Font variables: font-mono (Azeret Mono), font-body (DM Sans)
// - Global CSS and OpenFrame core styles
// - Authentication state management
// - Query client for API calls
// - Route protection when auth is enabled
```

The layout enforces dynamic rendering to prevent SSG issues and conditionally loads authentication components based on the app mode configuration. It establishes the foundation for all OpenFrame pages with consistent branding, typography, and core functionality.
