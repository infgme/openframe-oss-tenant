Centralized utility for navigating to log details pages across different components in the Flamingo platform. Handles parameter extraction from various log entry formats and constructs proper URLs.

## Key Components

- **navigateToLogDetails()** - Main navigation function that extracts log parameters and constructs the detail page URL
- Supports both UI-Kit LogEntry objects and transformed logs with originalLogEntry fields
- Handles multiple log ID formats (id, toolEventId) and required routing parameters

## Usage Example

```typescript
import { useRouter } from 'next/router'
import { navigateToLogDetails } from './log-navigation'

// In a dashboard component
const handleLogClick = (log: LogEntry) => {
  navigateToLogDetails(router, log)
}

// In a logs table with transformed entries
const handleRowClick = (transformedLog: any) => {
  // Works with logs that have originalLogEntry field
  navigateToLogDetails(router, transformedLog)
}

// Manual router usage
const router = useRouter()
navigateToLogDetails(router, {
  id: 'log-123',
  originalLogEntry: {
    ingestDay: '2024-01-15',
    toolType: 'tacticalrmm',
    eventType: 'alert',
    timestamp: '2024-01-15T10:30:00Z'
  }
})
```

The function automatically extracts required parameters (ingestDay, toolType, eventType, timestamp) from either the log object directly or from its originalLogEntry field, making it compatible with different data structures across the application.
