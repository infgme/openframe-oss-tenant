Integration test suite that validates the complete user invitation workflow, including creating invitations, accepting them, handling conflicts, and managing user lifecycle events.

## Key Components

- **UserInvitationsTest** - Main test class extending `AuthorizedTest` for authenticated test scenarios
- **Invitation Lifecycle Tests** - Tests for inviting, accepting, and revoking user invitations
- **Conflict Resolution Tests** - Validates proper handling of duplicate invitations and status conflicts
- **User Management Tests** - Tests user deletion, owner protection, and re-invitation of deleted users
- **Status Validation** - Comprehensive checks for invitation and user status transitions

## Usage Example

```java
// Test ordering ensures proper workflow sequence
@Order(1)
@Test
@DisplayName("Invite New user")
public void testInviteUser() {
    InvitationRequest invitationRequest = newUserInvitationRequest();
    Invitation apiInvitation = inviteUser(invitationRequest);
    Invitation dbInvitation = findInvitation(invitationRequest.getEmail());
    
    // Verify invitation created successfully
    assertThat(dbInvitation).isNotNull();
    assertThat(apiInvitation).usingRecursiveComparison()
        .ignoringFields("updatedAt", "expiresAt", "createdAt")
        .isEqualTo(dbInvitation);
}

// Example conflict handling test
@Test
@DisplayName("Check that already Accepted Invitation cannot be accepted")
public void testAcceptAcceptedInvitation() {
    Invitation dbInvitation = findInvitation(InvitationStatus.ACCEPTED);
    AcceptInvitationRequest request = acceptInvitationRequest(dbInvitation);
    
    InvitationConflictResponse response = attemptAcceptInvitation(request);
    assertThat(response).isEqualTo(alreadyAcceptedResponse());
}
```

The test suite uses `@Order` annotations to ensure tests run in sequence, building upon previous test states to validate the complete invitation and user management workflow in OpenFrame.
