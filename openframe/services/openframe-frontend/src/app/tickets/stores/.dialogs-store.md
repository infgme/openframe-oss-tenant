A Zustand store that manages dialog state for both current and archived conversations in the Flamingo platform, providing pagination, search, filtering, and error handling capabilities.

## Key Components

- **DialogsStore Interface**: Defines state structure for current/archived dialogs with loading states, pagination info, and search parameters
- **useDialogsStore**: Main Zustand store hook with dual state management for current and archived dialogs
- **fetchDialogs**: Core async function that handles GraphQL queries with search, filtering, and pagination
- **Navigation Methods**: `goToNextPage` and `goToFirstPage` for cursor-based pagination
- **Reset Methods**: `resetCurrentDialogs` and `resetArchivedDialogs` for clearing state

## Usage Example

```typescript
import { useDialogsStore } from './dialogs-store'

function DialogsComponent() {
  const {
    currentDialogs,
    isLoadingCurrent,
    currentError,
    fetchDialogs,
    goToNextPage,
    resetCurrentDialogs
  } = useDialogsStore()

  // Fetch current dialogs with search
  const searchDialogs = async (searchTerm: string) => {
    await fetchDialogs(false, searchTerm, false, null, ['ACTIVE'])
  }

  // Navigate to next page
  const handleNextPage = () => {
    goToNextPage(false)
  }

  // Reset dialogs state
  const handleReset = () => {
    resetCurrentDialogs()
  }

  return (
    <div>
      {isLoadingCurrent ? 'Loading...' : currentDialogs.map(dialog => (
        <div key={dialog.id}>{dialog.title}</div>
      ))}
    </div>
  )
}
```

The store provides comprehensive dialog management with intelligent pagination reset logic, error handling, and separate state management for current vs archived conversations.
