Utility functions for processing and organizing AI policy configuration data into hierarchical permission categories with proper grouping and data transformation.

## Key Components

- **`slugify()`** - Converts strings to URL-safe slugs by normalizing case and special characters
- **`buildPolicyGroups()`** - Transforms flat policy rules array into nested Map structure organized by policy groups and categories
- **`clonePolicyGroups()`** - Creates deep clone of policy groups Map to prevent mutation of original data
- **`mapToObject()`** - Generic utility to convert Map instances to plain JavaScript objects

## Usage Example

```typescript
import { buildPolicyGroups, clonePolicyGroups } from './ai-settings.utils'
import type { PolicyRule } from '../types/ai-policies'

// Transform policy rules into organized groups
const policyRules: PolicyRule[] = [
  {
    naturalKey: 'user-create',
    operation: 'Create User',
    policyGroup: 'User Management',
    category: 'CRUD Operations',
    tool: 'azure-ad',
    approvalLevel: 'manager',
    commandPattern: 'create user *'
  }
]

// Build hierarchical permission structure
const policyGroups = buildPolicyGroups(policyRules)
// Returns: Map<"User Management", [{ id: "user-management:crud-operations", name: "CRUD Operations", policies: [...] }]>

// Create safe copy for manipulation
const clonedGroups = clonePolicyGroups(policyGroups)
```

The utilities handle data normalization, hierarchical organization, and safe cloning operations essential for AI policy management interfaces.
