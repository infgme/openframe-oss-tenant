Provides centralized error handling utilities for API responses and user notifications in React applications.

## Key Components

- **`getErrorMessage(error: unknown): string`** - Safely extracts human-readable error messages from various error types (Error objects, strings, or unknown values)
- **`handleApiError(error, toast, defaultMessage): void`** - Combines error message extraction with toast notification display for consistent error handling across the application

## Usage Example

```typescript
import { useMutation } from '@tanstack/react-query'
import { useToast } from '@/hooks/use-toast'
import { handleApiError } from './handle-api-error'

function CreateUserForm() {
  const { toast } = useToast()
  
  const createUser = useMutation({
    mutationFn: async (userData) => {
      const response = await fetch('/api/users', {
        method: 'POST',
        body: JSON.stringify(userData)
      })
      if (!response.ok) throw new Error('Failed to create user')
      return response.json()
    },
    onError: (error) => {
      handleApiError(error, toast, 'Failed to create user')
    }
  })
  
  // Component implementation...
}
```

This utility standardizes error handling patterns across React Query mutations and other async operations, ensuring consistent user feedback through toast notifications while gracefully handling different error formats.
