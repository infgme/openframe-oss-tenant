This is a Next.js page component that renders the Policies and Queries management interface within the OpenFrame MSP platform. It serves as the main entry point for viewing and managing organizational policies and automated queries.

## Key Components

- **PoliciesAndQueries**: Default exported React component that renders the complete page layout
- **AppLayout**: Wrapper component providing the main application shell and navigation
- **ContentPageContainer**: Core layout container from OpenFrame with customized padding and header settings
- **PoliciesAndQueriesView**: The main content component containing the policies and queries management interface

## Usage Example

```typescript
// This page is automatically routed by Next.js App Router
// Access via: /policies-and-queries

// The component structure provides:
// - Full application layout with navigation
// - Content container without padding or header
// - Dedicated policies and queries management view

// Example routing configuration:
// app/policies-and-queries/page.tsx
export default function PoliciesAndQueries() {
  return (
    <AppLayout>
      <ContentPageContainer padding="none" showHeader={false}>
        <PoliciesAndQueriesView />
      </ContentPageContainer>
    </AppLayout>
  )
}
```

The page uses `'use client'` directive for client-side rendering and `dynamic = 'force-dynamic'` to ensure the page is dynamically rendered on each request, which is essential for real-time policy and query management in MSP environments.
