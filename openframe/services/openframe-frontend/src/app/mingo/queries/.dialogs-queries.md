GraphQL query definitions for the Mingo AI dialog system, providing endpoints to fetch dialogs, individual dialog details, and associated messages with comprehensive pagination and filtering capabilities.

## Key Components

- **`GET_MINGO_DIALOGS_QUERY`** - Fetches paginated list of dialogs with filtering and search functionality
- **`GET_MINGO_DIALOG_QUERY`** - Retrieves detailed information for a specific dialog including owner, status, and ratings
- **`GET_DIALOG_MESSAGES_QUERY`** - Fetches all messages within a dialog with support for various message types (text, tool execution, approvals, errors)

## Usage Example

```typescript
import { GET_MINGO_DIALOGS_QUERY, GET_MINGO_DIALOG_QUERY } from './dialogs-queries';

// Fetch dialogs with pagination and search
const dialogsResult = await graphqlClient.query({
  query: GET_MINGO_DIALOGS_QUERY,
  variables: {
    filter: { status: 'OPEN' },
    pagination: { limit: 10 },
    search: 'network issue'
  }
});

// Get specific dialog details
const dialogResult = await graphqlClient.query({
  query: GET_MINGO_DIALOG_QUERY,
  variables: { id: 'dialog_123' }
});

// Fetch dialog messages with cursor pagination
const messagesResult = await graphqlClient.query({
  query: GET_DIALOG_MESSAGES_QUERY,
  variables: {
    dialogId: 'dialog_123',
    cursor: null,
    limit: 50
  }
});
```

The queries support advanced message types including tool execution status, approval workflows, and error handling - essential for the AI-powered MSP automation features.
