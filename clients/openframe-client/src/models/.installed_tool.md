Defines data structures for tracking tool installations on endpoints, including installation status and command configurations for tool lifecycle management.

## Key Components

- **`ToolStatus`** - Enum representing the installation status of a tool (currently only `Installed`)
- **`InstalledTool`** - Main struct containing tool metadata, version info, command arguments, and installation status
- **Default implementations** - Provides sensible defaults for both types with automatic serialization support

## Usage Example

```rust
use crate::models::installed_tool::{InstalledTool, ToolStatus};
use crate::models::SessionType;

// Create a new installed tool instance
let tool = InstalledTool {
    tool_agent_id: "agent_123".to_string(),
    tool_id: "monitoring_tool".to_string(),
    tool_type: "system_monitor".to_string(),
    version: "1.2.3".to_string(),
    session_type: SessionType::Service,
    run_command_args: vec!["--monitor".to_string(), "--verbose".to_string()],
    tool_agent_id_command_args: vec!["--agent-id".to_string()],
    uninstallation_command_args: Some(vec!["uninstall".to_string(), "--force".to_string()]),
    status: ToolStatus::Installed,
};

// Serialize to JSON
let json = serde_json::to_string(&tool)?;

// Check tool status
match tool.status {
    ToolStatus::Installed => println!("Tool is ready to use"),
}
```

The struct supports optional uninstallation commands and uses serde for JSON serialization with lowercase field naming for the status enum.
