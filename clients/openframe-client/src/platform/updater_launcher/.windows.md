Windows-specific updater implementation that launches a detached PowerShell script to perform application updates without console interference.

## Key Components

- **`launch_updater()`** - Main function that creates and executes a PowerShell script for updating applications on Windows
- **Script Management** - Generates unique temporary PowerShell script files using UUID for isolation
- **Process Creation** - Uses Windows-specific `CREATE_NO_WINDOW` flag for detached execution
- **Parameter Passing** - Forwards updater parameters (binary path, service name, target executable, update state) to PowerShell script

## Usage Example

```rust
use crate::platform::windows::launch_updater;
use crate::platform::UpdaterParams;

// Configure update parameters
let params = UpdaterParams {
    binary_path: "C:\\temp\\app-update.zip".to_string(),
    service_name: "OpenFrameService".to_string(),
    target_exe: "C:\\Program Files\\OpenFrame\\openframe.exe".to_string(),
    update_state_path: "C:\\temp\\update-state.json".to_string(),
};

// Launch the Windows updater
launch_updater(params).await?;
```

The function creates a temporary PowerShell script with execution policy bypass, runs it detached from the current console using the `CREATE_NO_WINDOW` flag, and passes all necessary update parameters as command-line arguments to handle the update process independently.
