A React hook and Zustand store for detecting and managing deployment environment information (cloud, self-hosted, or development) with global state management and convenient static helpers.

## Key Components

- **`useDeploymentStore`** - Zustand store managing deployment detection state globally
- **`useDeployment`** - Main React hook for accessing deployment information with loading states
- **`getDeployment`** - Static helper to access deployment info outside React components
- **Static helpers** - `isCloudDeployment`, `isSelfHostedDeployment`, `isDevelopmentDeployment`, `getDeploymentType` for quick environment checks

## Usage Example

```typescript
// In React components
function MyComponent() {
  const { 
    isLoading, 
    isCloud, 
    isDevelopment, 
    deploymentType, 
    hostname 
  } = useDeployment()

  if (isLoading) return <div>Detecting environment...</div>

  return (
    <div>
      <p>Environment: {deploymentType}</p>
      <p>Hostname: {hostname}</p>
      {isCloud && <p>Running on cloud platform</p>}
    </div>
  )
}

// Outside React components
function apiConfig() {
  if (isCloudDeployment()) {
    return { apiUrl: 'https://api.flamingo.run' }
  }
  return { apiUrl: 'http://localhost:3000' }
}

// Get full deployment info
const deployment = getDeployment()
console.log('Deployment type:', getDeploymentType())
```

The hook automatically initializes deployment detection on first use and caches the result globally. It provides both reactive state for React components and static functions for non-component contexts.
