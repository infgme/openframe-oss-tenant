A React hook that manages data for the Run Script page, fetching script details and filtering compatible devices based on supported platforms and status.

## Key Components

- **useRunScriptData**: Main hook that coordinates script and device data fetching
- **fetchScriptDetails**: Retrieves script information using tactical API client
- **fetchDevices**: Fetches devices filtered by script's supported platforms and active/offline status
- **searchDevices**: Enables device search functionality
- **State Management**: Separate loading, error, and data states for scripts and devices

## Usage Example

```typescript
import { useRunScriptData } from './use-run-script-data'

function RunScriptPage({ scriptId }: { scriptId: string }) {
  const {
    scriptDetails,
    isLoadingScript,
    scriptError,
    devices,
    isLoadingDevices,
    devicesError,
    filteredCount,
    searchDevices,
    refetchScript
  } = useRunScriptData({ scriptId })

  if (isLoadingScript) return <div>Loading script...</div>
  if (scriptError) return <div>Error: {scriptError}</div>

  return (
    <div>
      <h1>{scriptDetails?.name}</h1>
      <p>Compatible devices: {filteredCount}</p>
      <input 
        onChange={(e) => searchDevices(e.target.value)}
        placeholder="Search devices..."
      />
      {isLoadingDevices ? (
        <div>Loading devices...</div>
      ) : (
        devices.map(device => (
          <div key={device.id}>{device.hostname}</div>
        ))
      )}
    </div>
  )
}
```

The hook follows a two-stage loading pattern: first fetching script details, then using the script's supported platforms to filter and fetch compatible devices with ACTIVE/OFFLINE status only.
