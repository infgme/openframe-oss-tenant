Provides standardized tool type definitions and normalization utilities for consistent handling of MSP platform tool identifiers across the Flamingo ecosystem.

## Key Components

- **StandardToolKey**: Union type defining canonical tool identifiers (`TACTICAL`, `FLEET`, `MESHCENTRAL`, `OPENFRAME_CHAT`, `OPENFRAME_CLIENT`)
- **toolAliasToKey**: Mapping object that handles common naming variants and aliases for each tool
- **keyToLabel**: Human-readable display names for each standard tool key
- **keyToUiKitType**: Maps standard keys to UI Kit component types
- **normalizeToolKey()**: Converts various tool name formats to canonical StandardToolKey
- **toolKeyToLabel()**: Returns display-friendly label for a given tool key
- **toStandardToolLabel()**: Combines normalization and label conversion in one function
- **toUiKitToolType()**: Maps tool identifiers to UI Kit types with fallback handling

## Usage Example

```typescript
// Normalize various tool name formats
const tactical1 = normalizeToolKey('tactical-rmm'); // Returns 'TACTICAL'
const tactical2 = normalizeToolKey('TACTICALRMM'); // Returns 'TACTICAL'
const mesh = normalizeToolKey('mesh'); // Returns 'MESHCENTRAL'

// Get display labels
const label = toolKeyToLabel('TACTICAL'); // Returns 'Tactical'
const directLabel = toStandardToolLabel('fleet-mdm'); // Returns 'Fleet'

// Convert to UI Kit types
const uiType = toUiKitToolType('openframe-chat'); // Returns 'OPENFRAME_CHAT'
const systemType = toUiKitToolType('unknown-tool'); // Returns 'SYSTEM' (fallback)

// Handle undefined/null values safely
const safeLabel = toStandardToolLabel(undefined); // Returns ''
const safeKey = normalizeToolKey(''); // Returns undefined
```
