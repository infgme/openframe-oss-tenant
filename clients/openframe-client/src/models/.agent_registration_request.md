This Rust struct defines the data structure for agent registration requests in the Flamingo MSP platform, providing a standardized format for new agents to register with the OpenFrame system.

## Key Components

- **`AgentRegistrationRequest`** - Main struct representing the registration payload with camelCase JSON serialization
- **`hostname`** - The agent's system hostname identifier
- **`agent_version`** - Version string of the registering agent software
- **`organization_id`** - Optional organization identifier (omitted from JSON if empty)
- **`os_type`** - Operating system type classification

## Usage Example

```rust
use crate::AgentRegistrationRequest;

// Create a new registration request
let registration = AgentRegistrationRequest {
    hostname: "server-01.example.com".to_string(),
    agent_version: "1.2.3".to_string(),
    organization_id: "org-123".to_string(),
    os_type: "linux".to_string(),
};

// Serialize to JSON for API transmission
let json_payload = serde_json::to_string(&registration)?;
// Results in: {"hostname":"server-01.example.com","agentVersion":"1.2.3","organizationId":"org-123","osType":"linux"}

// Example with empty organization_id (will be omitted)
let minimal_registration = AgentRegistrationRequest {
    hostname: "workstation-05".to_string(),
    agent_version: "1.2.3".to_string(),
    organization_id: String::new(), // Will not appear in JSON
    os_type: "windows".to_string(),
};
```

The struct uses serde attributes to automatically handle JSON serialization with camelCase field names and conditional field inclusion based on the organization_id value.
