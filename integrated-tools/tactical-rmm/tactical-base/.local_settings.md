Django local development settings configuration file that extends custom settings and sets up a development environment with relaxed security for the TacticalRMM platform.

## Key Components

- **Environment Variables**: Uses template substitution for dynamic configuration (`${VARIABLE}` syntax)
- **Security Overrides**: Disables CSRF, session security, and SSL/TLS for local development
- **Database Configuration**: PostgreSQL connection with environment-based credentials
- **Service Integration**: Redis caching, NATS messaging, and MeshCentral management settings
- **Development Features**: Enables Swagger documentation and beta API endpoints
- **Directory Mappings**: Configures paths for executables, logs, and community scripts

## Usage Example

```python
# This file is typically used in Django's settings module chain
# Import in your main settings.py:
from .local_settings import *

# Environment variables should be set before Django startup:
# DJANGO_SEKRET=your-secret-key
# POSTGRES_DB=tacticalrmm
# POSTGRES_USER=postgres
# POSTGRES_PASSWORD=password
# POSTGRES_HOST=localhost
# POSTGRES_PORT=5432
# REDIS_HOST=localhost
# TACTICAL_DIR=/opt/tactical

# The settings automatically configure:
# - PostgreSQL database connection
# - Redis caching backend  
# - Disabled security features for development
# - CORS allowing all origins
# - MeshCentral integration (disabled)
```

**⚠️ Warning**: This configuration disables critical security features and should only be used in local development environments, never in production.
