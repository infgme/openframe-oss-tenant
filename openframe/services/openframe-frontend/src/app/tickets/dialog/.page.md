A Next.js page component that renders the detailed view for a specific dialog/ticket, with automatic redirection when no ID is provided.

## Key Components

- **DialogDetailsPage** - Main page component that handles dialog detail routing
- **AppLayout** - Wrapper component providing the application layout structure  
- **DialogDetailsView** - Component that renders the actual dialog details content
- **dynamic export** - Forces dynamic rendering to support search parameters

## Usage Example

This page is accessed via URL with search parameters:

```typescript
// URL: /dialogs?id=12345
// The page will:
// 1. Extract the 'id' from searchParams
// 2. Redirect to /tickets if no ID provided
// 3. Render DialogDetailsView with the dialog ID

// Component structure:
<AppLayout>
  <DialogDetailsView dialogId="12345" />
</AppLayout>
```

The page uses `force-dynamic` rendering to ensure `useSearchParams` works correctly in the `AppLayout` component. If no dialog ID is provided in the URL parameters, users are automatically redirected to the tickets overview page. This pattern ensures proper error handling for missing or invalid dialog references while maintaining a clean user experience.
