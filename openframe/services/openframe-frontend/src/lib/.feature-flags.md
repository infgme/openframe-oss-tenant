Centralized feature flag management system that controls feature availability across the application using runtime environment configuration.

## Key Components

- **`featureFlags`** - Main feature flags object containing all available features
- **`organizationImages`** - Controls organization image/logo display and upload functionality
- **`ssoAutoAllow`** - Controls SSO auto-allow domain feature
- **`FeatureFlagKey`** - TypeScript type for feature flag keys

## Usage Example

```typescript
import { featureFlags } from './feature-flags'

// Check if organization images are enabled
if (featureFlags.organizationImages.enabled()) {
  // Show image upload UI
  renderImageUpload()
}

// Check specific organization image capabilities
if (featureFlags.organizationImages.uploadEnabled()) {
  enableImageUpload()
}

if (featureFlags.organizationImages.displayEnabled()) {
  showOrganizationLogo()
}

// Check SSO auto-allow feature
if (featureFlags.ssoAutoAllow.enabled()) {
  enableDomainAutoApproval()
}

// Type-safe feature flag key usage
function checkFeature(key: FeatureFlagKey) {
  return featureFlags[key]
}
```

The feature flags are backed by runtime environment configuration, allowing for dynamic feature control without code deployments. Each feature group can have multiple related capabilities (like upload vs display for organization images).
