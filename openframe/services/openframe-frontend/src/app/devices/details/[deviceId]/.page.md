A Next.js client component that renders a device details page with dynamic routing support.

## Key Components

- **DeviceDetailsPage**: Default exported React component that handles device-specific route parameters
- **Dynamic rendering configuration**: Forces dynamic rendering to support search parameters in AppLayout
- **Route parameter handling**: Uses React's `use()` hook to resolve async params from Next.js 13+ app router

## Usage Example

This component is automatically rendered by Next.js when navigating to device detail routes:

```typescript
// Route: /devices/[deviceId]/page.tsx
// URL: /devices/abc123 -> deviceId = "abc123"

// The component structure:
<AppLayout>
  <DeviceDetailsView deviceId="abc123" />
</AppLayout>

// Example navigation to this page:
import { useRouter } from 'next/navigation'

const router = useRouter()
router.push(`/devices/${deviceId}`)
```

The page leverages Next.js 13+ app router conventions with async params, requiring the `use()` hook to resolve the Promise-wrapped parameters. The `force-dynamic` directive ensures proper SSR behavior when the AppLayout component uses `useSearchParams`.
