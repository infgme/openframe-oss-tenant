TypeScript environment configuration file that provides type declarations for Vite build tool and enables proper type checking for static asset imports in the project.

## Key Components

- **Vite Client Types Reference**: Imports Vite's built-in type definitions for development environment features
- **Static Asset Module Declarations**: Type definitions for importing image files (.png, .jpg, .jpeg, .gif, .svg) as string URLs

## Usage Example

With these declarations, you can safely import static assets in your TypeScript files:

```typescript
// Import images with proper type checking
import logoImage from './assets/logo.png'
import heroBackground from './assets/hero.jpg'
import iconSvg from './assets/icon.svg'

// TypeScript knows these are strings (file URLs)
function MyComponent() {
  return (
    <div>
      <img src={logoImage} alt="Logo" />
      <div style={{ backgroundImage: `url(${heroBackground})` }}>
        <img src={iconSvg} alt="Icon" />
      </div>
    </div>
  )
}
```

This configuration ensures TypeScript understands that image imports resolve to string URLs (the processed file paths that Vite generates), preventing type errors during development and build processes.
