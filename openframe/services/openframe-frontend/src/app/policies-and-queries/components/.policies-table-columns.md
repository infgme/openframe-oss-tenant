This file provides table configuration for displaying policy data in the Flamingo platform's security compliance management interface, defining column structure and row actions for policy tables.

## Key Components

- **`getPolicyTableRowActions()`** - Factory function that returns an array of row actions (More menu and Details button) with Flamingo's design system styling
- **`getPolicyTableColumns()`** - Factory function that returns table column definitions for policy data display including name, status, category, and enabled state
- **`Policy`** - TypeScript interface imported from types module that defines the policy data structure

## Usage Example

```typescript
import { getPolicyTableColumns, getPolicyTableRowActions } from './policies-table-columns'
import { Table } from "@flamingo-stack/openframe-frontend-core/components/ui"

function PoliciesTable({ policies }: { policies: Policy[] }) {
  const handleMoreActions = (policy: Policy) => {
    // Open actions menu
  }
  
  const handleViewDetails = (policy: Policy) => {
    // Navigate to policy details
  }

  const columns = getPolicyTableColumns()
  const rowActions = getPolicyTableRowActions(handleMoreActions, handleViewDetails)

  return (
    <Table
      data={policies}
      columns={columns}
      rowActions={rowActions}
    />
  )
}
```

The configuration includes responsive column widths, custom cell renderers with Flamingo's typography tokens (DM Sans, Azeret Mono), and design system color classes for consistent styling across the platform.
