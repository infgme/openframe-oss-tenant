A centralized configuration file that defines the tab navigation structure and components for device detail views in the OpenFrame interface.

## Key Components

- **DEVICE_TABS**: Array of `TabItem` objects defining all available device tabs with their icons, labels, and associated components
- **getDeviceTab()**: Utility function to retrieve a specific tab configuration by ID
- **getTabComponent()**: Function to get the React component associated with a specific tab ID

## Usage Example

```typescript
import { DEVICE_TABS, getDeviceTab, getTabComponent } from './device-tabs'

// Access all available tabs
const allTabs = DEVICE_TABS

// Get a specific tab configuration
const securityTab = getDeviceTab('security')
console.log(securityTab?.label) // "Security"
console.log(securityTab?.hasAlert) // false

// Get the component for rendering a specific tab
const HardwareComponent = getTabComponent('hardware')
if (HardwareComponent) {
  return <HardwareComponent device={deviceData} />
}

// Use in tab navigation
const TabNavigation = () => (
  <div>
    {DEVICE_TABS.map(tab => (
      <button key={tab.id}>
        <tab.icon />
        {tab.label}
        {tab.hasAlert && <AlertBadge type={tab.alertType} />}
      </button>
    ))}
  </div>
)
```

The configuration supports alert indicators for tabs that need attention, with customizable alert types ('error', 'warning') for different severity levels.
