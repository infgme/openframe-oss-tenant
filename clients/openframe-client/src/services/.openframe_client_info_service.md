Service for managing OpenFrame client information including version tracking, update status, and binary path configuration. Handles persistent storage of client metadata in JSON format.

## Key Components

- **OpenFrameClientInfoService**: Main service struct that manages client information persistence
- **new()**: Constructor that initializes the service with a secured directory path
- **get()**: Retrieves client info from JSON file, returns default if file doesn't exist
- **save()**: Persists client info to JSON file with pretty formatting
- **update_version()**: Updates the current version and timestamp
- **set_update_status()**: Sets client update status and target version
- **set_binary_path()**: Updates the binary path configuration

## Usage Example

```rust
use openframe_client_info_service::OpenFrameClientInfoService;
use crate::platform::directories::DirectoryManager;

// Initialize the service
let directory_manager = DirectoryManager::new()?;
let client_service = OpenFrameClientInfoService::new(directory_manager)?;

// Get current client info
let info = client_service.get().await?;
println!("Current version: {}", info.current_version);

// Update version
client_service.update_version("1.2.3".to_string()).await?;

// Set update status
use crate::models::openframe_client_info::ClientUpdateStatus;
client_service.set_update_status(
    ClientUpdateStatus::Updating, 
    Some("1.3.0".to_string())
).await?;

// Configure binary path
client_service.set_binary_path("/usr/bin/openframe".to_string()).await?;
```

The service provides atomic operations for client metadata management with proper error handling and logging throughout the update lifecycle.
