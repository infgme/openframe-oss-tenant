A client-side route guard component that restricts access to certain routes based on the current application mode, providing a fallback UI for unauthorized access attempts.

## Key Components

- **RouteGuard**: Main component that wraps child components and controls access
- **RouteGuardProps**: Interface defining the component props (children)
- **isRouteAllowedInCurrentMode**: Imported utility function that determines route permissions
- **Access denied UI**: Fallback component with branded styling for restricted routes

## Usage Example

```typescript
import { RouteGuard } from './components/route-guard'

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <RouteGuard>
      {children}
    </RouteGuard>
  )
}

// Or wrap specific pages
export default function AdminPage() {
  return (
    <RouteGuard>
      <div>Admin content here...</div>
    </RouteGuard>
  )
}
```

The component uses Next.js's `usePathname` hook to get the current route and checks it against allowed routes for the current app mode. If access is denied, it displays a centered error message with ODS (likely OpenFrame Design System) styling classes. The guard is particularly useful for static exports where server-side route protection isn't available.
