HTTP client for retrieving tool assets from the Flamingo tool API with authentication handling.

## Key Components

- **ToolApiClient** - Main client struct containing HTTP client, base URL, and configuration service
- **new()** - Constructor that initializes the client with required dependencies
- **get_tool_asset()** - Async method to fetch tool assets using authenticated requests

## Usage Example

```rust
use reqwest::Client;
use crate::tool_api_client::ToolApiClient;
use crate::services::agent_configuration_service::AgentConfigurationService;

// Initialize the client
let http_client = Client::new();
let base_url = "https://api.flamingo.run".to_string();
let config_service = AgentConfigurationService::new();

let tool_client = ToolApiClient::new(http_client, base_url, config_service);

// Fetch a tool asset
let asset_bytes = tool_client
    .get_tool_asset("tool123".to_string(), "/assets/config.json".to_string())
    .await?;

// Use the asset data
let asset_content = String::from_utf8(asset_bytes.to_vec())?;
```

The client automatically handles Bearer token authentication by retrieving access tokens from the configuration service. It constructs URLs in the format `{base_url}/tools/agent/{tool_id}{asset_path}` and returns raw bytes for flexibility in handling different asset types. Error handling includes both HTTP failures and authentication issues with detailed context.
