A React hook that fetches and manages available SSO providers for user invitations, providing loading states and error handling for invitation-based authentication flows.

## Key Components

- **`useInviteProviders`** - Main hook that accepts an invitation ID and returns provider data, loading state, and error information
- **`SSOProvider`** - Interface defining the structure of SSO provider objects with provider name and enabled status
- **`InviteProvidersResponse`** - Type definition for the API response containing available provider names

## Usage Example

```typescript
import { useInviteProviders } from './use-invite-providers'

function InvitationForm({ inviteId }: { inviteId: string }) {
  const { providers, loading, error } = useInviteProviders(inviteId)

  if (loading) return <div>Loading providers...</div>
  if (error) return <div>Error: {error}</div>

  return (
    <div>
      <h3>Available Sign-in Options</h3>
      {providers.map(provider => (
        <button key={provider.provider} disabled={!provider.enabled}>
          Sign in with {provider.provider}
        </button>
      ))}
    </div>
  )
}

// Handle case with no invitation
function PublicSignIn() {
  const { providers } = useInviteProviders(null) // Returns empty array
  // Component logic...
}
```

The hook automatically refetches when the `invitationId` changes and gracefully handles cases where no invitation is provided by returning an empty providers array.
