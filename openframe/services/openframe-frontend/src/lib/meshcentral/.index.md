This file serves as the main entry point for the MeshCentral integration library, exporting all core functionality for remote device management and file operations through MeshCentral's WebSocket API.

## Key Components

### Core Exports
- **WebSocketManager** - Manages WebSocket connections to MeshCentral server
- **MeshControlClient** - Main client for device control operations
- **MeshCentralFileManager** - Handles file system operations on remote devices

### File Management Suite
- **FileOperations** - Core file system operations (create, delete, rename, etc.)
- **FileUploader/FileDownloader** - Handle file transfer operations with progress tracking
- **FileBinaryProtocol** - Manages binary data exchange for file transfers
- **FileErrorHandler** - Centralized error handling for file operations

### Configuration & Utilities
- **buildWsUrl, MESH_USER, MESH_PASS** - Connection configuration helpers
- **canAccessFiles()** - Permission validation utility

## Usage Example

```typescript
import { 
  MeshCentralFileManager, 
  canAccessFiles, 
  buildWsUrl 
} from './meshcentral-integration'

// Check user permissions
const userRights = { mesh: 0x00000020, site: 0x00000008 }
if (canAccessFiles(userRights)) {
  // Initialize file manager
  const fileManager = new MeshCentralFileManager({
    wsUrl: buildWsUrl('wss://meshcentral.example.com'),
    deviceId: 'device123'
  })
  
  // Perform file operations
  const files = await fileManager.listDirectory('/home/user')
  console.log('Available files:', files)
}
```

The library provides a complete solution for integrating MeshCentral's remote access capabilities into applications, with robust file management, permission checking, and error handling.
