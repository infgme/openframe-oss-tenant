This file provides column configuration for dialog tables in the Flamingo MSP platform, defining how dialog data is displayed across different screen sizes with responsive design and filtering capabilities.

## Key Components

- **`getDialogTableColumns()`** - Main factory function that returns configured table columns for dialog data
- **`DialogTableColumnsOptions`** - Configuration interface for customizing column behavior
- **Column Definitions** - Four table columns: Title, Source, Created Date, and Status
- **Status Mapping** - Logic for converting dialog status values to visual variants

## Usage Example

```typescript
import { getDialogTableColumns } from './dialog-table-columns'

// Basic usage
const columns = getDialogTableColumns()

// With organization lookup and archive mode
const archivedColumns = getDialogTableColumns({
  organizationLookup: {
    'org-123': 'Acme Corp',
    'org-456': 'Tech Solutions'
  },
  isArchived: true
})

// Use with table component
<Table
  columns={columns}
  data={dialogs}
  // ... other table props
/>
```

The configuration supports responsive breakpoints (`sm`, `lg`), conditional filtering based on archive status, and rich cell rendering including device cards, timestamps, and status tags with appropriate color variants (success, warning, error, info).
