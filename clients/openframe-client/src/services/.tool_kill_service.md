A service for gracefully stopping and force-killing tool and asset processes by searching for processes matching specific command patterns and terminating them with retry logic and verification.

## Key Components

- **ToolKillService**: Main service struct that handles process termination operations
- **stop_tool()**: Stops all processes matching a tool's command pattern
- **stop_asset()**: Stops processes for specific asset within a tool
- **stop_processes_by_pattern()**: Generic method to find and terminate processes by command pattern
- **stop_process_with_retry()**: Handles individual process termination with retry logic
- **try_graceful_stop()** / **try_force_kill()**: Implements graceful then forceful termination strategies
- **wait_for_process_exit()**: Monitors process termination with timeout
- **build_tool_cmd_pattern()** / **build_asset_cmd_pattern()**: Creates OS-specific command patterns

## Configuration Constants

- **GRACEFUL_SHUTDOWN_TIMEOUT_SECS**: 5 seconds for graceful termination
- **FORCE_KILL_TIMEOUT_SECS**: 3 seconds for force kill verification  
- **MAX_KILL_RETRIES**: 3 attempts for force kill
- **PROCESS_CHECK_INTERVAL_MS**: 500ms intervals for process exit monitoring

## Usage Example

```rust
use tool_kill_service::ToolKillService;

// Create service instance
let kill_service = ToolKillService::new();

// Stop all processes for a tool
kill_service.stop_tool("my_tool").await?;

// Stop specific asset process
kill_service.stop_asset("asset_123", "my_tool").await?;
```

The service first attempts graceful termination (SIGTERM), waits for the process to exit, then falls back to force kill (SIGKILL) with retries if needed. It builds OS-specific command patterns to identify target processes and provides comprehensive logging throughout the termination process.
