A state management module for tracking software update progress through different phases of the update process.

## Key Components

- **`UpdatePhase`** - Enum defining the sequential stages of an update: validation, download, extraction, updater preparation, launch, and completion
- **`UpdateState`** - Structure holding the target version and current phase of an ongoing update
- **`new()`** - Constructor that initializes a new update state with the target version and sets the initial phase to validating
- **`set_phase()`** - Method to transition the update state to a different phase

## Usage Example

```rust
use update_state::{UpdateState, UpdatePhase};

// Initialize a new update process
let mut update = UpdateState::new("v2.1.0".to_string());
println!("Starting update to {}", update.target_version);

// Progress through update phases
update.set_phase(UpdatePhase::Downloading);
println!("Current phase: {:?}", update.phase);

update.set_phase(UpdatePhase::Extracting);
update.set_phase(UpdatePhase::Completed);

// Check if update is finished
if update.phase == UpdatePhase::Completed {
    println!("Update to {} completed successfully!", update.target_version);
}
```

The module supports serialization/deserialization via serde, making it suitable for persisting update state across application restarts or communicating update progress to external systems.
