Main entry point for the OpenFrame Chat Tauri application that provides token management, system tray functionality, and configuration through command-line arguments.

## Key Components

- **`run()`** - Main application entry point that configures and starts the Tauri app
- **State Management Structs**:
  - `ServerUrlState` - Manages server URL configuration
  - `DebugModeState` - Tracks debug mode setting
- **Tauri Commands**:
  - `greet()` - Simple greeting function for testing
  - `get_token()` - Retrieves current authentication token
  - `get_server_url()` - Returns configured server URL
  - `get_debug_mode()` - Returns debug mode status
- **System Tray** - Provides minimize-to-tray functionality with show/quit menu
- **Token Watcher Integration** - Monitors and manages authentication tokens

## Usage Example

The application is launched with command-line arguments:

```rust
// Command line usage:
// ./openframe-chat --openframe-token-path "/path/to/token" --openframe-secret "secret_key" --serverUrl "https://api.flamingo.com" --devMode

// From frontend (JavaScript), call the exposed commands:
// invoke('get_token') -> Returns current auth token
// invoke('get_server_url') -> Returns configured server URL  
// invoke('get_debug_mode') -> Returns debug mode boolean
// invoke('greet', { name: 'User' }) -> Returns greeting string
```

The app runs in the system tray and prevents normal window closing, instead hiding the window when the user attempts to close it. The tray icon provides access to show the window or quit the application entirely.
