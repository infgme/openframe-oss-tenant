Provides utilities for forcefully logging out users and managing authentication tokens in the browser environment.

## Key Components

- **`forceLogout(options?)`** - Main function that clears all authentication data and optionally redirects the user
- **`hasStoredTokens()`** - Utility to check if authentication tokens exist in localStorage
- **`clearStoredTokens()`** - Safely removes authentication tokens from localStorage
- **`ForceLogoutOptions`** - Interface defining configuration options for logout behavior

## Usage Example

```typescript
import { forceLogout, hasStoredTokens, clearStoredTokens } from './force-logout'

// Force logout with default behavior (redirect to auth page)
await forceLogout()

// Force logout with custom options
await forceLogout({
  reason: 'session expired',
  shouldRedirect: false,
  redirectPath: '/login'
})

// Check if user has stored authentication tokens
if (hasStoredTokens()) {
  console.log('User has stored tokens')
}

// Clear tokens without full logout process
clearStoredTokens()
```

The `forceLogout` function performs a complete authentication cleanup by removing tokens from localStorage, clearing the auth store state, and handling redirects based on the current app mode (SaaS tenant vs standard). It includes error handling for each cleanup step and respects SSR environments by checking for `window` availability.
