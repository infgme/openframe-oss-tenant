A React hook for managing policy data fetching, searching, and state management with integrated error handling and toast notifications.

## Key Components

- **`usePolicies`** - Main hook that manages policy state and operations
- **`fetchPolicies`** - Core function for retrieving policies from the Fleet API with optional search parameters
- **`searchPolicies`** - Wrapper function for performing policy searches with term updates
- **`refreshPolicies`** - Function to reload policies with success/error feedback via toasts

## State Management

The hook maintains several state variables:
- `policies` - Array of policy objects
- `isLoading` - Loading state for API operations
- `error` - Error message string for failed requests
- `searchTerm` - Current search query

## Usage Example

```typescript
import { usePolicies } from './hooks/use-policies'

function PoliciesComponent() {
  const { 
    policies, 
    isLoading, 
    error, 
    searchPolicies, 
    refreshPolicies 
  } = usePolicies()

  const handleSearch = async (term: string) => {
    try {
      await searchPolicies(term)
    } catch (error) {
      // Error handling is done automatically by the hook
    }
  }

  const handleRefresh = async () => {
    await refreshPolicies()
  }

  if (isLoading) return <div>Loading policies...</div>
  if (error) return <div>Error: {error}</div>

  return (
    <div>
      {policies.map(policy => (
        <div key={policy.id}>{policy.name}</div>
      ))}
    </div>
  )
}
```

The hook automatically fetches policies on mount and provides comprehensive error handling with user-friendly toast notifications for both success and failure scenarios.
