A modal component for inviting multiple users to join the system with email validation and role selection. Provides a dynamic form interface where users can add/remove invitation rows and send bulk invitations.

## Key Components

- **AddUsersModal** - Main modal component with form state management
- **InviteRow** - Type definition for email/role pairs
- **Dynamic row management** - Add/remove invitation rows with validation
- **Email validation** - Real-time regex validation with visual feedback
- **Role selection** - Dropdown for assigning user roles (currently Admin only)
- **Submission handling** - Async invite processing with toast notifications

## Usage Example

```typescript
import { AddUsersModal } from './add-users-modal'

function UserManagement() {
  const [isModalOpen, setIsModalOpen] = useState(false)

  const handleInviteUsers = async (rows: InviteRow[]) => {
    // Send invitations to backend
    const response = await fetch('/api/users/invite', {
      method: 'POST',
      body: JSON.stringify({ invitations: rows })
    })
    
    if (!response.ok) {
      throw new Error('Failed to send invitations')
    }
  }

  const handleInvitesCompleted = async () => {
    // Refresh user list or perform other actions
    console.log('Invitations sent successfully')
  }

  return (
    <>
      <Button onClick={() => setIsModalOpen(true)}>
        Add Users
      </Button>
      
      <AddUsersModal
        isOpen={isModalOpen}
        onClose={() => setIsModalOpen(false)}
        invite={handleInviteUsers}
        onInvited={handleInvitesCompleted}
      />
    </>
  )
}
```

The component handles form validation, prevents submission of invalid emails, and provides user feedback through toast notifications. The modal automatically resets its state when closed and supports responsive design with mobile-friendly layouts.
