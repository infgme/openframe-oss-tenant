A data transfer object for filtering device queries with multiple criteria including status, type, OS, organization, and tags.

## Key Components

- **DeviceFilterInput** - Main DTO class with Lombok annotations for automatic getters/setters
- **statuses** - List of `DeviceStatus` enums to filter by device status
- **deviceTypes** - List of `DeviceType` enums to filter by device category
- **osTypes** - List of operating system type strings for OS-based filtering
- **organizationIds** - List of organization ID strings for multi-tenant filtering
- **tagNames** - List of tag name strings for tag-based filtering

## Usage Example

```java
// Build a filter for active Windows/Mac devices in specific organizations
DeviceFilterInput filter = DeviceFilterInput.builder()
    .statuses(List.of(DeviceStatus.ACTIVE, DeviceStatus.ONLINE))
    .deviceTypes(List.of(DeviceType.DESKTOP, DeviceType.LAPTOP))
    .osTypes(List.of("Windows", "macOS"))
    .organizationIds(List.of("org-123", "org-456"))
    .tagNames(List.of("production", "managed"))
    .build();

// Use in device service query
List<Device> filteredDevices = deviceService.findDevices(filter);
```

The class uses builder pattern for flexible filter construction and supports multiple values per criteria for inclusive filtering.
