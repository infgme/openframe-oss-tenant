HTTP client for handling OAuth authentication flows in the Flamingo platform. Supports both client credentials and refresh token authentication methods.

## Key Components

- **`AuthClient`** - Main authentication client struct that manages HTTP requests to OAuth endpoints
- **`new()`** - Constructor that initializes the client with a base URL and HTTP client
- **`authenticate_with_secret()`** - Performs client credentials OAuth flow using client ID and secret
- **`authenticate_with_refresh_token()`** - Refreshes access tokens using a refresh token

## Usage Example

```rust
use reqwest::Client;
use crate::auth_client::AuthClient;

// Initialize the auth client
let http_client = Client::new();
let auth_client = AuthClient::new(
    "https://api.flamingo.run".to_string(),
    http_client
);

// Authenticate with client credentials
let token_response = auth_client
    .authenticate_with_secret(
        "your_client_id".to_string(),
        "your_client_secret".to_string()
    )
    .await?;

// Later, refresh the token when needed
let refreshed_token = auth_client
    .authenticate_with_refresh_token(
        token_response.refresh_token.unwrap()
    )
    .await?;
```

Both authentication methods return an `AgentTokenResponse` containing the access token, refresh token, and expiration details. The client handles proper form encoding and error responses with detailed context for debugging.
