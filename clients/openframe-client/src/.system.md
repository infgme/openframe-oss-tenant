System monitoring module that collects and provides real-time hardware metrics including CPU usage, memory consumption, and disk space information for MSP platform monitoring.

## Key Components

- **`SystemMetrics`** - Serializable struct containing timestamped system metrics (CPU, memory, disk usage)
- **`SystemInfo`** - Thread-safe system information collector with mutex-protected `sysinfo::System` instance
- **`SystemInfo::new()`** - Creates a new system monitor with CPU and memory refresh capabilities
- **`SystemInfo::collect_metrics()`** - Gathers current system metrics and returns timestamped data

## Usage Example

```rust
use anyhow::Result;

fn main() -> Result<()> {
    // Initialize system monitor
    let system_info = SystemInfo::new()?;
    
    // Collect current metrics
    let metrics = system_info.collect_metrics()?;
    
    println!("CPU Usage: {:.2}%", metrics.cpu_usage);
    println!("Memory: {} / {} bytes", metrics.memory_used, metrics.memory_total);
    println!("Disk: {} / {} bytes", metrics.disk_used, metrics.disk_total);
    println!("Timestamp: {}", metrics.timestamp);
    
    Ok(())
}

// For continuous monitoring
fn monitor_system() -> Result<()> {
    let system_info = SystemInfo::new()?;
    
    loop {
        let metrics = system_info.collect_metrics()?;
        // Send metrics to monitoring dashboard or store in database
        std::thread::sleep(std::time::Duration::from_secs(30));
    }
}
```

The module uses `sysinfo` for cross-platform system information gathering and `sys_info` for disk space metrics, providing comprehensive system monitoring capabilities for the Flamingo MSP platform.
