GraphQL query definitions for retrieving log data, filters, and detailed log information in the Flamingo platform.

## Key Components

- **GET_LOG_FILTERS_QUERY** - Retrieves available filter options for logs (tool types, event types, severities, organizations)
- **GET_LOGS_QUERY** - Fetches paginated log entries with filtering, searching, and cursor-based pagination
- **GET_LOG_DETAILS_QUERY** - Retrieves detailed information for a specific log entry by its composite key

## Usage Example

```typescript
import { GET_LOGS_QUERY, GET_LOG_DETAILS_QUERY } from './logs-queries'

// Fetch logs with filtering and pagination
const logsResult = await client.query({
  query: GET_LOGS_QUERY,
  variables: {
    filter: {
      severity: 'ERROR',
      toolType: 'MONITORING'
    },
    pagination: {
      first: 20,
      after: 'cursor_token'
    },
    search: 'connection timeout'
  }
})

// Get detailed information for a specific log
const logDetails = await client.query({
  query: GET_LOG_DETAILS_QUERY,
  variables: {
    logId: 'evt_123',
    ingestDay: '2024-01-15',
    toolType: 'MONITORING',
    eventType: 'ALERT',
    timestamp: '2024-01-15T10:30:00Z'
  }
})
```

The queries support comprehensive log management with filtering by organization, tool type, severity levels, and full-text search capabilities. Log details include metadata like device information, timestamps, and structured event data.
