URL parameter resolver that processes tool URLs by substituting server URL placeholders with actual configuration values.

## Key Components

- **`ToolUrlParamsResolver`** - Main resolver struct that handles URL parameter substitution
- **`process()`** - Core method that replaces placeholder patterns with actual server URL values
- **`SERVER_URL_PLACEHOLDER`** - Constant defining the placeholder pattern (`${client.serverUrl}`)
- **`InitialConfigurationService`** - Dependency for retrieving server configuration

## Usage Example

```rust
use crate::services::InitialConfigurationService;
use crate::tool_url_params_resolver::ToolUrlParamsResolver;

// Create resolver with configuration service
let config_service = InitialConfigurationService::new();
let resolver = ToolUrlParamsResolver::new(config_service);

// Process URL with placeholder
let template_url = "https://${client.serverUrl}/api/tools/status";
let resolved_url = resolver.process(template_url)?;
// Result: "https://production.flamingo.com/api/tools/status"

// URLs without placeholders pass through unchanged
let static_url = "https://external-api.com/endpoint";
let unchanged = resolver.process(static_url)?;
// Result: "https://external-api.com/endpoint"
```

The resolver enables dynamic URL configuration for tools by replacing template placeholders with runtime server values, supporting flexible deployment environments while maintaining clean tool definitions.
