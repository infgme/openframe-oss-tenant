A utility module that provides type-safe access to runtime environment variables for the Flamingo/OpenFrame platform, with fallback mechanisms for different execution contexts.

## Key Components

**`getEnvVar(key: string)`** - Internal helper function that safely retrieves environment variables with multiple fallback strategies, handling Next.js runtime, browser window context, and Node.js process environments.

**`runtimeEnv`** - Main configuration object exposing typed methods for accessing specific environment variables:
- `tenantHostUrl()` - Multi-tenant host URL
- `sharedHostUrl()` - Shared services host URL  
- `gtmContainerId()` - Google Tag Manager container ID
- `appMode()` - Application mode (defaults to 'oss-tenant')
- `appType()` - Application type (defaults to 'openframe-dashboard')
- `appUrl()` - Production app URL
- `devUrl()` - Development server URL
- `enableDevTicketObserver()` - Development ticket observer toggle
- `authCheckIntervalMs()` - Authentication check interval in milliseconds
- `authLoginUrl()` - Authentication login URL
- `featureOrganizationImages()` - Organization images feature flag
- `featureSSOAllowDomain()` - SSO domain allowlist feature flag

## Usage Example

```typescript
import { runtimeEnv } from './runtime-config'

// Get configuration values with proper typing
const tenantUrl = runtimeEnv.tenantHostUrl()
const isDevMode = runtimeEnv.appMode() === 'development'
const authInterval = runtimeEnv.authCheckIntervalMs() // Returns number
const gtmId = runtimeEnv.gtmContainerId() // Returns string | undefined

// Use in application logic
if (runtimeEnv.enableDevTicketObserver()) {
  console.log('Development ticket observer enabled')
}

// Configure authentication
const loginUrl = runtimeEnv.authLoginUrl()
```
