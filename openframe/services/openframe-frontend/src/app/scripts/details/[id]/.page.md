A Next.js client-side page component that renders detailed information for a specific script using dynamic routing. This component extracts the script ID from URL parameters and displays the script details within the application layout.

## Key Components

- **ScriptDetailsPage**: Default exported React component that serves as the main page component
- **useParams hook**: Next.js navigation hook for accessing dynamic route parameters
- **AppLayout**: Wrapper component providing consistent application structure
- **ScriptDetailsView**: Child component responsible for rendering script-specific details

## Usage Example

This page component is typically used in a Next.js app router structure:

```typescript
// File structure: app/scripts/[id]/page.tsx
// Accessible via routes like: /scripts/abc123, /scripts/def456

// The component automatically receives the script ID from the URL
// Example URLs:
// - /scripts/my-script-123 → id = "my-script-123"
// - /scripts/automation-script → id = "automation-script"

// Usage in routing:
// User navigates to /scripts/some-id
// → ScriptDetailsPage renders
// → Extracts "some-id" from params
// → Passes it to ScriptDetailsView component
```

The component uses Next.js 13+ app router conventions with the `'use client'` directive, indicating it requires client-side rendering for interactivity. The ID parameter is safely extracted with type checking to ensure it's a string before passing to the child component.
