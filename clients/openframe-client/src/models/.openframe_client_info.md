Defines data structures for tracking OpenFrame client version information and update status within the Flamingo MSP platform. This module provides serializable types for monitoring client updates and maintaining version state.

## Key Components

- **`ClientUpdateStatus`** - Enum representing the four possible update states (Current, Updating, Updated, Failed)
- **`OpenFrameClientInfo`** - Main struct containing client version data, binary path, and update timestamps
- Both types implement `Default`, `Clone`, `Serialize`, and `Deserialize` traits for easy data handling

## Usage Example

```rust
use openframe_client_info::{OpenFrameClientInfo, ClientUpdateStatus};

// Create client info with current status
let client_info = OpenFrameClientInfo {
    current_version: "1.2.3".to_string(),
    target_version: Some("1.2.4".to_string()),
    status: ClientUpdateStatus::Updating,
    binary_path: "/usr/local/bin/openframe".to_string(),
    last_update_check: Some("2024-01-15T10:30:00Z".to_string()),
    last_updated: Some("2024-01-14T08:15:00Z".to_string()),
};

// Serialize to JSON for API responses
let json = serde_json::to_string(&client_info)?;

// Check if client needs updating
match client_info.status {
    ClientUpdateStatus::Failed => println!("Update failed for client"),
    ClientUpdateStatus::Current => println!("Client is up to date"),
    _ => println!("Update in progress"),
}
```

The `ClientUpdateStatus` enum uses lowercase serialization for consistent API responses, while the main struct tracks comprehensive client state including version numbers, file paths, and timestamps for update operations.
