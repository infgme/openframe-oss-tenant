A React hook for archiving resolved dialogs with batch processing, error handling, and user feedback via toast notifications.

## Key Components

- **`archiveResolvedDialogs`** - Main function that filters resolved dialogs and archives them via API calls
- **`isArchiving`** - Boolean state indicating if archive operation is in progress
- **Toast Integration** - Uses Flamingo's toast system for user feedback on success/failure
- **Error Handling** - Comprehensive error handling with partial success scenarios

## Usage Example

```typescript
import { useArchiveResolved } from './use-archive-resolved'
import { Dialog } from '../types/dialog.types'

function DialogManager() {
  const { archiveResolvedDialogs, isArchiving } = useArchiveResolved()

  const handleArchiveClick = async () => {
    const dialogs: Dialog[] = [
      { id: '1', status: 'RESOLVED' },
      { id: '2', status: 'OPEN' },
      { id: '3', status: 'RESOLVED' }
    ]
    
    const success = await archiveResolvedDialogs(dialogs)
    if (success) {
      // Refresh dialog list or update UI
      console.log('Archive operation completed')
    }
  }

  return (
    <button 
      onClick={handleArchiveClick}
      disabled={isArchiving}
    >
      {isArchiving ? 'Archiving...' : 'Archive Resolved Dialogs'}
    </button>
  )
}
```

The hook automatically filters for dialogs with `RESOLVED` status, batches API calls using `Promise.allSettled`, and provides detailed feedback about partial successes or failures through toast notifications.
