A React hook for fetching comprehensive device details from multiple API sources (GraphQL, Tactical RMM, and Fleet) and merging them into a unified Device object.

## Key Components

- **`useDeviceDetails`** - Main hook providing device fetching and state management
- **`createDevice`** - Helper function that merges data from multiple API sources into a unified Device object
- **State Management** - Handles loading states, error handling, and automatic data polling
- **Multi-source Data Fetching** - Combines data from GraphQL API, Tactical RMM, and Fleet APIs

## Usage Example

```typescript
import { useDeviceDetails } from './hooks/use-device-details'

function DeviceDetailsPage({ machineId }: { machineId: string }) {
  const { 
    deviceDetails, 
    isLoading, 
    error, 
    fetchDeviceById, 
    startPolling, 
    stopPolling 
  } = useDeviceDetails()

  useEffect(() => {
    if (machineId) {
      fetchDeviceById(machineId)
      startPolling(machineId, 30000) // Poll every 30 seconds
    }
    
    return () => stopPolling()
  }, [machineId])

  if (isLoading) return <div>Loading device details...</div>
  if (error) return <div>Error: {error}</div>
  if (!deviceDetails) return <div>Device not found</div>

  return (
    <div>
      <h1>{deviceDetails.displayName}</h1>
      <p>Status: {deviceDetails.status}</p>
      <p>OS: {deviceDetails.operating_system}</p>
      <p>Last Seen: {deviceDetails.lastSeen}</p>
    </div>
  )
}
```

The hook automatically handles data normalization, IP address filtering, and provides both silent updates and user-facing loading states. It merges hardware details, software inventory, user information, and MDM status from multiple monitoring platforms into a single comprehensive device object.
