Configuration types for AI policy management and editing state tracking in the OpenFrame platform.

## Key Components

### CustomPolicyState
Interface for managing custom AI policy configurations with the following properties:
- `enabled` - Whether custom policies are active
- `baseTemplateId` - Reference to the base policy template
- `originalRules` - Map of original approval level rules
- `changes` - Map tracking rule modifications
- `existingOverrides` - Record of current rule overrides
- `baseTemplateForDisplay` - Full template details for UI display

### EditSnapshot
Interface for capturing AI model editing session state:
- `provider` - AI service provider identifier
- `model` - Specific model name/version
- `templateId` - Associated policy template ID
- `policyGroups` - Grouped permission categories by classification
- `customBaseTemplateId` - Custom template reference for specialized policies

## Usage Example

```typescript
import type { CustomPolicyState, EditSnapshot } from './ai-settings'

// Initialize custom policy state
const policyState: CustomPolicyState = {
  enabled: true,
  baseTemplateId: 'template-123',
  originalRules: new Map([['action.create', 'ADMIN']]),
  changes: new Map([['action.create', 'MANAGER']]),
  existingOverrides: { 'action.delete': 'ADMIN' },
  baseTemplateForDisplay: null
}

// Create editing snapshot
const snapshot: EditSnapshot = {
  provider: 'openai',
  model: 'gpt-4',
  templateId: 'template-456',
  policyGroups: new Map([['security', []]]),
  customBaseTemplateId: 'custom-789'
}
```
