A Next.js client-side authentication page component that dynamically loads the main auth page with a loading state while preventing server-side rendering.

## Key Components

- **Auth()** - Default exported page component that renders the dynamically imported AuthPage
- **AuthPage** - Dynamically imported auth page component from `@app/auth/pages/auth-page`
- **AuthLoading** - Loading component displayed during dynamic import
- **dynamic export** - Next.js configuration forcing dynamic rendering

## Usage Example

This page component is typically used as a route page in Next.js App Router:

```typescript
// app/auth/page.tsx
'use client'

import nextDynamic from 'next/dynamic'
import AuthLoading from './loading'

export const dynamic = 'force-dynamic'

const AuthPage = nextDynamic(
  () => import('@app/auth/pages/auth-page'),
  { ssr: false, loading: () => <AuthLoading /> }
)

export default function Auth() {
  return <AuthPage />
}
```

The component uses `nextDynamic` with `ssr: false` to ensure the auth page only renders on the client-side, which is common for authentication flows that require browser APIs or sensitive client-side logic. The `force-dynamic` export tells Next.js to always render this page dynamically rather than statically.
