A comprehensive test suite for validating OpenFrame authentication token lifecycle operations including refresh and logout functionality.

## Key Components

- **`testRefreshTokens()`** - Validates token refresh with tenant ID, ensuring new cookies differ from old ones
- **`testRefreshTokensWithoutTenantId()`** - Tests token refresh without specifying tenant ID
- **`testLogout()`** - Verifies logout clears authentication cookies with tenant ID
- **`testLogoutWithoutTenantId()`** - Tests logout functionality without tenant ID
- **UnauthorizedTest** - Base test class providing authentication test utilities
- **AuthApi/AuthFlow** - API utilities for authentication operations

## Usage Example

```java
// Example test execution flow
@Test
public void exampleTokenLifecycle() {
    // 1. Get test user and login
    User user = getUser();
    Map<String, String> cookies = login(user);
    
    // 2. Refresh tokens
    Map<String, String> refreshedCookies = refresh(user, cookies);
    setCookies(refreshedCookies);
    
    // 3. Verify user access with new tokens
    MeResponse response = me();
    assertThat(response.getUser().getId()).isNotNull();
    
    // 4. Logout and clear cookies
    Map<String, String> clearedCookies = logout(user, refreshedCookies);
    assertThat(clearedCookies).isEqualTo(clearedCookies());
}
```

The test suite covers both tenant-aware and tenant-agnostic authentication flows, with known issues documented for 500 error responses that should return 401 on invalid token refresh attempts.
