An enumeration that defines the possible states of user invitations within the OpenFrame system.

## Key Components

- **PENDING** - Initial state when invitation is sent but not yet responded to
- **ACCEPTED** - Final state when recipient accepts the invitation
- **REVOKED** - Terminal state when invitation is cancelled or withdrawn

## Usage Example

```java
// Creating an invitation with pending status
InvitationStatus status = InvitationStatus.PENDING;

// Checking invitation state
if (status == InvitationStatus.PENDING) {
    // Send reminder email
    notificationService.sendReminder(invitation);
}

// Updating invitation status
invitation.setStatus(InvitationStatus.ACCEPTED);

// Switch statement for status handling
switch (invitation.getStatus()) {
    case PENDING:
        handlePendingInvitation(invitation);
        break;
    case ACCEPTED:
        activateUserAccount(invitation);
        break;
    case REVOKED:
        cleanupInvitation(invitation);
        break;
}
```

This enum provides type safety for invitation workflow management and ensures only valid status transitions occur throughout the invitation lifecycle.
