Provides cross-platform utilities for managing OpenFrame log directories with appropriate platform-specific paths and permissions.

## Key Components

- `get_log_directory()` - Returns platform-specific log directory paths (Windows: `%ProgramData%\OpenFrame\logs`, macOS: `/Library/Logs/OpenFrame`, Linux: `/var/log/openframe`)
- `ensure_log_directory()` - Creates the log directory with proper permissions and validates write access
- `can_write_to_directory()` - Tests directory write permissions by creating a temporary file

## Usage Example

```rust
use openframe::platform::{get_log_directory, ensure_log_directory};

// Get the platform-appropriate log directory
let log_path = get_log_directory();
println!("Log directory: {}", log_path.display());

// Ensure the directory exists with correct permissions
match ensure_log_directory() {
    Ok(path) => {
        println!("Log directory ready: {}", path.display());
        // Now safe to create log files in this directory
    }
    Err(e) => {
        eprintln!("Failed to setup log directory: {}", e);
    }
}
```

The module handles platform-specific requirements like macOS root:admin ownership and proper Unix permissions (755/775), while gracefully handling permission failures when not running with sufficient privileges.
