A client-side page component for registering new devices in the OpenFrame platform, providing automated installation commands for different operating systems.

## Key Components

- **NewDevicePage**: Main component handling device registration workflow
- **Platform Selection**: OS-specific installation command generation (macOS, Windows, Linux)
- **Organization Selector**: Dropdown for selecting target organization
- **Command Generator**: Builds platform-specific installation commands with registration secrets
- **Script Download**: Generates executable installation scripts for local execution
- **Antivirus Exclusions**: Shows required folder paths for antivirus software

## Usage Example

```typescript

export default function NewDevicePage() {
  // Component handles:
  // 1. Organization selection with auto-default selection
  // 2. Platform detection and command generation
  // 3. Registration secret management
  // 4. Binary URL construction from GitHub releases
  
  const command = useMemo(() => {
    // Generates platform-specific installation commands:
    // macOS: curl + tar + sudo execution
    // Windows: PowerShell with Invoke-WebRequest + Expand-Archive
    return buildInstallCommand(platform, serverUrl, initialKey, selectedOrgId)
  }, [platform, serverUrl, initialKey, selectedOrgId])
}

// Features include:
// - Automatic organization selection (defaults to "Default" org)
// - Cross-platform binary download from GitHub releases
// - Script generation for local execution
// - Clipboard integration for command copying
// - Toast notifications for user feedback
```

The component integrates with OpenFrame's core frontend components and provides a streamlined device onboarding experience with automated installation script generation and antivirus exclusion guidance.
