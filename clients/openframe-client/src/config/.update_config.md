Configuration constants that define timeouts, retry limits, and connection parameters for update system operations including binary downloads, consumer creation, and NATS message handling.

## Key Components

- **Download Configuration**: `MAX_DOWNLOAD_RETRIES`, `DOWNLOAD_TIMEOUT_SECS`, `MIN_BINARY_SIZE_BYTES` - Controls binary download behavior with retry limits, timeout duration, and minimum file size validation
- **Consumer Retry Settings**: `MAX_CONSUMER_CREATE_RETRIES`, `INITIAL_RETRY_DELAY_MS`, `MAX_RETRY_DELAY_MS` - Defines exponential backoff parameters for consumer creation attempts
- **Connection Management**: `RECONNECTION_DELAY_MS` - Sets delay between reconnection attempts
- **NATS Message Configuration**: `CONSUMER_ACK_WAIT_SECS`, `CONSUMER_MAX_DELIVER` - Controls message acknowledgment timeout and maximum delivery attempts

## Usage Example

```rust
use std::time::Duration;

// Configure download timeout
let timeout = Duration::from_secs(DOWNLOAD_TIMEOUT_SECS);

// Validate binary size
if binary_size < MIN_BINARY_SIZE_BYTES {
    return Err("Binary too small".into());
}

// Implement retry logic with exponential backoff
let mut delay = INITIAL_RETRY_DELAY_MS;
for attempt in 0..MAX_CONSUMER_CREATE_RETRIES {
    match create_consumer().await {
        Ok(consumer) => return Ok(consumer),
        Err(_) if attempt < MAX_CONSUMER_CREATE_RETRIES - 1 => {
            tokio::time::sleep(Duration::from_millis(delay)).await;
            delay = (delay * 2).min(MAX_RETRY_DELAY_MS);
        }
        Err(e) => return Err(e),
    }
}
```
