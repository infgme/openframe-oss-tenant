A data transfer object representing a user entity in the OpenFrame platform with essential authentication and authorization information.

## Key Components

- **User fields**: Core user attributes including ID, email, password, tenant ID, and display name
- **Role-based authorization**: List of roles for permission management
- **Lombok annotations**: Automatic generation of getters, setters, constructors, and builder pattern
- **Multi-tenant support**: Tenant ID field for isolating users across different organizations

## Usage Example

```java
// Create a new user using builder pattern
User user = User.builder()
    .id("user-123")
    .email("admin@example.com")
    .displayName("John Doe")
    .tenantId("tenant-456")
    .roles(List.of("ADMIN", "USER"))
    .password("hashedPassword")
    .build();

// Access user properties
String userEmail = user.getEmail();
List<String> userRoles = user.getRoles();
boolean isAdmin = user.getRoles().contains("ADMIN");

// Update user display name
user.setDisplayName("John Smith");
```

This DTO is typically used for transferring user data between API layers, authentication services, and the frontend, supporting OpenFrame's multi-tenant architecture with role-based access control.
