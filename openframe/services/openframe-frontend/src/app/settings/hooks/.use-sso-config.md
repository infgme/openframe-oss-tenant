A React hook that provides SSO (Single Sign-On) configuration management functionality for interacting with authentication provider APIs.

## Key Components

- **Types**: `AvailableProvider`, `ProviderConfig`, and `UpdateSsoPayload` for type-safe SSO data structures
- **useSsoConfig Hook**: Returns four memoized functions for managing SSO providers:
  - `fetchAvailableProviders` - Retrieves list of available SSO providers
  - `fetchProviderConfig` - Gets configuration for a specific provider
  - `updateProviderConfig` - Updates provider settings (client ID, secret, tenant ID, etc.)
  - `toggleProviderEnabled` - Enables/disables a provider

## Usage Example

```typescript
import { useSsoConfig } from './use-sso-config'

function SsoSettings() {
  const { fetchAvailableProviders, updateProviderConfig, toggleProviderEnabled } = useSsoConfig()

  const handleUpdateProvider = async () => {
    await updateProviderConfig('microsoft', {
      clientId: 'your-client-id',
      clientSecret: 'your-secret',
      msTenantId: 'tenant-id',
      autoProvisionUsers: true,
      allowedDomains: ['company.com']
    })
  }

  const handleToggleProvider = async () => {
    await toggleProviderEnabled('microsoft', true)
  }

  // Component implementation...
}
```

The hook uses proper error handling and URL encoding for provider names, making it safe for various SSO provider integrations including Microsoft Azure AD with tenant-specific configurations.
