GraphQL service for managing dialog and message operations in the Flamingo platform. Handles resumable dialogs and message retrieval with automatic token management and pagination.

## Key Components

- **DialogGraphQLService**: Main service class providing GraphQL operations for dialogs
- **ResumableDialog**: Interface for dialog metadata including status, ratings, and timestamps
- **Message/MessageData**: Extended message types with dialog mode and owner information
- **MessagesConnection**: Paginated message collection with cursor-based navigation
- **GraphQL Queries**: Pre-defined queries for dialog and message fetching

## Usage Example

```typescript
import { dialogGraphQLService } from './dialogGraphQLService'

// Get the current resumable dialog
const dialog = await dialogGraphQLService.getResumableDialog()
if (dialog) {
  console.log(`Dialog: ${dialog.title} (${dialog.status})`)
}

// Fetch messages for a dialog with pagination
const messages = await dialogGraphQLService.getDialogMessages(
  'dialog-123',
  null, // cursor (null for first page)
  10    // limit
)

if (messages) {
  messages.edges.forEach(edge => {
    const message = edge.node
    console.log(`${message.owner.type}: ${message.messageData}`)
  })
}

// Clean up resources
dialogGraphQLService.dispose()
```

The service automatically handles GraphQL client initialization, token management through `tokenService`, and provides error handling with fallback returns. It fetches all available messages by automatically paginating through results until completion.
