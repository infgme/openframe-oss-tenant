Utility functions for OpenFrame frontend URL management, handling environment-specific base URLs and asset path generation.

## Key Components

- **`getBaseUrl()`** - Returns environment-appropriate base URL (relative for browser, absolute for server)
- **`getAssetUrl(path)`** - Generates absolute URLs for static assets with proper path formatting

## Usage Example

```typescript
import { getBaseUrl, getAssetUrl } from './utils'

// Get base URL (environment-aware)
const baseUrl = getBaseUrl()
// Browser: returns ''
// Server: returns configured URL or 'http://localhost:4000'

// Generate asset URLs
const logoUrl = getAssetUrl('/images/logo.png')
const styleUrl = getAssetUrl('styles/main.css')

// Use in components
function Header() {
  return (
    <img src={getAssetUrl('/logo.svg')} alt="OpenFrame" />
  )
}
```

The utilities automatically handle browser vs server environments, using relative URLs in the browser for performance and absolute URLs on the server for SSR compatibility. Asset URL generation ensures consistent path formatting regardless of whether paths include leading slashes.
