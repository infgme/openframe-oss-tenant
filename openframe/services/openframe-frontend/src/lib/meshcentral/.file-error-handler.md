Handles error processing and management for MeshCentral file operations, providing standardized error classification, recovery logic, and user-friendly messaging for various file system and connection failures.

## Key Components

### Types
- **`FileErrorType`** - Union type defining 15+ error categories (auth, session, connection, file operations)
- **`FileError`** - Interface containing error type, message, details, timestamp, and retry flag

### Class: FileErrorHandler
- **`registerHandler()`** - Register custom error handlers for specific error types
- **`handleError()`** - Process errors through registered handlers and maintain history
- **`parseError()`** - Convert raw error messages to structured FileError objects
- **`createError()`** - Generate FileError instances from error strings
- **`isRetryable()`** - Determine if an error type supports retry operations
- **`getErrorHistory()`** - Access error history for debugging and analytics

## Usage Example

```typescript
// Initialize with global error callback
const errorHandler = new FileErrorHandler((error) => {
  console.log(`File error: ${error.message}`)
  if (error.retryable) {
    scheduleRetry()
  }
})

// Register custom handler for auth errors
errorHandler.registerHandler('autherror', (error) => {
  redirectToLogin()
})

// Process incoming error
const rawError = { error: 'permission denied', details: 'Access to /admin blocked' }
const fileError = errorHandler.parseError(rawError)
if (fileError) {
  errorHandler.handleError(fileError)
}

// Create custom error
const customError = errorHandler.createError('disk full', 'Upload size: 500MB')
errorHandler.handleError(customError)

// Check error analytics
const errorCounts = errorHandler.getErrorCounts()
console.log(`Auth errors: ${errorCounts.get('autherror') || 0}`)
```
