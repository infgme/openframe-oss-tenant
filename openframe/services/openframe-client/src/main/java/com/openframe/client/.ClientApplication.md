Main Spring Boot application entry point for the OpenFrame client application, responsible for bootstrapping the client service and configuring component scanning across multiple OpenFrame modules.

## Key Components

- **ClientApplication** - Main application class with Spring Boot configuration
- **@SpringBootApplication** - Enables auto-configuration, component scanning, and configuration
- **@ComponentScan** - Configures package scanning for dependency injection
- **Component exclusion** - Excludes CassandraHealthIndicator from automatic registration

## Usage Example

The application is started using standard Spring Boot patterns:

```java
// Application startup
public static void main(String[] args) {
    SpringApplication.run(ClientApplication.class, args);
}

// The component scan includes these packages:
// - com.openframe.data (data layer components)
// - com.openframe.client (client-specific components) 
// - com.openframe.core (core framework components)
// - com.openframe.security (security components)
// - com.openframe.kafka.producer (Kafka messaging)

// CassandraHealthIndicator is explicitly excluded:
@ComponentScan.Filter(
    type = FilterType.ASSIGNABLE_TYPE,
    classes = CassandraHealthIndicator.class
)
```

The application serves as the main entry point for the OpenFrame client service, automatically wiring dependencies from across the OpenFrame ecosystem while selectively excluding the Cassandra health indicator component.
