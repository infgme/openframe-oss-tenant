Centralized API client that handles authentication, token refresh, and provides a unified interface for making HTTP requests across the Flamingo platform with automatic retry logic for auth failures.

## Key Components

- **`ApiClient` class** - Main HTTP client with singleton pattern
- **`getAuthHeaders()`** - Manages Bearer token authentication from localStorage
- **`refreshAccessToken()`** - Handles automatic token refresh with request queuing
- **`request()`** - Core method with 401 handling and retry logic
- **HTTP convenience methods** - `get()`, `post()`, `put()`, `patch()`, `delete()`
- **`external()`** - For requests to external APIs outside the base URL
- **`me()`** - Shortcut for user profile endpoint

## Usage Example

```typescript
import { apiClient, type ApiResponse } from './api-client'

// Get user data
const response: ApiResponse<User> = await apiClient.get('/api/users/123')

if (response.ok) {
  console.log('User:', response.data)
} else {
  console.error('Error:', response.error)
}

// Create new resource
const createResponse = await apiClient.post('/api/tickets', {
  title: 'New ticket',
  priority: 'high'
})

// External API call
const externalData = await apiClient.external('https://api.example.com/data', {
  skipAuth: true
})
```

The client automatically handles token refresh on 401 responses, queues requests during refresh operations, and forces logout if authentication fails permanently.
