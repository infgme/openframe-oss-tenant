Zustand store for managing device state in the OpenFrame MSP platform, handling device inventory, filtering, sorting, and real-time status tracking with optimistic updates.

## Key Components

**Interfaces:**
- `Device` - Core device entity with status, metrics, and metadata
- `DeviceFilter` - Multi-criteria filtering options for device lists
- `DevicesState` - Complete store state including actions and computed data

**Store Actions:**
- `setDevices/addDevice/updateDevice/removeDevice` - CRUD operations for device management
- `selectDevice` - Device selection for detailed views
- `setFilter/setSorting/clearFilters` - UI state management for filtering and sorting
- `refreshDevices/setLoading/setError` - Async operation state handling

**Selectors:**
- `selectFilteredDevices` - Computed filtered and sorted device list
- `selectDeviceStats` - Real-time device status statistics

## Usage Example

```typescript
import { useDevicesStore, selectFilteredDevices, selectDeviceStats } from './devices-store'

function DeviceList() {
  const {
    devices,
    selectedDevice,
    setFilter,
    setSorting,
    selectDevice,
    updateDevice
  } = useDevicesStore()
  
  const filteredDevices = useDevicesStore(selectFilteredDevices)
  const stats = useDevicesStore(selectDeviceStats)
  
  // Filter by online servers only
  const filterOnlineServers = () => {
    setFilter({
      status: ['online'],
      type: ['server']
    })
  }
  
  // Update device status (real-time updates)
  const markDeviceOffline = (deviceId: string) => {
    updateDevice(deviceId, {
      status: 'offline',
      lastSeen: new Date().toISOString()
    })
  }
  
  return (
    <div>
      <div>Total: {stats.total}, Online: {stats.online}</div>
      {filteredDevices.map(device => (
        <div key={device.id} onClick={() => selectDevice(device)}>
          {device.name} - {device.status}
        </div>
      ))}
    </div>
  )
}
```
