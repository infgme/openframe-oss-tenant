This service module provides cross-platform installation and management of the OpenFrame client service with Windows SCM integration, automatic privilege checking, and configuration handling.

## Key Components

- **Service** - Main service struct handling installation, status checking, and cross-platform service management
- **windows_service_main()** - Windows Service Control Manager entry point with proper shutdown handling
- **InstallationInitialConfigService** - Handles initial configuration setup during installation
- **CrossPlatformServiceManager** - Manages service registration across Windows, macOS, and Linux
- **PermissionUtils** - Validates administrator/root privileges for installation

## Usage Example

```rust
use anyhow::Result;
use crate::service::Service;
use crate::installation_initial_config_service::InstallConfigParams;

// Check if service is already installed
if Service::is_installed() {
    println!("Service is already installed");
} else {
    // Install service with configuration
    let params = InstallConfigParams {
        // Configuration parameters here
    };
    
    Service::install(params).await?;
    println!("Service installed successfully");
}

// Create and run service instance
let service = Service::new();
// Service will be managed by platform-specific service manager
```

The service automatically handles binary installation to standard system locations (`/usr/local/bin` on Unix, `C:\Program Files\OpenFrame\bin` on Windows), manages existing installations through uninstall procedures, and integrates with platform service managers (Windows SCM, systemd, launchd).
