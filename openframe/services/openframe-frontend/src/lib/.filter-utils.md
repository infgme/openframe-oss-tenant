Unified filter utilities for table components across the platform, providing consistent deduplication and transformation of filter options from various backend formats.

## Key Components

- **FilterOption** - Interface defining the standard structure for filter options with `id`, `label`, and `value` properties
- **deduplicateFilterOptions()** - Generic function that removes duplicate filter options by ID using a Map for efficient deduplication
- **transformOrganizationFilters()** - Specialized function that normalizes organization data from different backend formats into consistent filter options

## Usage Example

```typescript
import { deduplicateFilterOptions, transformOrganizationFilters, FilterOption } from './filter-utils'

// Remove duplicates from filter options
const duplicateOptions: FilterOption[] = [
  { id: 'org-1', label: 'Acme Corp', value: 'org-1' },
  { id: 'org-1', label: 'Acme Corp', value: 'org-1' }, // Duplicate
  { id: 'org-2', label: 'Tech Inc', value: 'org-2' }
]

const uniqueOptions = deduplicateFilterOptions(duplicateOptions)
// Returns: [{ id: 'org-1', label: 'Acme Corp', value: 'org-1' }, { id: 'org-2', label: 'Tech Inc', value: 'org-2' }]

// Transform backend organization data
const backendOrgs = [
  { id: 'uuid-1', name: 'Acme Corp' },
  { value: 'uuid-2', label: 'Tech Inc' },
  { id: 'uuid-3', name: 'null' } // Special case handled as 'System'
]

const filterOptions = transformOrganizationFilters(backendOrgs)
// Returns normalized FilterOption[] with consistent structure
```

The utilities handle edge cases like null values, missing data, and different backend response formats while ensuring type safety through generic constraints.
