Provides consistent device status mapping and operating system identification utilities for the Flamingo OpenFrame platform, ensuring uniform status representation across table and grid views.

## Key Components

- **DeviceStatusVariant**: Type for status visual variants (`success`, `error`, `warning`, `info`, `critical`)
- **DeviceCardStatus**: Type for device card states (`active`, `inactive`, `offline`, `warning`, `error`)
- **DeviceStatusConfig**: Interface combining label, variant, and card status for complete status configuration
- **getDeviceStatusConfig()**: Maps device status strings to standardized configuration objects
- **getDeviceOperatingSystem()**: Resolves operating system types using centralized OS platform identification

## Usage Example

```typescript
import { getDeviceStatusConfig, getDeviceOperatingSystem } from './device-status'

// Get status configuration for a device
const statusConfig = getDeviceStatusConfig('ONLINE')
// Returns: { label: 'ONLINE', variant: 'success', cardStatus: 'active' }

// Use in component rendering
const DeviceRow = ({ device }) => (
  <tr>
    <td>{device.name}</td>
    <td>
      <StatusBadge 
        variant={getDeviceStatusConfig(device.status).variant}
        label={getDeviceStatusConfig(device.status).label}
      />
    </td>
  </tr>
)

// Get normalized OS platform ID
const osType = getDeviceOperatingSystem('Windows 11')
// Returns: 'windows'
```

The utility handles case-insensitive status mapping and provides fallback configurations for unknown statuses, ensuring robust status display across the application.
