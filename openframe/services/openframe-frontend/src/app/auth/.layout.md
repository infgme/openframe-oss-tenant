A minimal layout component for authentication pages that sets up metadata and provides a passthrough container for auth-related pages.

## Key Components

- **metadata**: Static metadata object defining page title and description for SEO
- **AuthLayout**: Default export function component that renders children without additional wrapper elements

## Usage Example

```typescript
// This layout automatically wraps authentication pages
// Pages placed in the same directory will inherit this layout

// Example auth page that uses this layout:
export default function SignInPage() {
  return (
    <div className="auth-container">
      <h1>Sign In</h1>
      <form>
        {/* Sign in form content */}
      </form>
    </div>
  )
}

// The layout ensures consistent metadata across all auth pages
// while allowing maximum flexibility for page-specific styling
```

The layout uses a fragment (`<>{children}</>`) to avoid adding unnecessary DOM elements, making it ideal for authentication flows where the page components need direct control over their container structure and styling.
