Main entry point for the OpenFrame agent that provides a command-line interface for installing, uninstalling, and running the OpenFrame client service with proper privilege checking and cross-platform support.

## Key Components

### Commands
- `Install` - Installs OpenFrame client as a system service with configuration parameters
- `Uninstall` - Removes the OpenFrame client service from the system
- `Run` - Executes the client directly without service wrapper
- `RunAsService` - Internal command used by service manager (hidden)
- `CheckPermissions` - Diagnostic command to verify required privileges

### Privilege Management
- `ensure_admin_privileges()` - Cross-platform function to verify admin/root privileges
- `check_capabilities_and_warn()` - Validates specific system capabilities and logs warnings

### Installation Arguments
- `InstallArgs` - Configuration struct containing server URL, initial key, organization ID, and local mode flag

## Usage Example

```rust
// Install OpenFrame as a service with configuration
./openframe install --serverUrl "https://api.example.com" --initialKey "key123" --orgId "org456"

// Run directly without service installation
./openframe run

// Uninstall the service
./openframe uninstall

// Check current permissions (diagnostic)
./openframe check-permissions
```

The application ensures proper privilege escalation before performing system-level operations and provides comprehensive logging throughout the service lifecycle. It supports both Unix-like systems (using UID checks) and Windows (using Windows API privilege verification).
