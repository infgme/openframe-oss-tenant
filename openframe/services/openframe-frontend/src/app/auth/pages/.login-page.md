A Next.js page component that handles user login flow with SSO providers and tenant discovery, automatically redirecting authenticated users and managing authentication state.

## Key Components

- **LoginPage**: Default exported React component that orchestrates the login flow
- **Authentication hooks**: Uses `useAuth` for login logic and `useAuthStore` for auth state
- **Navigation effects**: Automatic redirects based on authentication status and app mode
- **Tenant discovery**: Automatically discovers available tenants for a given email
- **SSO integration**: Handles single sign-on authentication with multiple providers

## Usage Example

```typescript
// This is a Next.js page component - no direct usage needed
// Access via route: /login

// The component handles these flows automatically:
// 1. Redirects authenticated users to dashboard (unless auth-only mode)
// 2. Discovers tenants for provided email
// 3. Redirects to /auth if no email provided
// 4. Renders login UI with available SSO providers

// Navigation flow:
// /auth → /login (with email) → /dashboard (after auth)
//    ↑                           ↓
//    └─────── back button ────────┘
```

The component integrates with the auth system to provide a seamless login experience, handling tenant discovery, SSO provider selection, and automatic navigation based on authentication state. It uses `AuthLayout` for consistent styling and `AuthLoginSection` for the actual login interface.
